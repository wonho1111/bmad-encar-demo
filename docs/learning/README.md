## 추천 학습 동선

**"데이터가 어디 있고 → 어떻게 화면에 나오나" 순서**가 가장 이해하기 쉽습니다:

```
00 개요 (전체 그림 잡기)
  ↓
01 DB (모든 것의 토대)
  ↓
04 웹 프론트 (그 데이터가 화면에 나오는 모습 — 가장 친숙)
  ↓
02 백엔드 API (AI 검색 요청 처리의 뼈대)
  ↓
03 LangGraph AI (이 프로젝트의 하이라이트)
  ↓
05 Flutter (같은 백엔드, 다른 화면 기술)
```

## 각 문서의 공통 구조

- **파일별 역할 설명** — 무슨 일을 하는 파일인지
- **함수/흐름 설명** — 데이터가 어떻게 흘러가는지
- **용어 풀이** — 처음 나오는 영어 약자·기술 용어를 한 줄로
- **핵심 개념 정리** — 그 파트에서 꼭 배워야 할 것 목록 (문서 맨 끝)

> 모르는 용어가 나오면, 각 문서 끝의 "핵심 개념 정리"부터 보세요.
