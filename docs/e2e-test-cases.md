# E2E 상세 테스트 케이스 (웹 + 모바일 + AI 스트레스 질의)

엔카 데모(중고차 직거래)의 **상세 E2E 테스트 케이스집**.
기존 `e2e-checklist.md`가 "체크 목록"이라면, 이 문서는 **각 케이스의 전제·절차·기대결과**까지 적은 실행 대본이다.

> *E2E(End-to-End)*: 사용자가 실제로 쓰는 흐름 전체를 처음부터 끝까지 통과시키는 테스트.
> *케이스(test case)*: "이 상황에서 이렇게 하면 이렇게 돼야 한다"를 한 줄 시나리오로 박은 검증 단위.

범례: ✅통과 / ❌실패 / ⚠️부분 / ➖해당없음
우선순위: **P0**(깨지면 데모 불가) / **P1**(핵심) / **P2**(보강)

## 3. Flutter 앱 테스트 방법 (에뮬레이터 없이)

> 질문: "브라우저로 해도 되는데 다른 방법 있을까? 에뮬레이터는 메모리가 없어서 안 될 듯."
> 답: **에뮬레이터 없이도 4가지 길**이 있고, 각각 검증하는 층이 다르다. 조합해서 쓴다.

*에뮬레이터(emulator)*: PC에 가상 스마트폰을 통째로 띄우는 도구 — RAM을 많이 먹어 6GB PC에선 부팅이 막힌다([[epic7-mobile-test-strategy]]에 실측: Pixel_7 offline 멈춤).

| 방법 | 무엇을 검증 | 메모리 부담 | 한계 |
|---|---|---|---|
| **① `flutter test` (위젯/단위)** | 로직·폼검증·모델·필터·dedupe 등 **화면 없는 검증** | 거의 없음(Dart VM) | 실제 렌더·터치·네이티브 권한은 못 봄 |
| **② Flutter 웹빌드 + Playwright/Chrome** | UI 흐름·클릭·입력 **E2E(개발 중)** | 낮음(브라우저만) | CanvasKit headless 렌더 이슈 → **headed(실제 크롬)** 로 |
| **③ 실제 폰 USB + mobile-mcp** | 네이티브 **최종 검증**(SafeArea·INTERNET권한·키보드) | PC 0(폰이 실행) | 폰·USB 연결 필요 |
| **④ 클라우드 디바이스 팜** | 실기기 자동 테스트(폰 없을 때 대안) | PC 0(클라우드) | 무료한도/설정·업로드 필요 |

*Playwright*: 실제 브라우저를 자동 조작하는 테스트 도구. *mobile-mcp*: 실기기/에뮬을 자동 조작하는 모바일판 MCP. *디바이스 팜(device farm)*: Firebase Test Lab·BrowserStack 등 클라우드의 진짜 기기에 앱을 올려 돌리는 서비스.

### 권장 조합 (이 PC 기준)
1. **개발 중 로직**: `cd app && flutter test` — 이미 위젯/단위 테스트 11종 있음(`app/test/`). 가장 빠르고 공짜.
2. **개발 중 UI E2E**: `flutter build web` 후 로컬 서버로 띄워 **claude-in-chrome / Playwright(headed)** 로 §2.4~2.7 흐름 점검.
   - ⚠️ `--web-renderer html`은 최신 Flutter에서 제거됨 → CanvasKit으로 가되 **headless 금지**(실제 크롬 창에서 렌더 확인).
3. **에픽 종료 최종 검증**: **실제 안드로이드 폰 USB 연결 + mobile-mcp** 로 §2.8 M-NATIVE(SafeArea·INTERNET·키보드·색상) 회귀. ← 메모리 무관, 이게 네이티브 진짜 확인.
4. **폰이 없을 때 대안**: **Firebase Test Lab**(무료한도 내 실기기) 에 release APK 업로드 → 스모크.
   - *스모크 테스트(smoke test)*: "켜지고 핵심만 돌아가나"를 빠르게 보는 1차 점검.

> 결론: **브라우저(②)로 UI 흐름은 충분히 자동검증 가능**하지만, SafeArea·release 권한 같은 **네이티브 고유 항목은 실폰(③)** 이 정답. 둘을 합치면 에뮬레이터 0으로 전부 커버된다.

## 4. AI 스트레스 질의셋 (20+개, 기존보다 복잡 / 의도적 비표준)

> 목적: 우리가 **프롬프트에 명시적으로 예외처리한 형식**(아라비아 숫자 `3천만원`·`이하/이상`·`세단→IN`·옵션 `ANY`·금융→C)을 **일부러 피해**, 모델이 비표준 표현을 얼마나 견디는지 본다.
> 그래서 일부는 **"통과해야 함(should-pass)"**, 일부는 **"스트레스(may-degrade, 깨질 수도 있음 — 그게 발견 목적)"** 로 표시한다.
> 머신용 파일: `api/docs/ai-e2e-hard-queryset.json` (`run_ab_eval.py`로 자동 순회 가능).

> 경로: **A**=Text-to-SQL(DB검색) / **B**=가이드 문서 RAG / **C**=거절(도메인 밖).
> *Text-to-SQL*: 자연어를 DB 질의문(SQL)으로 바꿔 실제 매물을 찾는 방식. *RAG(Retrieval-Augmented Generation)*: 문서를 찾아 근거로 답을 생성하는 방식.

### 4.1 구조형(A)이지만 비표준 표현 — "숫자·단위·부정·그룹핑" 함정

| ID | 질의 | 기대경로 | 노리는 함정 | 기대/관찰 | 난이도 |
|---|---|---|---|---|---|
| H01 | 삼천만원 이하 SUV 찾아줘 | A | **한글 숫자**(삼천) → 30,000,000 | price≤3천 SUV. 한글수사 파싱 실패 시 가격조건 누락 | 스트레스 |
| H02 | 2천에서 3천만원 사이 세단 | A | **범위(between)** 표현(`사이`) | price 20M~30M AND 세단 IN. 프롬프트엔 이하/이상만 | 스트레스 |
| H03 | 1억 이하 수입차 보여줘 | A | **억 단위**(미정규화) + **수입차**(국산 제외) | price≤100,000,000 AND manufacturer IN(BMW·벤츠·아우디·폭스바겐·토요타·혼다·렉서스·테슬라) | 스트레스 |
| H04 | 주행거리 5만 키로 안 넘는 차 | A | **구어 단위**(키로) + **부정 표현**(안 넘는=이하) | mileage≤50000 | 스트레스 |
| H05 | 뽑은 지 3년 안 된 차 있어? | A | **상대 연식**(나이→절대연식) | year≥2023 (2026 기준). "뽑은 지"를 연식으로 해석하는가 | 스트레스 |
| H06 | 그랜저급 세단 보여줘 | A | **모델명으로 차형 암시** | 그랜저≈대형/준대형 → 대형차(또는 세단 IN). 매핑표 밖 | 스트레스 |
| H07 | 국산차 중에 제일 비싼 거 | A | **국산 그룹핑** + 최상급 | manufacturer IN(현대·기아·제네시스·쉐보레·KG모빌리티·르노코리아) ORDER price DESC LIMIT 1 | 스트레스 |
| H08 | 수도권에 있는 전기차 | A | **지역 그룹핑**(수도권) | region IN(서울·경기·인천) AND fuel=전기 | 스트레스 |
| H09 | 디젤 빼고 SUV 보여줘 | A | **제외/부정**(fuel≠디젤) | body_type=SUV AND fuel != '디젤' | 스트레스 |
| H10 | 온 가족 6명이 탈 수 있는 차 | A | **인승을 사람수 구어로** | seats≥6 | 통과기대 |
| H11 | 3천만원대 차 보여줘 | A | **"대" 접미 범위** | price 30M~39.99M (정확히 "대"를 범위로). 흔히 ≤3천으로 오해 | 스트레스 |
| H12 | 가격 싼 순으로 정렬해서 보여줘 | A | **정렬만**(최상급 단어 없음) | ORDER price ASC, LIMIT 기본. "제일/가장" 없이 정렬 인식 | 통과기대 |
| H13 | 흰색 말고 다른 색 SUV | A | **색상 제외** | body_type=SUV AND color != '흰색' | 스트레스 |
| H14 | 무사고에 옵션 많은 차 추천 | A/B | **모호 옵션량**(옵션 많은) + 무사고 | accident_free=true는 확실. "옵션 많은"=무리한 SQL 금지(가드) | 스트레스 |
| H15 | 2~3년 된 기아 SUV | A | **상대연식 범위** + 제조사+차형 | year 2023~2024 AND 기아 AND SUV | 스트레스 |

### 4.2 의미형(B) 변형 — 캐주얼 어투(라우팅 헷갈림)

| ID | 질의 | 기대경로 | 노리는 함정 | 기대/관찰 | 난이도 |
|---|---|---|---|---|---|
| H16 | 처음 차 사는데 뭐 사지? | B | **추천 단어 없는 구어** | 초보 가이드(B). 조건 0이라 A로 새면 안 됨 | 통과기대 |
| H17 | 출퇴근만 하는데 기름값 안 깨지는 거 뭐 있어? | B | **연비를 구어로** | 연료/연비 가이드(B). "기름값"을 조건화 무리 안 함 | 스트레스 |

|---|---|---|---|---|---|
| H18 | 월 50만원 할부로 살 수 있는 차 보여줘 | C | **금융(할부)+월납**(데이터 없음) | 거절+갈림길 멘트(월납 컬럼 없음). "차 보여줘"에 끌려 A로 새는지 | 스트레스 |

### 4.4 복잡 멀티턴 (비표준 누적·전환·서수)

| ID | 턴 | 기대경로 | 노리는 함정 | 기대/관찰 |
|---|---|---|---|---|
| **H21-멀티** | 1) 수입 세단 보여줘 | A | 수입 그룹핑+세단 | manufacturer IN(수입) AND 세단 IN |
| | 2) 그중 검정말고 | A(REFINE) | **누적+색상 제외** | 직전 조건 유지 + color != '검정' |
| | 3) 제일 싼 걸로 | A(REFINE) | 누적+최저가 | 위 조건 ORDER price ASC LIMIT 1 |
| **H22-멀티** | 1) 삼천 이하 SUV | A | 한글숫자 | price≤3천 AND SUV |
| | 2) 아니 그냥 패밀리카 추천해줘 | B(RESET) | **주제전환 — 직전 SQL 폐기** | 패밀리 가이드. ⚠️오염게이트: 결과/SQL에 SUV·price 조건 남으면 위반 |
| | 3) 그런 차 중에 7인승 실매물 | A(REFINE) | B→A 전환+인승 | seats≥7 (패밀리=RV/SUV/승합 맥락 허용) |
| **H23-멀티** | 1) 사회초년생 탈만한 차 | B | 소프트 추천 | 초보 가이드(B) |
| | 2) 수입차로만 보여줘 | A | **소프트 의도 유지?** | 알려진 quirk([[ai-search-known-quirks]]): "저렴" 의도 소실 시 고가 수입차 혼입. 관찰 포인트 |

### 4.5 채점·게이트(스트레스 질의 기준)
- **통과기대(should-pass)**: 라우팅·결과집합 정확해야. 틀리면 **버그**.
- **스트레스(may-degrade)**: 깨지면 **개선 백로그**(즉시 버그 아님) — 단 다음 두 게이트는 깨지면 안 됨:
  - **오염 게이트**: RESET(H22-2) 후 직전 조건이 남으면 위반.
  - **dead-end 게이트**: C 거절(H18~H20)이 빈손이면 위반 — 반드시 갈림길 멘트로 유도.
- 권장: 각 질의 N=3 반복으로 flaky(같은 입력 다른 결과) 여부도 본다.
  - *flaky*: 같은 조건인데 실행마다 결과가 들쭉날쭉한 불안정성.
