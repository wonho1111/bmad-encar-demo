# AI 검색 A/B 테스트 질의셋 (Phase B 입력)

Gemini 생성모델 비교(예: `gemini-3.1-flash-lite` vs `gemini-2.5-flash-lite`)를 **실측**하기 위한 질의 모음이다.
기계가 읽는 정식 파일은 `ai-ab-test-queryset.json`이고, 이 문서는 그 개요다.

## 데이터 스냅샷 (기대 결과의 근거)
- `listings` **100건** (on_sale 94), `guide_documents` **12개** 기준.
- 각 질의의 "기대 결과"는 2026-06 실제 DB 카운트로 검증했다(아래 숫자는 그 시점 값).

## 경로(route) 의미
- **A = Text-to-SQL**: 조건 필터·정렬로 실제 매물을 찾는 질의. (가격·지역·차종·연료·주행거리 등)
- **B = 문서 RAG**: 상식·가이드 질문. `guide_documents` 임베딩 검색으로 답함.
- **C = 거절(guard)**: 중고차 검색 범위 밖. LLM 호출 없이 고정 거절 템플릿.

## 채점 기준 (Phase B)
1. **라우팅 정확도** — 모델이 고른 경로가 `route`와 일치하는가. (오분류가 데모를 깨는 1순위 지표)
2. **결과 정확도** — Path A는 기대 조건/최상급 매물 일치, Path B는 기대 문서 매칭, Path C는 거절 여부.
3. **멀티턴 맥락 유지** — 후속 질문에서 앞 조건을 누적/전환하는가.

## 구성 (총 24항목)
| 그룹 | 개수 | 예시 |
|---|---|---|
| Path A (단일) | 10 (A1~A10) | "3천만원 이하 SUV"(23건), "제일 싼 차"(쉐보레 올란도 490만), "디젤 SUV 7인승"(7건) |
| Path B (단일) | 7 (B1~B7) | "초보 첫차?", "전기차 유지비?", "할부 vs 리스?", "취득세 얼마?" |
| Path C (단일) | 3 (C1~C3) | "오늘 날씨?", "김치찌개 레시피", "비트코인" → 거절 |
| 멀티턴 | 4 (M1~M4) | M1: "3천이하 SUV"→"서울만"(3건)→"제일 싼 거"(**현대 코나 2350만**) |

### 멀티턴 하이라이트
- **M1**: 조건 누적(SUV→+서울→최저가). 정답 = 현대 코나 23,500,000원.
- **M2**: 전기차→+5천이하→최저가. 정답 = 현대 코나 일렉트릭 26,000,000원.
- **M3**: **경로 전환 테스트** — turn1 "초보 추천"(B, 가이드) → turn2 "그런 차 1500만 이하 실제 매물"(A, SQL).
- **M4**: 하이브리드 중형(4건)→+무사고.

## 사용법 (Phase B, 크레딧 충전 후)
1. `GEMINI_GENERATION_MODEL`을 후보 모델 A로 두고 24항목 전부 호출 → 라우팅·결과 기록.
2. 모델 B로 바꿔 동일 반복.
3. 두 모델의 ① 라우팅 정확도 ② 결과 정확도 ③ 응답 지연 ④ 실제 청구액을 표로 비교.
