# Story 7.2: 구매자 — 매물 탐색·상세·AI 검색

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 구매자(모바일),
I want 앱에서 매물을 필터로 검색하고 상세를 열람하며 AI 채팅으로 자연어 검색을 하고 싶다,
so that 폰으로 판매 중인 차를 찾고(FR9·10), 자연어 한 문장으로 매물을 추천받을 수 있다(FR12~18).

## Acceptance Criteria

1. **(AC1 — 매물 목록·필터, FR9·11 재현)** 앱 홈에서 진입하는 탐색 화면에서 `listings`를 조회해 **`on_sale` 매물만** 카드로 보여준다. 키워드(모델명 부분일치)·차종·색상·연료·변속기·지역·가격범위·연식범위 필터를 적용하면 그 조건으로 좁혀진다. 0건이면 "조건에 맞는 매물이 없습니다" 안내, 조회 실패면 그와 구분되는 한국어 에러를 보여준다.
   - *근거*: epics.md 7.2 "필터 검색/상세 조회를 하면 on_sale 매물만 보인다". FR11 단일 규칙 = web `buyerListingsQuery`(`.eq('status','on_sale')`)를 Flutter 쿼리로 이식. 사진 없음.

2. **(AC2 — 매물 상세, FR10·11 재현)** 카드를 누르면 그 매물의 상세(FR5 15필드 + 설명·옵션·상태)를 사진 없이 표시한다. 조회는 **`on_sale` + 해당 id**로만 — sold·삭제·없는 id면 "찾을 수 없습니다(판매완료·삭제 가능)" 안내. 조회 실패는 별도 한국어 에러. 단위는 원·km·cc(천단위 콤마).
   - *근거*: epics.md 7.2 "상세 조회 시 on_sale만". web `listings/[id]/page.tsx`의 maybeSingle + FR11 이중 방어 패턴 이식.

3. **(AC3 — AI 채팅, FR12~18 재현)** 전역 진입(FAB 또는 탭, nav-ia §2 R3)으로 여는 AI 채팅에서 자연어 질의를 보내면 **`/ai/search`를 호출**해 `{answer, listings[]}`를 받아 자연어 답변 + 매물카드로 렌더한다. 멀티턴(FR18): 직전 대화를 **클라이언트 상태로만 보관**하다 후속 질의에 `context`로 동봉(최대 12턴, content 2000자, query 1000자 — web 계약 동일). 매물카드를 누르면 상세(AC2)로 이동. 0건이면 answer만, 실패면 한국어 에러(fail-loud).
   - *근거*: epics.md 7.2 "자연어 질의 → /ai/search → answer+매물카드". web `aiSearch.ts`(요청/응답·헤더·에러)·`ChatAssistant.tsx`(멀티턴·롤백) 계약 동일 이식.

4. **(AC4 — 인증 헤더·환경변수)** `/ai/search` 호출은 `Authorization: Bearer <supabase access_token>` 헤더를 동봉한다(매 요청 현재 세션 토큰). API 주소는 코드 하드코딩 금지 — `String.fromEnvironment('API_BASE_URL')`(dart-define 주입, 7.1 SUPABASE_* 패턴과 동일)로 읽고, 누락 시 한국어로 "무엇이 비었는지" 알린다.
   - *근거*: 7.1 `env.dart`·`aiSearch.ts getApiBaseUrl` 선례. 값은 web `NEXT_PUBLIC_API_BASE_URL`과 동일(Cloud Run dev `encar-ai-api-dev` URL). anon이 아니라 사용자 JWT를 보냄(api는 사용자 토큰으로 RLS 읽기전용 검증).

5. **(AC5 — 네비·복귀·검증)** 홈(7.1)에 탐색·AI 진입 동선을 추가한다(nav-ia §1·§2: ②탐색 ③AI 전역). 상세에서 뒤로가면 출처(탐색/AI결과)로 복귀(R5). `flutter analyze`(0 error)·`flutter test`(목록 필터 파싱·매물 매핑·context 직렬화 등 단위 테스트)·`flutter build web`이 통과한다. 가능하면 listings 조회·`/ai/search`를 live HTTP 스모크로 실증(브라우저 픽셀 렌더는 시스템 라이브러리 미비 시 빌드+테스트로 갈음, 그 사실 표기).
   - *근거*: nav-ia §3 R5, 입력 프롬프트 테스트 전략(실폰·에뮬 없음 → build web + test).

## Dev Notes

### 핵심 계약(이월 자산 — 코드 아님, 규칙·계약·동작만)

**매물 가시성(FR11 단일 규칙):**
- 구매자에게는 `status='on_sale'`만. DB RLS(0002_listings)는 on_sale을 모두에게 공개하되 본인 매물(own)·admin도 통과시키므로(OR), 판매자가 본인 sold를 구매자 경로로 보면 샐 수 있다. → **앱 쿼리에 `.eq('status','on_sale')`을 명시**해 구매자 관점을 강제(이중 방어). [Source: web/src/lib/listings.ts:14-31, 0002_listings.sql:90-101]
- 시작점 함수 하나(`buyerListingsQuery` 상당)에서만 비롯되게 — 목록·상세 공유. 컬럼 문자열만 경로별로 다르게(목록=요약 7필드, 상세=15필드+status+seller_id).

**필터 입력 검증(web search/page.tsx 이식):**
- 키워드 = `model` 컬럼 `ilike '%kw%'`, LIKE 메타문자(`\ % _`) escape(안 하면 `%`가 "전부 일치"로 필터 무력화). [Source: web/src/app/(user)/search/page.tsx:49-51, 104]
- 드롭다운 = 목록에 있는 값일 때만 적용(목록 밖 값 무시 — 쿼리 오염 방지). 빈 값=미적용. [pickOption]
- 정수 = 순수 숫자열(`/^\d+$/`)·MAX_SAFE_INTEGER 이하만 통과(지수·16진·음수·소수 거름, bigint 범위 초과 방지). [asInt]
- price/year min>max면 swap(역전 입력 0건 혼란 방지). 한쪽만 있으면 그쪽만(gte/lte). [page.tsx:89-95, 113-116]
- 정렬 = `created_at desc, id desc`(시드 동시각 행 순서 안정화). [page.tsx:120]

**AI 검색 계약(web aiSearch.ts·ChatAssistant.tsx + api schemas/ai.py 동일):**
- `POST {API_BASE_URL}/ai/search`, headers `Authorization: Bearer <access_token>` + `Content-Type: application/json`, body `{query, context?}`. [Source: web/src/lib/api/aiSearch.ts:7-12]
- 200: `{answer: string, listings: ListingCardData[]}`. listings 원소 = id·manufacturer·model·year·price·mileage·region(7필드). 깨진 원소(필드 누락·타입 불일치)는 버린다(런타임 가드 — 안 하면 카드 렌더 도중 터져 화면 전체 날아감). [aiSearch.ts:91-115]
- 비200: `{error:{code,message}}` → message 우선, 없으면 상태코드 폴백(401 로그인필요/400·422 형식/5xx 일시오류). 네트워크 실패 → "서버 연결 실패". [aiSearch.ts:78-134]
- 멀티턴(FR18, 무상태): 대화는 클라 상태에만 — 새로고침/재진입 시 초기화. 후속 질의에 직전 대화를 `context`로 동봉. 최근 12턴(`max_length=12` 서버 강제)·각 content 2000자(`min_length=1, max_length=2000`)·query 1000자(`MAX_QUERY_LENGTH`). 빈(공백) 턴은 context에서 제거(min_length=1 위반→422 자초 방지). [Source: web/src/components/ai/ChatAssistant.tsx:22-51, api/app/schemas/ai.py:13,38-51]
- 전송 실패 시 낙관적 user 버블 롤백 + input 복원(답 없는 질의가 대화에 남거나 다음 context로 중복되는 것 방지). [ChatAssistant.tsx:98-108]
- 토큰은 매 전송 시 현재 세션에서 조회(만료 자동 갱신). web=`getSession()`, Flutter=`supabase.auth.currentSession?.accessToken`.

**네비·복귀(nav-ia-rules):**
- §1 R2: 홈은 1순위 과업(매물 탐색)으로 바로 착지. §2 R3: AI는 전역 진입(Flutter=FAB 또는 상시 탭). §3 R5: 상세 뒤로가기 = 출처 복귀(시스템 back) + 폴백(탐색). [Source: _bmad-output/planning-artifacts/nav-ia-rules.md §1·§2·§3]
