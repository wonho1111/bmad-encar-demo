## Story

As a 사용자,
I want 데스크톱·모바일 어디서든 화면이 안 깨지고 로딩·빈·에러 상태가 명확하길,
so that 어떤 상황에서도 서비스를 신뢰하고 쓸 수 있다.

> **이 스토리의 성격 (개발자 필독):** 8.1(토큰)에 이은 **두 번째 파운데이션 스토리**다. "화면"을 만드는 게 아니라, 뒤 에픽(9~16)이 상속할 **재사용 프리미티브 3종**을 `web/src/components/ui/`에 깐다 — (1) **반응형 그리드**(D5 무결성: 열 수로만 흡수), (2) **상태 프리미티브**(스켈레톤·빈 상태·에러 폴백), (3) **접근성 프리미티브**(포커스링·포커스 트랩 + 전역 접근성 규칙). **기존 화면(검색·상세·채팅·관리자 목록 등)을 이 프리미티브로 갈아끼우는 건 이 스토리 범위가 아니다** — 각 UI 에픽(9 카드·11 랜딩·12 채팅·15 관리자)이 소비한다. A3 외과적 변경: `ui/` 신규 프리미티브 + `globals.css` 전역 접근성 규칙만 건드린다. 검증은 8.1처럼 **임시 하니스**로 3개 뷰포트에서 직접 관찰(소비 화면이 아직 없으므로).

## Acceptance Criteria

1. **(반응형 그리드 — D5·FR36·UX-DR21)** 단일 코드 반응형 그리드 프리미티브(`<ResponsiveGrid>`)가 뷰포트 폭에 따라 **열 수만** 바꾼다: `≥1100px=4열` · `640~1099px=2열` · `<640px=1열`. 컴포넌트 **내부 가로 배치(신뢰속성 행·meta·옵션 칩·버튼 라벨)를 세로화하지 않는다** — 공간 부족은 `truncate`·"외 N"·열 축소로만 처리(줄바꿈 찌그러짐·2줄로 밀리는 버튼 = 금기). 가장자리 카드 부분 클리핑만 허용.

2. **(모바일 캐러셀 프리미티브 — UX-DR21)** `<640px`에서 캐러셀이 필요한 맥락(랜딩 인기/최신)을 위한 별도 프리미티브(`<CardCarousel>`)가 제공된다 — 가로 스크롤·스냅으로 **1.2장 노출**(다음 카드가 살짝 보여 스크롤 가능 신호). 캐러셀 내부도 카드 내부 배치를 세로화하지 않는다. (전체 목록은 1열 그리드를 쓰고, 랜딩만 캐러셀을 쓴다 — 소비처가 선택.)

3. **(상태 프리미티브)** 공용 상태 프리미티브가 정의되어 카드·목록·상세·랜딩·채팅이 상속한다: **스켈레톤 로딩**(`<Skeleton>` + 카드/행 조합), **빈 상태**(`<EmptyState>` — 아이콘 + 문구 + 선택적 CTA), **에러 폴백**(`<ErrorState>` — 문구 + 재시도). 문구는 EXPERIENCE.md Voice·States 표를 정본으로 하되, 프리미티브는 문구를 **props로 받는다**(하드코딩 금지 — 소비처가 맥락 문구 주입).

4. **(접근성 내장 — UX-DR22)** 접근성 기준이 프리미티브·전역 규칙에 내장된다:
   - **비색 신호 중복**: 색 단독 신호 금지(색 + 아이콘/텍스트). 상태·신뢰·선택 신호에 반영.
   - **터치 타깃 ≥44px**(주요 버튼 ≥52px), 아이콘 버튼·칩 히트영역 ≥44×44px — 프리미티브의 인터랙티브 요소에 최소 크기 내장.
   - **포커스링 = `brand-petrol`**: `:focus-visible`에 전역 링(globals.css), 웹 전 인터랙션 키보드 도달.
   - **포커스 트랩**(`<FocusTrap>` 또는 훅): 모달·바텀시트·드롭다운·로그인 게이트용 — 열림 시 포커스 이동+트랩, Esc 닫힘, 닫힐 때 트리거로 포커스 복귀.
   - **한국어 스크린리더 라벨**: 프리미티브의 아이콘/상태에 `aria-label`·`role`·`aria-live` 적용점 마련(빈/에러 상태 announce, 로딩 `aria-busy`).
   - **reduced-motion**: `prefers-reduced-motion: reduce` 시 스켈레톤 펄스·전환 축소/제거(전역 규칙 + 프리미티브 준수).

5. **(D5 무결성 검증)** 대표 뷰포트(데스크톱 ~1280 · 태블릿 ~834 · 모바일 ~390)에서 그리드·캐러셀·상태 프리미티브를 **대표 카드형 콘텐츠**로 렌더해 D5 무결성 위반(줄바꿈 어긋남·2줄로 밀리는 버튼·라벨 어긋남)이 **없음**을 직접 관찰·스크린샷으로 확인한다(B4). 포커스링·포커스 트랩·reduced-motion 동작도 함께 관찰. `next build`가 통과한다.
