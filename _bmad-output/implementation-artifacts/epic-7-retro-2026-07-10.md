## 2. 잘된 점 (What went well)

1. **계약 이식 전략 — 코드가 아니라 규칙을 옮겼다.**
   RLS·트리거·CHECK 제약이 DB에 박혀 있어, Flutter 앱은 재구현 없이 "통과"만 하면 됐다. 7.6에서 앱이 만든 데이터가 web과 동일 스키마/RLS/트리거를 탄다는 것을 라이브 실증(방 생성·재사용·본인매물 차단·sold FR11 3역할 임퍼소네이션).

2. **위젯 재사용으로 버그 표면 축소.**
   7.4 수정 폼을 신규 작성하지 않고 7.3 등록 폼(`SellScreen`)에 `editDetail` 파라미터를 더해 등록/수정 단일 위젯으로 통합. 신규 폼·신규 검증 0줄.

3. **'0 영향행(0 rows)' 패턴 정확성.**
   Supabase가 RLS로 막힌 쓰기를 예외가 아닌 0행으로 돌려주는 점을 이용해, `.select('id')` 길이로 0행을 잡아 한국어 거부로 분기. 거짓 성공 없음을 교차 소유주 테스트로 실증(B가 A 매물 수정→200·0행).

4. **정직한 검증 보고.**
   못 한 항목을 "했다"로 포장하지 않고 escalate로 명시 + 데이터/계약 레벨 갈음 근거를 남김. 테스트 행은 생성 후 정리(DB 원상복구, 잔존 0).

5. **이전 회고(Epic 6) 액션 이행.**
   Winston의 `nav-ia-rules.md`(역할별 내비/정보구조 규칙)를 근거로 buyer/seller→공통 홈, admin→모바일 차단(AR9) 구현. web UI 코드는 한 줄도 이식하지 않고 성공 — "이월되는 건 React 코드가 아니라 규칙 문서" 결정이 검증됨.

## 3. 아쉬운 점 / 반복된 함정 (What didn't)

1. **[구조적] 실기기·에뮬레이터·데스크톱 Chrome 부재 → 픽셀 E2E 6개 스토리 전부 미수행.**
   이 PC RAM 6GB로 에뮬레이터 부팅 불가, Playwright 픽셀 렌더는 시스템 라이브러리(libnspr4 등) 미비로 불가. 모든 스토리가 "픽셀 E2E 미수행 → 빌드 산출물 정적검증 + live HTTP 스모크로 갈음"이라는 동일 각주로 종료. → **"화면이 실제로 그렇게 보이는가"는 미검증 부채로 이월.**

2. **[구조적] AI 검색 라이브 호출 0회.**
   `app/.env.json`의 `API_BASE_URL`(Cloud Run dev URL)이 빈 값이라 7.2·7.6 모두 escalate. 앱↔web↔api 3자 계약 바이트 일치 정적 증명으로 갈음. → **앱에서 AI 검색을 실제로 눌러 카드가 뜨는 장면 미검증.**

3. **[기술] 최신 메이저 버전 파괴적 변경 사전 인지 실패.**
   `flutter_riverpod 3.x`가 `StateNotifier`를 legacy로 분리 → 7.1 초기 analyze 14건. 모던 `Notifier` API로 전환해 해소했으나, 라이브러리 메이저 버전 변경점을 착수 전에 못 봄.

4. **[테스트] 컨트롤러 단위 테스트 부재.**
   전역 Supabase 의존 때문에 컨트롤러 단위 테스트를 못 쓰고 live 스모크로 갈음(7.4 등). fake 클라이언트 도입 시 보강 가능.

## 4. 코드리뷰에서 반영된 실제 결함 (교훈 소재)

- **7.1** — 중복 이메일 가입 안내 버그(`code`가 아닌 `message`에 넣어 한국어 변환 실패) + 로그인 거짓성공 방지(`res.session == null` 가드) → 패치.
- **7.4** — (High) `updateListing`에 `.eq('status','on_sale')` 전제조건 추가(동시 sold 덮어쓰기 차단) / (High) 액션 시작 시 `actionError` 초기화 / (Med) `editingIdOverride`로 첫 프레임 제출이 등록으로 새는 경합 차단.
  - **교훈:** web에서 이미 존재하던 안전장치(setError(null), 상태 빗장)를 모바일 포팅 시 누락 → 리뷰가 잡음. **"web 계약 이식 = UI뿐 아니라 방어 로직까지"** 체크리스트화 필요.

## 7. 핵심 교훈 (Key takeaways)

1. **DB에 계약을 박아두면 멀티플랫폼 이식 비용이 급감한다** — Flutter는 RLS/트리거/CHECK를 재구현 없이 통과만.
2. **web 포팅 시 UI뿐 아니라 방어 로직(가드·에러 초기화·상태 빗장)까지 이식 체크리스트로 관리** — 리뷰가 3건 잡음.
3. **검증 환경 한계는 정직하게 escalate하고 갈음 근거를 남긴다** — 단, 갈음은 부채이지 완료가 아님. 픽셀 E2E·AI 라이브는 실기기·URL 확보 후 반드시 마감.
4. **최신 메이저 라이브러리는 착수 전 breaking change 확인** (Riverpod 3 Notifier).
