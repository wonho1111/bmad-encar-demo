## Story

As a 차장님 서비스 사용자,
I want 전 화면이 일관된 페트롤+앰버 디자인 언어(색·타이포·그림자)로 보이길,
so that 데모 티 안 나고 믿음직한 제품으로 느껴진다.

> **이 스토리의 성격 (개발자 필독):** 이건 "화면"을 만드는 스토리가 아니라, **뒤 에픽(9~16) 전부가 딛고 설 디자인 토큰·유틸·폰트·로고 파운데이션**을 까는 스토리다. 눈에 보이는 산출물은 (1) 새 `globals.css`(색·타이포·그림자·radius 토큰), (2) Pretendard가 로드된 `layout.tsx`, (3) 재사용 가능한 `<Logo>` 컴포넌트 1개다. **기존 화면(Button·AppHeader·검색·상세 등)을 이 토큰으로 갈아끼우는 건 이 스토리 범위가 아니다**(각 UI 에픽이 소비). A3 외과적 변경 — 딱 이 3개만 건드린다.

## Acceptance Criteria

1. **(색 토큰 중앙화 — UX-DR1)** Tailwind v4 `@theme`(CSS-first) 환경에서 DESIGN.md frontmatter의 색 토큰이 **라이트/다크 양쪽** CSS 변수로 `globals.css` 한 곳에 정의된다: `surface-base`·`surface-raised`·`ink-primary/secondary/muted`·`border-hairline`·`brand-petrol`·`brand-petrol-strong`·`petrol-deepest`·`accent-amber`·`price-emphasis`·`trust-green-bg/ink`·`placeholder-bg`·`warn-amber-bg/ink`·`danger`. 색 변경이 이 파일 한 곳에서만 이뤄진다.

2. **(Pretendard 로드 — UX-DR2)** Pretendard가 web에 로드되고(한글+라틴), `body` 기본 폰트가 Pretendard로 바뀐다(기존 Geist/Arial 제거). 타이포 스케일이 토큰화된다: display(36/800/1.15)·section(20/700/1.3)·card-title(16/600/1.4)·price(26/800/1.2)·body(15/500/1.6)·meta(13/500/1.5)·caption(12/500/1.4).

3. **(amber CTA 잉크 고정)** amber CTA(검색·문의) 버튼의 글자·아이콘 색은 어두운 잉크 `#1A1E1D`로 **고정**되며, 다크 테마에서도 자동 스왑되지 않는다(스왑하면 대비 1.89:1 실패). 즉 `accent-amber` 위 잉크 토큰은 라이트/다크 공통 상수여야 하고, `#F0A339` 배경 위에서 대비 8.02:1을 유지한다.

4. **(load-bearing 대비 AA)** DESIGN.md 조정치대로 `ink-muted`(라이트 5.06:1·다크 5.58:1), `price-emphasis`(라이트 흰바탕 3.68:1, 큰 텍스트 기준), `trust-green-ink`(라이트 5.51:1), `danger`(라이트 5.44:1)의 조정된 hex가 그대로 반영되어 WCAG AA를 충족한다(색을 임의로 바꾸지 않는다).

5. **(CRAFT BAR 유틸 — UX-DR3)** 겹 그림자(카드 기본 2단 `0 1px 2px rgba(0,0,0,.04)` + `0 10px 30px -14px rgba(0,0,0,.18)`·호버 리프트·떠 있는 요소 강한 겹 그림자), 카드 radius 16(칩 9·배지 11), 4pt 여백 리듬(4·8·12·16·20·24)이 **재사용 토큰/유틸**(Tailwind `--shadow-*`·`--radius-*` 토큰 등)로 확립된다. 다크모드는 그림자 대신 `border-hairline`·표면 밝기차로 층을 표현한다.

6. **(로고 컴포넌트 — UX-DR9/D7)** 로고 lockup(방향 A "차 배지" — petrol 라운드-스퀘어 배지 + 굵은 "차"(Pretendard 800) + "차장님" 워드마크)이 **토큰 기반 재사용 컴포넌트**(`<Logo>`)로 구현된다. 색·radius·폰트를 하드코딩하지 않고 위 토큰을 쓴다. *실제 아트워크는 추후 제작 — 현재 lockup 임시.* (내비 배선은 Epic 11, 이 스토리는 컴포넌트만 만든다.)

7. **(검증)** dev 서버를 띄워 임의 화면에서 (a) Pretendard가 실제 적용됨(폰트 폴백 아님), (b) 라이트/다크 토글(OS 설정) 시 토큰이 양쪽 다 유효, (c) `<Logo>`가 라이트/다크에서 깨지지 않고 렌더됨을 **직접 관찰**하고 스크린샷으로 확인한다(B4). 빌드(`next build`)가 통과한다.

## Dev Notes

### ⚠️ 최우선 — "This is NOT the Next.js you know"
`web/AGENTS.md`가 명시: 이 프로젝트의 Next.js(16.2.9)는 학습 데이터와 API·관례가 다를 수 있다. **코드 작성 전 `node_modules/next/dist/docs/`의 관련 가이드(폰트·metadata·layout)를 먼저 읽어라.** 특히 폰트 로딩 방식(next/font vs `<link>`)은 버전 관례를 확인하고 진행.

### Tailwind v4 = CSS-first (@theme). 옛 문법 금지
- **`tailwind.config.js`를 만들지 마라.** Tailwind v4는 CSS `@theme` 기반이다(project-context 버전 함정). 토큰은 전부 `globals.css` 안에서 정의한다.
- 현재 `globals.css`(전체 27줄)는 Next.js 기본 보일러플레이트(Geist·background/foreground 2토큰뿐)다. 이걸 **디자인 시스템 토큰 파일로 확장**하는 게 이 스토리의 핵심 산출물. 기존 `@theme inline` + `:root` + `@media dark` **패턴을 그대로 확장**하면 된다(이미 라이트/다크 스왑 골격이 있음 — 토큰만 15군으로 늘림).
- 라이트/다크 스왑은 **`prefers-color-scheme` 미디어쿼리**로 충분하다. 수동 테마 토글 UI는 이 스토리 범위 아님(A2 — 요청 안 된 기능 금지).

### Pretendard 로딩 (결정 필요 — 권장안 제시)
DESIGN.md·decision-log(D6)는 **"web=CDN(jsdelivr) 또는 self-host"** 둘 다 승인. Pretendard는 Google Fonts에 없어 `next/font/google`로는 못 불러온다. 두 길:
- **(권장) CDN jsdelivr dynamic-subset** — `globals.css` 상단 또는 `layout.tsx` `<head>`에 Pretendard dynamic-subset CSS를 링크(`https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/variable/pretendardvariable-dynamic-subset.css`). 데모 규모엔 가장 단순(A2), 폰트 파일 레포에 안 넣음. 단점=런타임 외부 CDN 의존.
- **(대안) self-host via `next/font/local`** — Pretendard variable `.woff2`를 `web/`에 넣고 `next/font/local`로 로드. FOUT/레이아웃 시프트 없고 외부 의존 0. 단점=레포에 폰트 바이너리 추가.
→ **권장: CDN dynamic-subset로 시작.** 오프라인/CDN 신뢰성이 문제되면 self-host로 승격. 어느 쪽이든 폴백 스택(`system-ui, sans-serif`)을 폰트 실패 대비로 남긴다. **로딩 방식은 반드시 `node_modules/next/dist/docs/` 폰트 가이드로 Next 16 관례 확인 후 확정.**
