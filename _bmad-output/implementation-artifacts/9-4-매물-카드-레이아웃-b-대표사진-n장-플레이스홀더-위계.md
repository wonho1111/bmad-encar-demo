As a 구매자,
I want 목록 카드에서 대표사진·가격·핵심 정보를 한눈에 보길,
so that 빠르게 훑고 관심 매물을 고른다.

1. **레이아웃 B 순서** — 위에서 아래로 정확히:
   **① 사진(5:3) → ② 신뢰속성 행 슬롯 → ③ 차량명 → ④ meta(주행·연료·지역) → ⑤ 가격(최상위) → ⑥ 옵션 칩 슬롯**
   - ③ `text-card-title`(16/600) `text-ink-primary`, 형식 `[제조사] 모델 · 연식`
   - ④ `text-meta`(13/500) `text-ink-muted`, **한 줄 가로 유지**
   - ⑤ `text-price`(26/800) `text-price-emphasis` — **카드에서 시각적으로 가장 큰 요소**(차량명보다 작으면 실패)
   - ②·⑥ 슬롯은 **값이 없으면 아무것도 렌더하지 않는다**(빈 높이·빈 테두리 금지). 값은 Epic 10이 채우며 지금은 항상 비어 있다.
   - ⚠️ 현재 `ListingCard.tsx`는 zinc 유틸(`border-zinc-200`·`text-zinc-500`)이다. 8.1 토큰(`bg-surface-raised`·`text-ink-*`·`rounded-card`·`shadow-card`)으로 **교체**한다 — 새 토큰을 만들지 않는다(전부 `globals.css`에 있다).
   (FR32, UX-DR4, DESIGN.md "매물 카드 — 레이아웃 B")

2. **사진 있으면 대표 1장 + "N장" 배지** — `image_url`이 있으면 5:3으로 대표 1장을 렌더하고, 사진 위 **우하단**에 다크 pill "N장"을 붙인다(`image_count` ≥ 1일 때만).
   - 크롭 = 클라 렌더 크롭(`object-fit: cover`, 중앙). 저장본을 다시 만들지 않는다(I14).
   - **`backdrop-filter` 없이도 읽혀야 한다** — 불투명 다크 pill이 기본, blur는 지원될 때만 얹는다(구형 WebView no-op, UX-DR6).
   - **폴백 상태의 흰 글자 대비를 계산해 기록**한다(≥4.5:1). 리뷰가 "폴백 대비 미검증"으로 지적한 항목이다(`review-accessibility.md` [medium]).
   (FR29, UX-DR6, I14)

3. **사진 없거나 로드 실패면 "사진 준비중" 플레이스홀더** — `bg-placeholder-bg` + 카메라 아웃라인 글리프 + "사진 준비중". **빈 영역·깨진 이미지 아이콘 금지**(의도적으로 보이게).
   - 트리거 3가지를 **모두** 처리한다: `image_url`이 **`null`** · **빈 문자열(`""`)** · **`onError`(로드 실패)**.
   - ⚠️ **`onError`가 9.0 이후 더 중요해졌다.** 서명 시절엔 발급 실패가 `null`로 와서 자동으로 플레이스홀더가 됐지만, `getPublicUrl`은 **파일이 없어도 문자열을 돌려준다**(존재를 확인하지 않는다). 이제 깨진 이미지를 막는 유일한 장치가 `onError`다 — 빠뜨리면 사진 없는 매물에서 깨진 아이콘이 그대로 노출된다.
   - ⚠️ **문구 대비를 계산해 확인한다.** `--ink-muted`(#676D69) on `--placeholder-bg`(#F1EFE9) = **약 4.4:1로 AA 4.5:1 미달 가능성이 높다**(개략 계산, 실측 필요). 미달이면 이 문구만 `text-ink-secondary`(#565F5D, 약 5.9:1)로 올린다. **다크 모드도 같이 잰다.** 토큰 값 자체는 바꾸지 않는다(다른 화면 회귀).
   (FR27, UX-DR7, `conventions.md` §4, `review-accessibility.md`)

4. **찜(♡) 버튼은 사진 밖, 위치·시각만** — 정보 영역 **우상단** 원형 버튼(그림자). **동작·토글·상태는 Epic 10.5의 몫이며 여기서 만들지 않는다.**
   - 동작이 없으므로 `disabled` + `aria-hidden="true"` + `tabIndex={-1}` — 스크린리더·키보드가 죽은 컨트롤을 잡지 않게.
   - 터치 타깃 **≥44×44px**(`review-accessibility.md` [high]).
   - ⚠️ **HTML 중첩 함정**: 현재 카드 전체가 `<Link>` 하나다. `<a>` 안의 `<button>`은 유효하지 않다. 루트를 `<article className="relative">`로 바꾸고 `<Link>`는 **내용 영역만** 덮은 뒤 찜 버튼을 그 밖에 `absolute`로 둔다.
   (FR32, DESIGN.md D3)

5. **대표 1장만 로드** — 대표 외의 이미지를 요청하지 않는다. `loading="lazy"` + `decoding="async"` + `aspect-ratio`로 **레이아웃 시프트 없이** 지연 로드. (NFR7)

6. **D5 반응형 무결성** — `/search`·홈 미리보기의 카드 목록을 `ResponsiveGrid`로 바꾼다(**≥1100px 4열 · 640~1099px 2열 · <640px 1열**). 현재는 `<ul className="flex flex-col gap-2">` 1열 고정이다.
   - **카드 내부 가로 배치(meta 한 줄·신뢰속성 행·옵션 칩)는 어떤 폭에서도 세로로 접히지 않는다.** 공간 부족은 `truncate`로만.
   - 검증은 **실제 브라우저 3개 뷰포트**(1280·800·390px)에서 눈으로 + 스크린샷(AC10).
   (D5, project-context 규칙13, UX-DR21)

### 2. 데이터 흐름 (입력에서 화면까지)

```
[서버 컴포넌트: /search · /]
  ① buyerListingsQuery → listings 행 N개 (status='on_sale'만, FR11)
  ② listing_images.select().in('listing_id', ids).order('sort_order').order('id')
  ③ coverImages(rows) → Map{ listing_id → { coverPath, count } }
  ④ getPublicUrl(bucket, coverPath) → 문자열 조립 (왕복 0회, 만료 없음)
  ⑤ image_url·image_count 병합 → ListingCard로 전달
[브라우저]
  ⑥ <img loading="lazy">가 공개 URL을 직접 로드
  ⑦ null / '' / onError → "사진 준비중" 플레이스홀더
```

### 3. `next/image`를 쓸 것인가 (짧은 결정 — 판단하고 기록만)

#### 1. #57 실측 — `canvas.toBlob`의 미지원 MIME 동작 (2026-07-19, 로컬 HeadlessChrome 151)

스토리가 "추측으로 고치지 말고 먼저 재라"고 지정한 항목. 실제 크로미움에서 캔버스를 만들어 직접 태웠다.

| 요청 MIME | `blob.type` | 앞 4바이트 | 판정 |
|---|---|---|---|
| `image/webp` | `image/webp` | `52 49 46 46` (RIFF) | 지원 |
| `image/jpeg` | `image/jpeg` | — | 지원 |
| `image/tiff` · `image/avif` · `not/a-real-mime` | **`image/png`** | **`89 50 4e 47`** | **null이 아니라 PNG 대체** |

→ **코드리뷰 2차의 의심이 맞았다.** 기존 폴백은 `if (blob) return blob;`이라 **미지원 브라우저에서도 첫 분기가 항상 참**이 되어 JPEG 폴백이 한 번도 실행될 수 없었다. 결과적으로 PNG가 저장되고 `extensionFor`가 `.jpg` 이름을 붙였다(확장자↔내용 불일치).

용량 실측(1600×1067 사진형 그라디언트+노이즈): **WebP 553KB · JPEG 503KB · PNG 4,827KB(약 9배)** — 버킷 5MB 상한에 닿고 AC4 저장본 규격 보장이 깨진다.

**처방:** `encodeWithFallback(toBlob)`으로 분리해 주입 가능하게 하고, 판정을 `blob !== null` → **`blob.type === 요청한 타입`**으로 교체(null을 주는 브라우저와 PNG를 주는 브라우저를 둘 다 잡는다). 둘 다 실패하면 규격 밖 파일을 조용히 저장하지 않고 throw.

**red 확인:** 옛 코드(`if (webp) return webp;`)로 되돌리자 신규 5건 중 2건이 실패(`expected 'image/png' to be 'image/jpeg'`) → 되돌려 11건 green.
