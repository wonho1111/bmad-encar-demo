# 차장님 — Experience Spine (작동 방식)

시각 정체성·토큰은 **DESIGN.md**가 원본. 이 문서는 구조·거동·플로우를 정의하고 토큰은 `{token}` 이름으로 참조한다.

## Foundation

- **멀티 서피스:** web(주력, 반응형) · Flutter 앱(구매자·판매자 공용, 웹과 동일 디자인 언어) · 관리자(web 전용, 이번 증분 최소 변경).
- **역할 통합(OI-1):** 가입 시 역할 선택 없음. 접근제어 = 소유권 기준. 판매자 = 구매자 동선의 상위집합 + 매물 등록/관리. 별도 판매자 홈 없음.
- **AI = 전역 1급 진입점(OI-2).** 문의 채팅은 **매물 상세에서만** 개시(전역 문의 메뉴 없음).
- Flutter는 웹 결정을 전부 상속하고 델타만 다름(→ Responsive & Platform · D14).

## Information Architecture

**서피스 목록**

| 서피스 | 목적 | 접근 |
|---|---|---|
| 랜딩(홈) | 딥 petrol AI 히어로 + 차종 칩 + 인기(view_count)·최신 발췌 | 공개(비로그인 열람, FR58) |
| 내 차 사기 — 목록 | 매물 그리드 + 필터 | 공개 |
| 매물 상세 | 갤러리 + 신뢰/차량/옵션/판매자 + 문의 CTA | 공개(문의는 로그인 게이트) |
| 내 차 등록·수정 | 사진·옵션·신뢰정보 입력 | 로그인 |
| AI로 찾기 | 자연어 대화 검색(4갈래) | 공개 |
| 채팅 — 목록/방 | 문의 대화 실시간 | 로그인 |
| 찜 | 관심 매물 모음 | 로그인 |
| 프로필 · 내 매물 관리 | 내 정보·내 매물·로그아웃 | 로그인 |
| 회원가입/로그인 | 역할 선택 없음 | 공개 |
| 관리자 | web 전용 관리 | admin |

**웹 상단 내비 (D8):** 로고 · **내 차 사기** · **AI로 찾기** · **내 차 팔기** | (비로그인) 로그인·내 차 등록 / (로그인) 찜♡ · 채팅🔔(안읽음 배지 FR57) · **프로필▾**(내 매물 관리·내 정보·로그아웃). 로그인 후 홈 = 공개 랜딩과 동일(FR58 단일 공개 홈). 모바일 웹 = 링크는 햄버거, 찜·채팅 아이콘은 상단 상시.

**앱 내비 (D12, FAB 없음):** 하단 4탭 **홈(AI검색) · 찜 · 채팅 · 내차팔기**. AI = 홈 최상단 큰 검색부(별도 AI 탭 없음, 웹 히어로의 앱 번역판). 프로필 = 우상단 아바타. 채팅 탭 = 목록 열람만, 개시는 상세에서만. **내 차 사기(목록/필터)는 별도 탭 없이 홈 하단 스크롤(인기·최신 발췌)·필터에서 도달**(웹은 상단 "내 차 사기" 링크).

**불변 규칙(nav-ia):** AI = 1급 진입 · 문의는 상세에서만 개시 · 삭제/구매완료 후 안전한 목록 복귀(R6) · 그 외 출처 복귀(R5).

## Component Patterns (거동)

- **매물 카드** (`mockups/card-final-1.html`): 탭 → 상세. 찜(♡) = **낙관적 토글**(즉시 채워짐, 비로그인 시 로그인 게이트). 신뢰 뱃지는 판매자 신고값(비검증)이라 클릭 액션 없음(정보 표시 전용). "N장" 배지 = 사진 장수만.
- **찜 토글**: 낙관적 반영 → 서버 확정. 실패 시 롤백 + 조용한 토스트. 비로그인 = 로그인 게이트 후 복귀.
- **AI 되묻기 칩**: tappable, 탭 시 **petrol 채움 "선택됨"** 상태 → 다음 AI 턴(FR46 "칩=타이핑과 동등 경로" 보장). 자유 타이핑 항상 병존. 상시 제안칩 vs 맥락 칩 스타일 구분. **되묻기 상한 = 최대 2~3턴** — 그 이상은 더 되묻지 않고 **현재까지 조건으로 결과를 강제 제시**하고 "조건을 더 좁히려면 검색 후 필터를 쓰세요"로 안내한다(무한 되묻기 금지, 막다른 루프 방지).
- **옵션 하이브리드 피커(등록, D11)** (`mockups/forms-2.html`): 기본 = 인기옵션 8칩(스마트키·내비·후방카메라·열선시트·통풍시트·선루프·크루즈·어라운드뷰) → 대부분 여기서 끝. "전체 옵션 더보기" = 카테고리 아코디언 체크리스트 + **옵션 검색창**(희소옵션 구조적 선택). 선택 = 칩 요약 + 개수. DB는 옵션 배열 그대로.
- **사진 갤러리(상세)**: 대표 5:3 + 썸네일 스트립 + "1/8" 카운터 + 좌우 화살표(웹) / 스와이프(앱).
- **사진 업로더(등록/수정)**: 업로드 실패·용량초과 = 인라인 오류 + 재시도 버튼(폼 제출 차단 아님). **대표사진 삭제 시 다음 순서 사진이 자동 대표 승격.**
- **폼 이탈 경고(내 차 등록/수정)**: 변경사항(특히 업로드한 사진)이 있는 채 뒤로가기·이탈 시 확인 다이얼로그 — "저장하지 않고 나가시겠어요? 작성한 내용이 사라져요." + [계속 작성]/[나가기]. 사진 여러 장 올린 뒤 유실 방지가 핵심. 변경 없는 폼은 경고 없이 이탈.
- **내 정보 수정(프로필▾ → 내 정보)**: 닉네임 변경(필수, 공백·길이 검증) · [저장]/[취소] · 성공 시 토스트("저장했어요"). 역할 통합(OI-1)이라 역할 필드 없음. 이메일·비밀번호 변경은 이번 증분 범위 밖.
- **채팅**: 실시간 송수신 + 멱등키(중복 방지) + 재연결 배너(비차단). 방 = 상세 "문의하기"에서 생성/재사용. **재연결 시 내 pending 큐 전송뿐 아니라 "마지막 수신 이후" 1회 재조회로 상대방이 보낸 놓친 메시지를 병합**(멱등 dedupe·순서 보정) — 끊긴 동안 상대 발화가 유실되지 않는다. **메시지 입력 2000자 상한**(기구현, 커밋 b720370) — 실시간 글자 카운터 + 초과 입력 방지. **자기 소유 매물 상세에는 "문의하기" 대신 "내 매물 관리"** 노출(자기 자신에게 문의 불가, DB `CHECK(buyer_id<>seller_id)`와 정합). 채팅방 헤더의 매물 요약이 sold/삭제되면 "판매완료"/"삭제됨" 배지 표시(대화 자체는 유지).
- **AI 라우팅 4갈래 거동(FR43~47)**: ① **구조형**(FR43) = 조건 충족 → 요약 답 + 결과 카드 그리드 인라인. ② **되묻기**(FR46) = 조건이 넓으면 "조건을 조금만 좁혀볼게요 —" + 맥락 칩(칩=타이핑과 동등 경로). ③ **조합형**(FR45) = 자연어 + 이전 턴 맥락을 합쳐 재검색(멀티턴 누적, 예산·인원 등 델타 반영). ④ **거절**(FR47) = 중고차 무관 질의 → 따뜻한 고정 템플릿(Voice 표) + 원 조건 재제안 칩(막다른 길 아님).

## State Patterns (서피스별)

| 서피스 | empty | loading | error / 기타 |
|---|---|---|---|
| 찜 | "아직 찜한 매물이 없어요…" | 스켈레톤 카드 | 401 → 로그인 게이트 · **sold/삭제 매물 = 회색 처리 + "판매완료"/"삭제된 매물" 비활성 배지, 상세 진입 차단**(기존 관계는 목록에 남되 상태 표기) |
| 내 매물 관리 | "아직 등록한 매물이 없어요. '내 차 등록'으로 첫 매물을 올려보세요." | 스켈레톤 행 | 401 → 로그인 게이트 · 각 행에 sold/판매완료 상태 표기 |

| 채팅 목록 | "아직 문의한 채팅이 없어요…" | 스켈레톤 행 | **오프라인/재연결 배너**(비차단, 재연결 시 초록) |
| AI/검색 0건(FR17) | "조건에 맞는 매물이 아직 없어요…" + 완화 칩 | 타이핑 인디케이터 | AI 거절 = 따뜻한 고정 템플릿 + 재제안 칩 |
| 카드/상세 사진 없음 | `{placeholder-bg}` + 카메라 글리프 + "사진 준비중" | 이미지 블러업 | — |
| 상세 | — | 갤러리 스켈레톤 | **sold 매물 = 구매자에게 미노출**(FR11, 상세 경로 포함, 신규 조회 경로 기준) · 문의 401 게이트 · **본인 소유 매물이면 "문의하기" 대신 "내 매물 관리"** 노출 |
| 매물 상세(URL 직접 접근) | — | 갤러리 스켈레톤 | **미존재·삭제·판매완료 ID로 상세 URL 직접 접근 = 404 화면**("매물을 찾을 수 없어요. 삭제됐거나 판매완료된 매물일 수 있어요." + "매물 목록으로" CTA, 중립 톤 — {danger} 아님) · **구매자는 sold도 직접 URL 접근 시 동일 404-류**(신규 조회 경로 비노출 + URL 직접접근 방어 정합) |
| 폼(등록/수정) | — | 저장 중 버튼 스피너 | 403 "권한이 없어요." · 검증 인라인 · **사진 업로드 실패/용량초과 = 인라인 오류 + 재시도, 대표사진 삭제 시 다음 사진 자동 대표 승격** · **변경사항(특히 업로드 사진) 있는 채 이탈 = 이탈 경고 다이얼로그**("저장하지 않고 나가시겠어요?…") |
| 전역 | — | — | 500 "잠시 문제가 생겼어요…" · **권한 위반(남의 매물 수정/삭제)=소유권 RLS 차단 + UI 미노출** · **로그인 만료 중 액션=401 게이트 후 원위치 복귀** |

- **포커스 상태**: 입력·버튼 포커스 = `{brand-petrol}` 링. 검색바 포커스 = petrol 강조.
- **오프라인/재연결(채팅, FR42)**: 비차단 배너 — 끊겨도 메시지 계속 작성 가능, 재연결 시 큐 전송 + 배너 초록 전환.

## Interaction Primitives

- **찜 토글** — 낙관적, 로그인 게이트, 실패 롤백.
- **되묻기 칩 select** — 탭 → petrol 채움 선택 상태 → AI 턴 트리거.
- **사진 스와이프/드래그 순서** — 갤러리 스와이프(앱), 업로더 드래그로 순서변경·"대표" 지정.
- **sticky 상세 문의 CTA** — 우 요약 컬럼 `position:sticky`(top=navbar), 모바일은 하단 고정 문의 바(가격+CTA 상시, top-shadow).
- **optimistic send(채팅)** — 전송 즉시 버블 표시(pending) → 서버 확정, 멱등키로 중복 차단. 입력 **2000자 상한**(기구현, b720370): 실시간 카운터 + 초과 입력 방지.
- **AI 검색 입력 상한** — AI 검색 입력창은 **500자 상한** + 실시간 카운터·초과 입력 방지(채팅 2000자와 별개). 매우 긴 질의로 인한 처리·비용 폭주 방지.
- **폼 이탈 가드(등록/수정)** — dirty 상태(변경/사진 업로드) 감지 → 뒤로가기·라우트 이탈·탭 닫기 시 확인 다이얼로그, 변경 없으면 무경고. 웹=`beforeunload`/라우트 가드, 앱=OS back 가로채기.

## Accessibility Floor

- **대비(load-bearing 조합):** `{brand-petrol}` on `{surface-raised}` 흰 바탕, `{price-emphasis}` amber 가격 on 흰 바탕, `{trust-green-ink}` on `{trust-green-bg}`, amber CTA 어두운 글자 — DESIGN.md 조정치로 모두 WCAG AA(본문 4.5:1, 큰 텍스트/UI 3:1) 충족(대비 타깃은 DESIGN.md Colors 참조). 히어로 밴드 위 흰 텍스트도 AA. **amber CTA 글자는 어두운 잉크 고정(흰색 금지).**
- **비색 신호 중복(non-color redundancy):** 색 단독 신호 금지 — 신뢰 = 초록 + ✓ + 텍스트 · 가격 = amber + 크기/굵기 · 안읽음 = 점 + 숫자 텍스트 · 선택된 칩 = petrol 채움 + ✓ · 재연결 성공 = 초록 + "다시 연결됐어요" 텍스트/✓.
- **터치 타깃 ≥44px**(모바일 주요 버튼 ≥52px: 등록·가입). **아이콘 버튼(찜♡·알림🔔·검색·햄버거)·칩 탭 영역은 시각 크기와 무관하게 히트영역 ≥44×44px.**
- **포커스 트랩:** 모달·바텀시트(모바일 필터)·드롭다운(`프로필▾`)·로그인 게이트 = 열림 시 포커스 이동+트랩, Esc로 닫힘, 닫힐 때 트리거 요소로 포커스 복귀.
- **한국어 스크린리더 라벨**: 아이콘 버튼(찜♡·채팅🔔·검색·햄버거)에 명시 라벨. 안읽음 배지 = 텍스트 대체. 찜 토글·선택 칩은 `aria-pressed`/`aria-selected` + 상태 라벨 변경("찜하기"↔"찜 취소").
- **AI 대화 스크린리더:** 메시지 발화자 role(차장님/나) 구분해 announce, 타이핑·스트리밍 상태는 `aria-live="polite"`로 과공지 없이 알림.
- **reduced-motion:** 히어로 글로우·카드 호버 리프트·배너 전환은 `prefers-reduced-motion: reduce` 시 축소/제거.
- **히어로 헤드라인 amber 음절**은 글로우 위에서도 대비 유지 — 텍스트 영역을 글로우 회피 구역(safe zone)으로 두거나 얇은 다크 스크림/섀도우로 명도차 확보.
- **포커스링 = `{brand-petrol}`**, 웹은 전 인터랙션 키보드 도달 가능.

## Key Flows

**① 구매자 "지수" — AI로 차 찾고 문의**
1. 홈 딥 petrol 히어로에 "3천만원대 무사고 흰색 SUV" 입력(또는 제안칩 탭).
2. AI가 요약 답 + 결과 카드 그리드(구조형, FR43①) 인라인 제시.
3. 조건이 넓으면 **되묻기 칩**(인원·예산·연료) — 지수가 "7인승" 칩 탭 → petrol 선택 상태 → 좁혀진 결과.
   - **[조합형, FR45]** 이어서 "예산은 4천까지"라고 자유 입력 → 이전 턴 조건에 델타를 합쳐 재검색(멀티턴 누적).
   - **[거절, FR47]** 중고차와 무관한 질문엔 따뜻한 고정 템플릿으로 정중히 선을 긋고 원 조건 재제안 칩 제시(막다른 길 아님).
4. 카드 탭 → 상세. 신뢰정보(초록) 확인 → 가격 확인.
5. **[climax]** sticky "문의하기" CTA 탭 → (로그인돼 있으므로) 채팅방 생성 → 첫 메시지 전송.
- **[failure]** 비로그인이면 문의 탭 시 로그인 게이트 → 로그인 후 **원 상세로 복귀**해 문의 이어감.

**② 판매자 "민호" — 사진·옵션 넣어 매물 등록**
1. 내 차 팔기 → 내 차 등록 폼.
2. 사진 업로더에 드래그(3/10) + "대표" 지정. ("사진은 선택…" 안내.)
3. **인기옵션 8칩**에서 선택 → 희소옵션은 "전체 옵션 더보기" 검색으로 추가.
4. 신뢰정보 토글(무사고·1인소유·비흡연) — 정직성 고지 배너 확인.
5. **[climax]** 모바일 "등록하기"(≥52px) 탭 → 등록 완료 → **내 매물 관리로 안전 복귀**(R6). 카드에 희소옵션 뱃지·"판매자 제공 정보" 노출.
- **[failure]** 필수 누락 시 인라인 검증 + 해당 필드 포커스, 제출 차단.

**③ 비로그인 방문자 — 홈 열람 → 찜 시도 → 로그인 게이트**
1. 홈·목록·상세 자유 열람(FR58).
2. 마음에 든 카드 ♡ 탭.
3. **[climax]** 로그인 게이트 등장("로그인이 필요해요.") → 가입/로그인.
4. 로그인 후 **원 위치로 복귀**, 찜이 낙관적으로 반영.
- **[failure]** 게이트 이탈 시 상태 변화 없음(찜 안 됨), 원 화면 유지.

**④ 채팅 실시간 주고받다 오프라인 → 재연결**
1. 상세에서 문의 개시 → 방 진입, 실시간 송수신.
2. 네트워크 끊김 → **비차단 배너** "연결이 끊겼어요. 다시 연결 중…".
3. 배너 뜬 채로 계속 타이핑·전송(메시지 pending 큐).
4. **[climax]** 재연결 → 큐 전송(멱등키로 중복 없음) → 배너 **초록 전환** 후 사라짐.
- **[failure]** 장기 실패 시 pending 유지 + 재시도, 데이터 유실 없음.

## Responsive & Platform

- **브레이크포인트:** ≥1100px = 4열 · 640~1099px = 2열 · <640px = 1열/모바일 캐러셀(1.2장). 카드 내부 가로 배치는 세로화 금지(D5), 폭은 **열 수로만** 흡수.
- **웹 모바일:** 내비 링크 → 햄버거, 필터 → 바텀시트, 찜·채팅 아이콘 상단 상시. 웹 반응형 + 웹모바일 vs 앱 일관성 예시: `mockups/consistency-1.html`.
- **네이티브 앱 구조 델타(공유 비주얼 언어):** 하단 4탭(D12) · 사진 갤러리 스와이프 · 뒤로가기 = OS back(R5) · 사진 = OS 갤러리/카메라 피커 · 폰트 = Pretendard 번들.
- **Flutter 델타(D14):** 색·토큰·타이포·카드 레이아웃 B·초록 신뢰뱃지·amber 가격·플레이스홀더·상세 구성·AI 히어로 톤·마이크로카피(D13) 전부 상속. 구현 = `GoRouter StatefulShellRoute`(탭 상태 보존) + Riverpod 검색 상태를 웹과 동일 형태로 미러링, Material 3 `NavigationBar` 재사용. 관리자는 앱 이월 없음. 참고: `mockups/app-home-2.html`.

## Inspiration & Anti-patterns

- **참조:** 엔카·K Car(P2P)·KB차차차("내차사기" 표준어)·첫차·당근·번개장터(P2P FAB=판매등록 관례).
- **Anti-patterns:**
  - AI 검색을 단일 FAB에 두기 = Material Design 안티패턴(구글 포토). 검색은 핵심기능 → 상단 검색바 + 탭 병행.
  - **검증 UI 위장** = 판매자 신고값을 "차장님이 검증함"처럼 보이게 하는 것 금지. 반드시 "판매자 제공 정보" 면책.
  - **개발용어 라벨**("매물 탐색"·"탐색") = 소비자 자연어로 치환.
