# Edge-Case & State-Coverage Review — 차장님 UX Spine (EXPERIENCE.md + DESIGN.md)

리뷰 대상: `EXPERIENCE.md` · `DESIGN.md` · `.decision-log.md` (2026-07-12, status:final)
방법: 서피스별 필수 상태(empty/cold-load/error/offline/permission/focus/disabled/partial) 및 명시된 엣지케이스 체크리스트를 전수 대조. **스파인이 다루지 못한 GAP만** 기록.

## 찜 (Favorites)

- **[critical]** 찜한 매물이 sold되거나 삭제됐을 때 찜 목록 렌더링 규칙 없음 (§찜). FR11("sold 매물=구매자 미노출")과 찜 목록의 교차 케이스가 미정의 — 목록에서 자동 제거되는지, "판매완료"/"삭제됨" 배지로 남는지 불명. *Fix:* 찜 목록 카드에 sold/삭제 상태를 명시(예: 흐림 처리 + "판매완료" 또는 "삭제된 매물이에요" 카피 + 탭 시 상세 진입 차단)하고 EXPERIENCE State Patterns 표에 행 추가.
- **[medium]** 찜 토글 연속 빠른 탭(race condition) 시 낙관적 반영이 중복 증가/깜빡임할 수 있는 디바운스·잠금 규칙 없음 (§찜 컴포넌트 패턴). *Fix:* 토글 진행 중 재탭 무시(락) 규칙을 Interaction Primitives에 한 줄 추가.

## 매물 상세

- **[critical]** 채팅방 상대(문의) 매물이 sold/삭제된 경우 채팅방 UI 처리 미정의 (§채팅 — §매물 상세 교차). *Fix:* 채팅방 헤더에 "판매완료"/"삭제된 매물" 배지 + 해당 매물 링크 비활성화 규칙을 State Patterns에 추가.
- **[critical]** 본인 매물에 "문의하기"를 시도하는 경우(buyer=seller) 차단 규칙이 스파인에 전혀 없음 (§매물 상세 climax 플로우 ①). *Fix:* sticky/모바일 문의 CTA를 소유자 본인 열람 시 숨김 또는 disabled 처리하는 규칙을 D9·Interaction Primitives에 명시.
- **[high]** 삭제/미존재 매물 URL 직접 접근(딥링크) 시 404 상태·카피 없음. State Patterns 표 §5 에러 카피는 401/403/500만 정의, 404 누락. *Fix:* "매물을 찾을 수 없어요" 류 404 카피 추가 + sold 매물 구매자 접근도 같은 처리로 통일할지 명시.
- **[medium]** 로그인 세션이 문의 진행 "도중"(문의 CTA 클릭 후 채팅방 생성 API 호출 시점 등) 만료되는 케이스는 다루지 않음 — 현재는 CTA 클릭 이전 게이트만 정의(Flow① failure). *Fix:* API 호출 중 401 수신 시 동일 게이트로 폴백 + 원 상세 복귀 규칙 명시.
- **[medium]** 옵션 0개로 등록된 매물의 상세 "③ 옵션 전량" 섹션 빈 상태 카피 없음 (§D9). *Fix:* "등록된 옵션이 없어요" 류 빈 상태 문구 추가.
- **[low]** "④ 판매자 정보"의 "다른 매물 N건" 링크에서 N=0인 경우(이 매물이 유일한 매물) 처리 미정의. *Fix:* N=0이면 링크 비활성 또는 숨김.
- **[low]** 사진 갤러리 좌우 화살표가 첫 장/마지막 장 경계에서 disabled(또는 순환) 되는지 정의 없음.

## 내 차 등록·수정 폼

- **[critical]** 사진 업로드 중 실패(네트워크 오류)·용량초과·미지원 포맷·10장 초과 시도의 에러 상태가 전혀 정의되지 않음 (§D11 사진 업로더) — 명시적으로 요청된 체크 항목. *Fix:* 업로더에 실패/용량초과/포맷오류 인라인 에러 + 재시도 버튼 상태를 State Patterns·Component Patterns에 추가.
- **[high]** 대표사진 삭제 시 다음 사진이 자동으로 대표로 승격되는 규칙이 없음 (§사진 업로더, 요청된 체크 항목). *Fix:* "대표 삭제 시 순서상 다음 사진이 자동 대표 지정" 규칙을 Component Patterns에 명시.
- **[high]** 폼 작성 중 이탈(뒤로가기/다른 링크 클릭) 시 임시저장 또는 이탈 경고가 전혀 없음 — 입력 손실 위험. *Fix:* 최소한 "저장하지 않고 나가시겠어요?" 확인 다이얼로그 규칙 추가(임시저장 자체는 스코프 아웃이어도 손실 경고는 필요).
- **[high]** 남의 매물 수정 페이지에 소유권 없이 직접 URL 접근했을 때 403이 "제출 시점"에만 뜨는지, 폼 진입 자체를 막는지 불명확 (§소유권 검증). *Fix:* 폼 렌더링 전 소유권 체크로 403 즉시 표시하도록 명시.
- **[medium]** sold(판매완료)된 본인 매물의 수정/삭제 가능 여부 및 UI(disabled 처리 여부)가 정의되지 않음 (§프로필·내 매물 관리 교차). *Fix:* sold 매물은 수정 폼 진입 차단 또는 읽기 전용 배지 규칙 추가.
- **[medium]** 매물 삭제 액션의 확인 다이얼로그(되돌릴 수 없음 경고)가 정의되지 않음.
- **[medium]** "전체 옵션 더보기" 검색창에서 검색 결과 0건일 때 빈 상태 카피 없음 (§D11).
- **[low]** 제출 버튼이 필수값 미충족 시 proactive하게 disabled 되는지, 아니면 클릭 후 인라인 검증만인지 불명확(현재는 후자로 읽힘 — 명시화 필요).
- **[low]** 가격/주행거리 등 숫자 필드의 범위·형식 검증(0, 음수, 비정상 큰 값) 규칙 없음.

## AI로 찾기

- **[high]** 매우 긴 질의(용어 요청에 명시)에 대한 처리 없음 — 채팅은 2000자 상한이 이미 구현돼 있는데(레포 커밋 이력) AI 검색 입력에는 길이 제한/경고 UI가 스파인에 없음. *Fix:* AI 검색 입력에도 동일 상한 + 초과 시 카피/카운터 정의.
- **[high]** 되묻기(FR46) 무한 루프 방지 규칙이 없음 — 사용자가 애매한 답을 반복해도 종료 조건(N턴 후 강제 결과 제시 등)이 정의돼 있지 않음. *Fix:* "N턴 이상 되묻기 지속 시 최선 결과로 폴백" 규칙을 D10에 추가.
- **[medium]** 멀티턴 컨텍스트가 유지되는지, 이전 턴 조건이 화면에 어떻게 표시(활성 필터 칩 등)되는지 정의 없음 — 팀 메모리에 이미 "멀티턴 경로전환 시 소프트 예산 의도 누락" 알려진 한계가 있는데 스파인이 이를 다루지 않음. *Fix:* 이전 턴 조건을 대화 상단에 "적용중" 칩으로 노출하는 규칙 추가 검토.
- **[medium]** AI 응답 대기(타이핑 인디케이터) 중 사용자가 새 메시지를 또 보낼 수 있는지(동시 요청) 여부 불명. *Fix:* 대기 중 입력창 disabled 또는 큐잉 규칙 명시.
- **[medium]** AI 결과 카드 그리드에 노출된 매물이 이후 sold로 바뀌는 stale-result 케이스 처리 없음.
- **[low]** LLM 공급자 타임아웃/일시 장애가 일반 500과 동일 카피로 처리되는지, 별도 안내("잠시 후 다시 물어봐 주세요" 등)가 필요한지 불명.
- **[low]** 대화 히스토리가 새로고침/재방문 시 유지되는지 여부 정의 없음.

## 채팅 목록/방

- **[critical]** "상대가 나간 방" 상태가 전혀 정의되지 않음(요청된 체크 항목) — 채팅 나가기/떠나기 기능 자체가 스파인에 없어, 상대가 나가면 UI가 어떻게 반응하는지 공백. *Fix:* 최소 "상대방이 대화를 나갔어요" 배너 상태 추가하거나, 나가기 기능이 스코프 아웃이면 그 사실을 문서에 명시.
- **[high]** 재연결 시 "보내지 못한 내 메시지" 큐 전송만 다루고, 오프라인 동안 **상대방이 보낸 메시지**를 재연결 후 어떻게 갭 보정(재동기화)하는지 정의 없음 (§Flow④, 요청된 체크 항목: "재연결 갭 보정"). *Fix:* 재연결 시 마지막 수신 시각 이후 메시지를 재조회(fetch)하는 규칙을 D-flow④에 명시.
- **[high]** 채팅 메시지 2000자 상한(레포에 이미 구현된 기능, 커밋 `b720370`)의 UI 상태(카운터·경고·초과 시 전송 차단 카피)가 이 UX 스파인에 전혀 반영되지 않음. *Fix:* 입력창에 글자수 카운터/상한 도달 시 인라인 경고 상태 추가.
- **[medium]** 안읽음 배지(FR57)가 방 진입 시 언제/어떻게 clear 되는지(즉시 vs 메시지 읽음 확인 후) 규칙 없음.
- **[medium]** 메시지 전송 실패가 네트워크 문제가 아니라 권한/상태 문제(예: 방이 닫힘, 매물 삭제됨)일 때의 구분된 에러 카피 없음 — 현재는 오프라인 배너 하나만 정의됨.
- **[low]** 긴 채팅 이력의 페이지네이션(과거 메시지 더보기) 상태 없음.

## AI/목록 공통 — 검색·필터

- **[medium]** "AI/검색 0건(FR17)" 상태가 AI 대화형 검색과 일반 "내 차 사기" 목록의 필터 결과 0건에 동일하게 적용되는지 표에서 불명확(한 행에 뭉뚱그려짐). *Fix:* 두 서피스를 분리해 완화 칩 UI가 순수 필터 목록에도 적용되는지 명시.
- **[medium]** 목록/그리드의 무한스크롤 또는 페이지네이션 로딩(다음 페이지 로딩 중, 마지막 페이지 도달) 상태가 정의되지 않음.
- **[low]** 정렬(최신순 등)·필터 칩의 "선택됨" active 상태 스타일이 정의되지 않음(되묻기 칩만 선택 상태가 명시됨).
- **[low]** 필터 API 실패(500) 시 필터 UI 자체의 처리(이전 결과 유지 vs 에러 표시)가 불명확.

## 랜딩(홈)

- **[medium]** 랜딩의 "인기"/"최신" 섹션 자체의 cold-load 스켈레톤이 State Patterns 표에 없음(찜/채팅/AI/카드/상세/폼만 정의, 랜딩 섹션 로딩 상태 누락).
- **[low]** 데모 데이터가 4장 미만일 때 "인기"/"최신" 섹션이 부분 채움으로 어떻게 보이는지(빈 슬롯 처리) 정의 없음.
- **[low]** 차종 빠른 진입 칩의 선택/활성 상태 스타일 정의 없음.

## 반응형 (D5) 경계 케이스

- **[medium]** 카드의 신뢰속성 0개 + 옵션 0개 + 사진 없음이 동시에 겹치는 "완전 빈 카드" 상태에서 세로 공간이 과도하게 비어 "깨진 느낌"을 줄 수 있는데, 이 조합에 대한 최소 높이/자리 채움 규칙이 없음(신뢰뱃지·옵션칩 각각의 부분 표시는 허용됐지만 전부 없는 조합은 미검토).
- **[low]** 320px대 극협소 뷰포트에서 truncate/"외 N"만으로 흡수 불가능한 최소 지원 폭(design floor)이 명시돼 있지 않음.

## 프로필 · 내 매물 관리 / 회원가입

- **[high]** "내 정보" 수정 화면 자체가 IA 표에만 이름이 있을 뿐, 필드·검증·저장 성공/실패 상태가 전혀 정의되지 않음(서피스 전체가 사실상 미명세).
- **[high]** 신규 유저의 "내 매물 관리" 매물 0건 빈 상태 카피가 정의되지 않음(찜·채팅은 빈 상태 카피가 있는데 이 서피스만 누락).
- **[medium]** 회원가입 시 이메일 중복(422)·비밀번호 형식 오류 등 폼 검증 에러 카피가 정의되지 않음(가입 안내 카피 D13은 있으나 실패 케이스 없음).
- **[medium]** 로그인 실패(잘못된 자격증명) 카피/상태 정의 없음.
- **[low]** 로그아웃 액션에 확인 다이얼로그가 있는지 불명.
- **[low]** 관리자가 회원을 정지시킨 경우, 정지된 세션의 강제 로그아웃/재로그인 시도 시 안내 카피가 정의되지 않음(Epic 6 관리자 정지 기능과 연계된 프론트엔드 상태 공백).

## 관리자

- **[low]** 이번 증분은 "관리자 web 전용, 최소 변경"으로 명시돼 있어 상세 상태 재정의가 의도적으로 스코프 아웃일 가능성이 높음. 다만 non-admin이 관리자 라우트 접근 시 403 처리가 이 스파인에서 명시적으로 재확인되지 않음(기존 구현에 의존). *Fix:* 최소한 "기존 admin 가드 유지, 변경 없음"이라는 한 줄이라도 문서화하면 향후 회귀 방지에 도움.

## 커버리지 총평

- **잘 다뤄진 영역:** 찜/채팅 empty·오프라인 배너, AI 0건/거절 카피, 카드 신뢰뱃지 부분표시, 사진 placeholder, sold 매물 기본 원칙(FR11), 로그인 게이트 기본 플로우, 반응형 그리드 열 흡수 원칙(D5) — 스파인 레벨에서 견고함.
- **구조적 공백:** (1) **sold/삭제된 매물이 찜·채팅 등 "이미 관계를 맺은" 표면에 계속 남아있을 때**의 표시 규칙이 일관되게 비어있음(찜 목록, 채팅방 헤더 모두). (2) **자기 매물 문의 차단(buyer≠seller)** 이 요청 스펙에 명시됐음에도 스파인 어디에도 없음. (3) **사진 업로드 실패/용량초과/대표 승격** 등 폼의 실패 경로가 통째로 비어있음(성공 경로만 정의). (4) **채팅 2000자 상한**처럼 이미 백엔드/레포에 구현된 기능이 이 UX 스파인 문서에 반영되지 않은 문서-구현 드리프트가 있음.
