## Decisions

### D1 — 컬러 팔레트 방향 [🔄 재검토 중 2026-07-12]
_사용자 피드백 흐름: 순수 ②안 카드가 "너무 데모 앱 같다" → petrol 강도 3단(T1/T2/T3, `.working/palette-blend-1.html`) 비교 → **사용자가 이전 세션 목업 첨부**(딥 petrol 히어로 밴드 + 흰 검색바 + amber "검색"버튼 + petrol 제안칩, 하단 밝은 본문/카드그리드). 그 히어로 배경 톤을 선호하나 **"AI 검색 히어로 배경이 허전"**._

**수렴 방향:** 본문=밝게(T2) + **히어로만 딥 petrol 밴드**(T3 몰입감). 히어로가 유일한 몰입 순간, 나머지 밝음. → 히어로 배경 "허전함"을 채우는 처리 3안 탐색: 글로우/메시 · 차 실루엣 라인아트 · AI 모티프. 창작물 `.working/hero-treatment-1.html`(예정).

**⚠️ IA 충돌 메모(⑤에서 해결):** 사용자 목업 상단 내비에 **"문의" 전역 메뉴** 존재 → nav-ia-rules/FR38("문의는 상세에서만 개시, 전역 문의 메뉴 없음")과 충돌. 내비 단계에서 재정리.
**참고:** 사용자 목업 = petrol 히어로 헤드라인 "원하는 차를 말로 찾으세요"("말"·"로" amber), 제안칩 4개(가성비 좋은 첫차·4천만원대 전기 SUV·주행거리 짧은 무사고 세단·7인승 디젤 패밀리카), 카테고리칩(전체·경차·세단·SUV·전기·친환경·수입·화물·승합), "지금 인기/많이 보고 있는 매물" 섹션.

### ✅ D1(확정) — 컬러 시스템: "밝은 본문 + 딥 petrol 히어로" (T2 절충)
_확정 2026-07-12 · 창작물: `.working/palette-blend-1.html`(강도 비교) → T2 선택_

- **본문(목록·상세·폼 등):** 밝은 표면(surface-base `#FAFAF8`, raised `#FFFFFF`) 위에 petrol을 **구조·CTA·포커스**로 확실히(소심하지 않게) — 솔리드 내비/버튼/섹션헤더/검색 포커스에 petrol.
- **amber는 가격 + 핵심 CTA에만** 국한(money = amber). 화면당 amber 남발 금지.
- **다크모드:** 함께 지원(palette 다크 토큰 세트 존재).
- **"데모 티" 교정 원칙(중요):** 원인은 색이 아니라 **소심한 petrol + 그림자·타이포 위계 부족**이었음. 구현 시 **CRAFT BAR 필수** — 겹 그림자(0 1px 2px / 0 8px 24px -12px), 강한 타이포 위계(가격 24~28px/800, 제목 16px/600, meta 13px/muted), 넉넉한 여백(16~20px), 카드 radius 14~16px.

### ✅ D2(확정) — AI 검색 히어로: 딥 petrol 밴드 + "H2 차 실루엣 on H1 글로우"
_확정 2026-07-12 · 창작물: `.working/hero-treatment-1.html`_

- **딥 petrol 그라데이션 히어로 밴드**(≈`#14514E`→`#0F3D3E`) — 랜딩 최상단, AI 자연어 검색 히어로(FR33). 나머지 페이지는 밝음(다크 히어로 → 라이트 본문 전환).
- **배경 깊이감:** H1 글로우/메시(petrol·amber 빛무리 + 미세 노이즈)를 **베이스**로 은은히 + 그 위에 **H2 차 실루엣**(오른쪽 가장자리로 흐릿하게 흘러나가는 대형 차 라인아트). → "허전함" 해소 + 자동차 정체성.
- 히어로 텍스트: 헤드라인 일부 음절 amber 강조(예 "말"·"로"), 흰 검색바 + amber "검색" 버튼, petrol 반투명 제안칩.
- **목업은 색 배치 참고만** — 구속 스펙 아님(사용자 지시). 내비 등은 우리 IA 규칙 우선(→ "문의 전역 메뉴" 충돌 무효화).

#### ↳ D2-a(개정) — 히어로의 비로그인 행동 규약 (2026-07-14, Story 8.5 코드리뷰 + party-mode)

_시각 정의는 **불변** — 딥 petrol 밴드·H2 차 실루엣·흰 검색 pill·amber 검색 버튼·제안칩. 색·타이포·레이아웃 토큰 변경 없음. 바뀌는 것은 **비로그인일 때의 동작**뿐._

- **변경 사유:** 코드리뷰에서 `/ai/search`가 비로그인에 열려 있고 **호출당 Gemini 실비**가 발생함이 드러났다. AI 검색 = **행동**(외부 유료 API 호출)이므로 로그인 필수로 확정(아키텍처 I2' 원칙). 대안이던 "익명 N회 무료 + 카운터 표시"는 **폐기** — Cloud Run 다중 인스턴스에서 카운터를 공유할 수 없어 화면에 약속한 숫자를 지킬 수 없다(“2번 남음”이라 띄우고 실제로는 7번 되는 UI = **거짓말하는 UI**). **지킬 수 없는 숫자는 표시하지 않는다.**
- **입력창은 살아 있다.** 비활성화·자물쇠 아이콘·"로그인 필요" 사전 안내를 **붙이지 않는다** — 시도하기 전에 거절하지 않는다. 꺼진 히어로는 히어로가 아니라 잠긴 문의 사진이다.
- **게이트는 제출 시점에.** 비로그인이 질의를 입력해 제출하거나 **예시 칩을 클릭하면**(칩 = 그 문장으로 즉시 검색 시도 — 입력과 **동일 취급**, 규칙을 하나로) 로그인 게이트가 뜬다. 게이트는 ① 입력한 문장을 보여주고 ② `/listings` 탈출구를 제공한다.
- **문장을 삼키지 않는다.** `redirectedFrom`은 **경로만** 나르고 폼 상태는 못 나른다 → 질의는 sessionStorage로 보존해 로그인 복귀 시 입력창에 **복원**한다(커서 끝). 복원 직후 값은 제거(1회용).
- **자동 실행은 하지 않는다.** 복원까지만 하고 마지막 클릭은 사용자 몫이다. 근거: ♡ 찜은 무료·멱등이라 복귀 후 자동 반영이 안전하지만, **AI는 두 번 실행되면 두 번 청구된다** — 트리거를 *페이지 로드*에 매달면 새로고침·뒤로가기·이중렌더가 전부 재과금이 되고, 막으려면 멱등성 가드(상태머신)를 얹어야 한다. 의도치 않은 LLM 과금은 이번에 고친 바로 그 버그다.
- **잔여 횟수 카운터는 어디에도 두지 않는다**(N회 허용안과 함께 폐기).
- **원칙 재해석 — "게이트는 결과 뒤에" → "게이트는 가치 뒤에".** (B)안에서 이 원칙은 "AI 결과 뒤에"였고 그건 로그인 필수 하에선 불가능하다. 하지만 알맹이는 *"요구하기 전에 가치를 준다(value before ask)"* 이고, 그 **가치가 반드시 AI 답변일 필요는 없다.** 비로그인 방문자의 첫 질문은 "여기 살 만한 차가 **있긴** 있나?"이고 — 그건 순수 DB 조회로 충족된다. **Story 11.4(인기·최신 매물 그리드)가 히어로 바로 아래에서 이미 그 답을 준다.** 두 번째 질문("내 조건으로 물어보면?")이 태어난 뒤에야 게이트를 만난다. 게이트는 결과를 뺏는 벽이 아니라, **질문을 맡아뒀다 돌려주는 문**이다.
- **자기모순 정정:** D2는 "AI가 첫인상"이라 말하고 `EXPERIENCE.md` 페르소나 ③은 "열람+찜이 첫인상"이라 말해 왔다. 이 틈 위에 Story 11.3이 지어졌다. 이제 **열람(+11.4 매물 그리드)이 첫 가치, AI는 그 다음 행동**으로 정렬한다. 페르소나 ③에도 AI 시도 단계를 반영함.

### ✅ D3(확정) — 매물 카드: 레이아웃 B · 초록 신뢰뱃지 · amber 가격 전용
_확정 2026-07-12 · 창작물: `.working/card-variants-1.html`(A/B/C) → `.working/card-final-1.html`(최종 대형)_

- **레이아웃 B(아래 한 줄형):** 사진(깨끗, "N장" 배지만) → **신뢰속성 전용 행**(사진 바로 아래) + "판매자 제공 정보" 면책(ink-muted 11px) → 차량명 + 가격 → 희소옵션 칩.
- **정보 위계(FR32):** 차량명 "[제조사] 모델·연식" → meta(주행·연료·지역, muted) → **가격 최상위**(26px/800, price-emphasis `#E08A1B` amber).
- **신뢰속성 색 = 차분한 초록**(무사고/1인소유/비흡연): bg `#E7F3EC`/ink `#1F7A44`(light), bg `#1E3A2A`/ink `#7FCE9E`(dark), ✓ 글리프. **amber는 가격 전용**(신뢰뱃지에 amber 금지 — 겹침 방지). 부분 신뢰속성 허용(일부만 표시).
- **찜(♡)=사진 밖** 우상단 원형 버튼(그림자). **N장 배지=사진 위** 다크 글래스 pill 우하단.
- **희소옵션 칩(FR31):** petrol 톤 아웃라인, 카드 3~4개 · 그리드 밀도에선 "대표 1개 + 외 N개" 압축.
- **사진 준비중 상태(FR27):** placeholder-bg `#F1EFE9`, 중앙 카메라 아웃라인 글리프 + 조용한 문구 — 의도적으로 보이게(깨진 느낌 금지).
- **CRAFT:** 겹 그림자(0 1px 2px / 0 10px 30px -14px) + 호버 리프트, radius 16px(카드)/9px(칩), 여백 18~20px, 사진 5:3.
- **구현 메모:** "N장" 배지 `backdrop-filter: blur()`는 구형 WebView(Flutter)에서 no-op 가능 → 불투명 다크 pill 폴백으로도 가독 OK.

### ✅ D4(확정·governing) — UI 카피는 소비자 자연어, 개발/스펙 용어 금지
_확정 2026-07-12 (사용자 지침 1)_

실제 UI에 **개발단 용어("매물 탐색", "탐색" 등) 노출 금지** → 소비자가 쓰는 자연어로. 도메인 표준어("매물"은 엔카/K Car도 사용)는 허용하되 "탐색" 같은 기술 티는 배제.
- **최종 치환(D13에서 확정):** `매물 탐색 → 내 차 사기`(엔카·K Car·KB 표준 "내차사기", 내 차 팔기와 사기↔팔기 대칭), `AI 검색(내비 라벨) → AI로 찾기`. "내 차 팔기"·"내 차 등록" 유지. (~~중고차~~=근거 없는 임시값이라 폐기 — 레퍼런스는 전부 "내차사기/내차구매".)
- **적용 범위:** 내비·버튼·섹션 헤더·빈 상태·에러 등 모든 사용자 표면. 스펙 문서 내부 용어(FR·표면명)와 UI 라벨을 구분.

### ✅ D5(확정·governing·전 UI) — 반응형 무결성: 배치로 흡수, 내부 찌그러짐 금지
_확정 2026-07-12 (사용자 지침 2) — **카드뿐 아니라 모든 UI 기본 방침**_

- 가로폭 축소 시 **개별 컴포넌트 내부 레이아웃을 절대 찌그러뜨리지 않는다.** 가로 배치가 의도된 요소(카드의 신뢰속성 행·meta·옵션 칩, 태그·텍스트 등)는 **가로 유지 강제**(줄바꿈으로 세로화 → 어긋남 = 금기).
