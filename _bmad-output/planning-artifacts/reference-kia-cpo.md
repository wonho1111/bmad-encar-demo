# UI/UX 레퍼런스 조사 — 기아 인증중고차 (Kia Certified Pre-Owned)

**대상**: 기아 인증중고차(cpo.kia.com) — 기아 공식(제조사 직영) 중고차 인증·판매 플랫폼
**조사일**: 2026-07-11
**조사 방법**: agent-reach(exa_search 웹검색 + Jina Reader 페이지 실제 추출 + YouTube 자막 검색). 스크린샷 불가 → 실제 사이트 HTML/텍스트를 직접 추출해 구조 확인.
**주의**: 유튜브에서 "기아 인증중고차 UI/앱 워크스루" 전용 영상은 검색되지 않음(주로 딜러 비교·후기 콘텐츠). 대신 실제 웹사이트를 Jina Reader로 3개 페이지(홈/목록/상세) 직접 크롤링해 화면 구조를 원문 그대로 확인했으므로 신뢰도는 오히려 더 높음.

## 0. 표면별 개요 (① 웹 ② 모바일웹 ③ 앱)

기아 인증중고차는 **별도의 전용 앱이 없다.** 매물 탐색·구매·판매 전 과정은 `cpo.kia.com` 단일 웹사이트(반응형, PC/모바일 겸용)에서 이뤄진다.
- "기아 인증중고차 홈페이지는 별도의 앱을 설치할 필요 없이 웹으로 접속해 바로 회원가입 후 자유롭게 이용 가능한 점이 장점" (출처: ittong.kr 후기)
- 기아의 통합 앱 **"Kia App"**(iOS 앱스토어 844MB, 평점 4.5/2.1만개)은 존재하지만, 이는 신차오너용 커넥티드카 앱(원격시동·공조·디지털키·충전소 등)이며, "마이카" 섹션에 "차량 선택부터 구매·결제·인수까지" CPO 구매 흐름이 **부가 기능으로 통합**되어 있을 뿐 CPO 전용 앱은 아님. 즉 ③ 앱 표면은 "없음(웹 중심) + 신차 통합앱 안에 구매흐름 일부 편입" 으로 명시.
- 모바일웹은 별도 m.도메인이 아니라 `cpo.kia.com` 동일 도메인의 반응형 레이아웃으로 추정(기아 그룹 사이트 리뉴얼 사례에서 "모바일 환경에서도 한 화면 안에서 핵심 정보 확인–선택–행동이 이어지도록" 설계했다는 근거 확인, 단 브레이크포인트별 실측 스크린샷은 확보 못함 — "확인 불가"로 남김).

## 1. 매물 카드

목록 페이지(`cpo.kia.com/products/`)에서 실제 추출한 카드 구조:

- **대표사진**: 카드마다 이미지가 2장 겹쳐 노출됨 — ① 모델 실루엣/렌더 이미지(회색 배경 스튜디오컷, 예: `GLSD22AGT_K8.png`) ② 실제 매물 실사(merchandising 사진, 예: `car/6bc6a792-...`). 즉 **렌더 이미지를 우선 노출하고 실사를 보조로 함께 보여주는 이중 이미지 구조**로 보임(정확한 스와이프/오버레이 동작은 확인 불가, 두 URL이 함께 나열됨).
- **정보 위계** (위→아래 순): 타임딜 배지+잔여시간 → 연식+모델명("2022 K8 Premium") → 트림/파워트레인 상세("1.6 가솔린 하이브리드 A/T 시그니처") → 출고연월+주행거리+차량번호 뒷자리("2022년 02월(22년형)ㆍ 59,741kmㆍ 124마6797") → **가격(정가 취소선+할인가 볼드, "3,011만원 ~~3,511만원~~ 500만원 할인")** → 뱃지 리스트 → **사진 개수 숫자**(카드 우측/하단에 작은 숫자로, 예: 57, 16, 26).
- **지역 정보 없음**: 엔카·헤이딜러류와 달리 카드에 "지역(시/도)"이 없음. 전국 단일 재고로 보고 **탁송(집앞 배송)** 전제이기 때문으로 추정 — 국내 제조사 직영 모델의 특징.
- **뱃지 종류**: `타임딜`(할인+카운트다운), `예약중인 차량`(+"N명 상담예약중"), `오토할부혜택`, `보험이력없음`, `세이프티케어`, `스페셜위크`. 무사고 여부를 "보험이력없음"이라는 구체적 문구로 표기(엔카의 "무사고"보다 더 명확/보수적인 워딩).
- **찜·비교 버튼**: 카드 좌상단에 **"비교하기"** 체크박스형 버튼이 항상 노출(엔카의 하트 찜과 달리 처음부터 "비교" 기능이 1급 시민). 찜(하트) 자체는 카드에 노출되지 않고, 상세페이지에만 "계약금 결제하고 찜할 수 있습니다"라는 문구 확인 — **찜이 유료 예약(디파짓)과 결합된 개념**으로 설계됨(단순 북마크가 아님). 이는 일반 중고차 플랫폼과의 큰 차이점.
- **필터 칩**: 목록 상단에 총 매물 수("948대")와 함께 `타임딜`, `예약차량 제외` 같은 토글이 카드 목록 바로 위에 배치.

## 2. 이미지 처리

- 카드: 렌더 이미지 + 실사 이미지 2장(상세 스와이프 동작은 미확인).
- 상세페이지: "본 차량의 이미지는 실제와 다를 수 있습니다" 고지문이 최상단에 명시 — 렌더/실사 혼용을 사용자에게 미리 알리는 패턴.
- **360도 VR + 실사 결합**: 3자 후기 기사(foxcg.com)에 따르면 "차량을 클릭하면 고화질 360도 VR 화면이 실행되어 외관 스크래치, 실내 시트 상태까지" 확인 가능하다고 서술됨(직접 페이지 크롤링으로는 VR 위젯 자체를 확인 못했으나 두 개의 독립 출처가 동일하게 언급 — 신뢰도 중간).
- **품질검수 증빙 사진**: 상세페이지에 "개선항목" 섹션이 별도로 있고 "외관 및 내장 6건", "기능수리 및 소모품 4건"처럼 **개선 전/후 작업 항목이 건수로 집계되어 노출**됨(딥링크로 사진 확인 가능 구조로 추정, 실제 사진 크롤링은 페이지네이션/JS 렌더라 미확인).
- **사진 개수 배지**: 카드에 숫자만(예: "57")로 표기되고 "장" 같은 단위 텍스트는 없음 — 아이콘(카메라 등)과 조합됐을 가능성이 높으나 Jina 텍스트 추출로는 아이콘 확인 불가.
- **플레이스홀더**: 사진 없음 케이스는 발견 못함(전 매물이 실사+렌더 최소 2장 보유 — 직영 상품화 프로세스상 무사진 매물이 없는 구조로 추정).

## 3. 목록/필터·검색

- 목록 페이지 상단에 **퀵필터 칩**(홈페이지 상단에도 동일 노출): `가성비 최고`, `1천만원대`, `타임딜`, `보험이력없음`, `전기차량`, `하이브리드`. URL 쿼리(`?filter=classifications[]=LITE`, `?filter=minPrice=...&maxPrice=...`)로 직접 딥링크되는 구조 — 필터 상태가 URL에 인코딩됨(공유 가능).
- 목록 페이지 자체는 "필터" 섹션 헤더만 텍스트로 노출되고 상세 필터 옵션(가격/연식/주행거리/차종/연료/트림 등 슬라이더·드롭다운)은 JS 렌더 영역이라 원문 추출로는 상세 항목 확보 못함 — **"확인 불가"**로 남김(다만 필터 존재 자체와 URL 파라미터 방식은 확인됨).
- **정렬 옵션**: 원문 추출 범위에서 명시적 정렬 드롭다운 텍스트는 확인 못함(확인 불가).
- **AI/자연어 검색**: 확인 안 됨 — 기아 인증중고차는 자연어 검색 기능이 없는 것으로 보임(우리 프로젝트의 AI 자연어 검색이 상대적 차별점이 될 수 있음).
- **차종으로 찾기**: 홈 하단에 모델 아이콘 그리드(레이, 쏘울, EV3, EV4, EV5, EV6, EV9, PV5, 니로 등)로 브랜드 라인업을 시각적으로 탐색하는 섹션 — 텍스트 검색이 아니라 **아이콘 그리드 탐색**이 메인 진입점 중 하나.

## 4. 매물 상세

Jina Reader로 실제 상세페이지(`/products/detail/?id=12823`) 전체 구조를 순서대로 추출:

1. 실제와 다를 수 있다는 이미지 고지
2. **차량 정보**(차량번호, 차종, 인승/도어, 배기량, 외장/내장 색상)
3. **제원 정보**(엔진 출력/토크, 연비 등급·도심/고속도로/복합연비·CO2, 섀시/서스펜션/브레이크, 규격 전장/전폭/전고/축거)
4. **보험항목**(내 차 피해 횟수, 용도변경 여부) + **압류/저당 여부**
5. **옵션 정보**(선택옵션 목록을 아이콘+텍스트로, "이 차량에는 300만원 상당의 선택옵션이 장착" 처럼 옵션의 신차 당시 금전가치까지 명시)
6. **잔여 보증 내역**(차체/냉난방/엔진/동력전달/배출가스/커스터마이징 6개 항목별 잔여 km·기간, "최대 1년 2만km 연장" 보증)
7. **개선항목**(품질검수 증빙, 외관·내장/기능수리·소모품 건수)
8. **최근 거래가**(동급 매물 최근 1~3개월 거래가 — 가격 신뢰 근거 제공, 데이터 부족 시 "수집 중"으로 정직하게 표기)
9. 대출/캐시백 배너 → **가격 + 월 할부금 계산기**(선수금 30%/금리 7%/48개월 가정 노출)
10. **추천 차량**(같은 카드 UI 재사용)
11. 하단 신뢰 요소 나열: "7일 안에 환불 가능", "계약금 결제하고 찜할 수 있음", "제조사가 인증한 차량", "리멤버스 혜택"

**CTA**: 전화 상담 버튼은 눈에 띄지 않음(비대면 온라인 계약이 기본 흐름). "찜"이 곧 "계약금 결제 예약"과 결합되어 있어 일반 중고차 플랫폼의 무료 찜과 성격이 다름. "공유" 버튼 존재.

## 5. 랜딩/홈

홈(`cpo.kia.com/`) 실제 구조:

1. 퀵필터 칩 6개(가성비 최고/1천만원대/타임딜/보험이력없음/전기차량/하이브리드)
2. **히어로 배너**(이벤트/견적 유도 배너 1장)
3. **"실시간 인기차량"** — 카드 캐러셀(5대)
4. **"이런 차는 어떠세요?"** — 큐레이션 탭(스페셜위크/EV차종추천/첫차강력추천/신혼부부추천/베스트SUV) 전환 시 카드 리스트 갱신
5. **"차종으로 찾기"** — 모델 아이콘 그리드
6. **"기아 인증중고차 소개"** — 4개 카드형 소개 섹션(신차 트레이드인 혜택 / 제조사가 인증하는 차량 / 쉽고 빠른 내 차 팔기 / 믿을 수 있는 내 차 사기), 각각 이미지+타이틀+한줄설명+"자세히보기" 링크

별도 회사소개 페이지(`kia.com/kr/buy/rent-used-car/used-car`)에서는 **수치 기반 신뢰 요소**를 전면에: "6년/12만km 미만 차량만 매입", "200개 항목 인증검수", "EV 배터리 등급 AAA·신차대비 95~100% 주행거리", "최소 1년 2만km 보증", "최대 50만 포인트 지급", "기아 커넥트 최대 1년 무상".

## 6. 반응형

- 동일 도메인(`cpo.kia.com`)이 PC/모바일 겸용으로 운영되는 것으로 확인(별도 m. 서브도메인 없음).
- 기아 그룹 웹사이트 리뉴얼 케이스스터디(ipartners.co.kr)에서 확인된 설계 원칙: "**모바일 환경에서도 한 화면 안에서 핵심 정보 확인 – 선택 – 행동이 이어지도록**" 구매 중심 UX 구조 설계. "메뉴와 화면은 줄이고, 선택과 행동은 더 또렷하게" — 콘텐츠 나열형이 아니라 **다음 행동(CTA)을 명확히 하나만 제시**하는 방향.
- 그리드 열수, 브레이크포인트 등 정량적 수치는 실측하지 못함 — **확인 불가**로 남김(추정 금지).

## 7. 디자인 언어

- **브랜드 컬러**: 과거 레드에서 **블랙(Midnight Black) / 화이트(Polar White) 조합**으로 전면 교체(2021년 리브랜딩, 사명도 "기아자동차"→"기아"). 디자인 철학은 **"Opposites United"** — 무제한적 표현을 상징하는 블랙/화이트 대비. (출처: blog.kiacareers.co.kr, studio-d-d.com)
- **브랜드 슬로건**: "Movement that inspires"(일상의 움직임 속 영감) — CPO 소개 문구에도 이 철학이 이어져 있음.
- **디자인 가치 키워드**: 진보적(progressive), 경건함(holiness/절제), 단순함(simple), 책임감(responsibility) — 기아 글로벌디자인담당 인터뷰(newspim.com)에서 확인.
- **톤**: 후기 기사에서 "군더더기 없는 카테고리가 무척이나 인상적", "직관적" 이라는 평가 — 메뉴 수를 최소화(내차사기/내차팔기/소개 정도의 최상위 구조)해 제조사 특유의 절제된 고급감을 냄. 옵션/가격 정보량은 매우 많지만(제원표, 옵션가치 환산 등) 이를 카드형 컴포넌트로 잘게 쪼개 어수선함을 줄임.

## 우리 프로젝트 적용 시사점

1. **카드에 "사진 개수" 숫자 배지 도입** — 기아 CPO 카드는 가격 옆에 사진 매수를 작은 숫자로 항상 노출한다. 우리 매물 카드에도 대표사진 위 우측 하단 등에 "N장" 배지를 붙이면, 사진이 부실한 매물과 충실한 매물을 목록에서 바로 구분할 수 있어 이미지 계약(사진 개수·순서) 확장 작업과 자연스럽게 맞물린다.
2. **뱃지를 "무사고" 대신 구체적 문구로** — 기아는 "보험이력없음"처럼 근거를 특정해 표기한다. 우리도 "무사고"라는 모호한 배지보다 "보험이력조회 결과 없음" 식으로 출처를 명시하는 문구가 신뢰도에 유리하다(단, 우리는 데모 데이터라 실제 보험이력 조회 연동은 없음을 전제로, 문구만이라도 참고).
3. **"최근 거래가" 위젯** — 상세페이지에 동급 매물의 최근 거래가를 보여줘 "이 가격이 합리적인지"를 사용자가 스스로 판단하게 한다. 우리 서비스도 매물 상세에 "유사 매물 평균가" 같은 참고 정보를 붙이면 헤이딜러류 직거래 서비스의 약점(가격 불투명성)을 보완할 수 있다(우리 AI 검색 기능과 결합하면 차별화 포인트가 될 수 있음).
4. **랜딩의 "차종으로 찾기" 아이콘 그리드 + "이런 차는 어떠세요" 탭형 큐레이션** — 텍스트 검색바 하나만 덜렁 놓기보다, 홈 화면에 모델 아이콘 그리드와 상황별 큐레이션 탭(첫차/신혼부부/베스트SUV 등)을 두면 검색어를 못 정한 사용자의 진입장벽을 낮출 수 있다. 우리 AI 자연어 검색은 이미 있으므로, 랜딩에서는 "AI 검색으로 바로 물어보기"를 강조하는 히어로 + 보조로 카테고리 그리드를 두는 조합이 유효해 보인다.
5. **반응형은 "모바일 원스크린 결정" 원칙** — 기아 사례는 모바일에서 정보 나열이 아니라 "확인 → 선택 → 행동"이 한 화면 흐름 안에서 끊기지 않게 설계했다. 우리 반응형 작업 시 모바일 상세페이지는 스크롤 하나로 사진→핵심정보→CTA(문의/찜)가 이어지도록 섹션 순서를 재점검할 근거로 쓸 수 있다.
6. **디자인 언어(절제된 무채색 + 카드형 정보 분해)** — 기아는 블랙/화이트 중심의 절제된 팔레트와, 많은 정보량(제원/옵션/보증)을 여러 개의 작은 카드 컴포넌트로 쪼개 어수선함을 줄인다. 우리 UI 개편 시에도 매물 상세의 많은 필드(옵션/보증/사고이력 등)를 하나의 긴 표 대신 섹션 카드 단위로 나누는 패턴이 참고할 만하다. 단, 우리는 브랜드 고급감보다 "직거래 신뢰"가 우선이므로 색상까지 그대로 따라갈 필요는 없다.
