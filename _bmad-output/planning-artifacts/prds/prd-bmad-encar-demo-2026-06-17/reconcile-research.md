## 2. 항목별 상세

### 강하게 반영된 부분 (모순 없음)

- **배포/Vercel(항목 10):** 연구 7장의 재검토 노트(경부하 시연 → Vercel 단일 배포, 부하 증가 시 백엔드만 Cloud Run 분리)가 `NFR4`와 Addendum "Vercel 배포 유의사항"에 그대로 옮겨졌다.
  - **무상태(stateless):** PRD `FR18`, `NFR5`, Addendum 멀티턴 메모 모두 "서버·DB에 AI 대화 미저장, 클라이언트 보관" 일관.
  - **연결 풀링:** Addendum "Supabase 연결 풀링 설정 필요" — 연구 7.2 유의 3가지 중 ②와 일치.
  - **번들 500MB / 함수 300초 / 무거운 의존성 회피:** Addendum에 수치까지 명시 — 연구와 일치.
- **폴링 채팅:** 연구 7.2 "폴링이면 WebSocket 불필요"[확실] → `FR20`, `NFR1`, 범위 밖(웹소켓 제외)에 반영.
- **필터+벡터 함정(항목 7):** 연구가 직접 권한 "정밀 조건은 SQL 경로로 분리"가 `FR14/FR15`, 4장 경로 A/B 분리로 구현됨. 함정을 설계로 회피 → 모순 없음.
- **pgvector·2경로 RAG:** Addendum "벡터 검색: Supabase pgvector", RAG 코퍼스(차량 상식·구매 가이드 문서)를 **필수 산출물**로 명시 — 연구 5장과 일치.

### 부분 반영 / 약화된 부분

- **SQL 안전장치(항목 4, [확실]):** 연구는 "프롬프트 규칙만으로는 안전하지 않다 → **결정론적 검증 필수**"를 핵심 결론으로 강조했다. PRD/Addendum는 "SELECT 전용·범위 제한·화이트리스트 컬럼·판매완료 제외" + RLS 권장까지는 담았으나, 연구가 **필수**로 본 다음 3가지가 명시적으로 보이지 않는다:
  - **읽기전용 DB 롤**(SELECT만 GRANT) + `statement_timeout`
  - **앱 레벨 결정론적 검증**(`sqlparse` 파싱으로 SELECT만 통과)
  - **기본 `LIMIT` 강제**(과도 조회 방지)
  - → 데모 수준에선 치명적이지 않으나, "어떻게 SELECT 전용을 보장하는가"가 프롬프트 의존으로 흐를 위험. 아키텍처 단계에서 결정론적 검증·읽기전용 롤을 명시 권장.

- **Gemini 프라이버시(항목 9):** 연구 6.2는 "무료 등급은 입력/응답이 **학습에 사용될 수 있음**, 민감정보(연락처 등) 프롬프트 미포함 설계" 권고. PRD에는 비용(NFR6)만 반영되고 이 프라이버시 경고는 누락.

### 누락된 결정 (연구엔 있으나 PRD/Addendum에 없음)

- **벡터 인덱스 HNSW(항목 5):** HNSW 채택·시작 파라미터, "IVFFlat은 빈 테이블/마이그레이션에 넣지 말 것" 경고가 전혀 없다. 인덱스 선택은 검색 품질·성능 직결 결정.
- **임베딩 차원 일치(항목 6, [확실]):** "임베딩 모델 출력 차원 = `vector(N)` 컬럼 차원, 불일치 시 미동작" — DB 스키마에 직접 영향을 주는 [확실] 제약인데 어디에도 없다. (데모 권장 768/1536도 미반영)
- **SQL Agent 방식(항목 3):** Agent(자가수정·최대 3회 재시도) 대 Chain 결정이 없다.
- **[미확인] 인수인계(항목 11):** 연구가 "구현 전 반드시 확정"으로 못박은 3가지 — ① Gemini 생성/임베딩 모델의 **정확한 이름·차원·무료 한도**, ② Text-to-SQL용 모델 등급(Flash 충분 여부 검증), ③ pgvector/LangGraph **버전 고정** — 이 PRD·부록에 재확인 항목으로 명시 전달되지 않는다.

### 모순(contradiction)

- **명확한 모순 없음.** Addendum이 LLM을 "Gemini Flash 계열"로 적은 것은 연구의 Flash 권고와 일치한다. (연구가 [미확인]으로 단 정확한 모델명 확정 의무만 누락된 상태이지, 어긋난 값은 아님.)

## 3. 결론

배포·Vercel 한도·풀링·무상태 등 **연구가 핵심 리스크로 다룬 배포 영역은 충실히 반영**됐고 모순은 없다. 다만 DB 스키마·검색 품질에 직접 영향을 주는 [확실] 제약 일부(임베딩 차원 일치, HNSW 인덱스)와 SQL 안전장치의 결정론적 검증 부분, 그리고 "구현 전 재확인" [미확인] 항목 인수인계가 PRD/Addendum에서 빠져 있다. 저위험 데모 PRD 기준으로 대부분 **아키텍처 단계로 넘길 수 있는 구현 디테일**이나, 임베딩 차원/SQL 결정론적 검증/HNSW는 다운스트림에서 놓치지 않도록 Addendum에 한 줄씩 보강하는 것을 권장한다.
