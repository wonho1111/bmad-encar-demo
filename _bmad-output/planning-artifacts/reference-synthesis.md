- **작성일**: 2026-07-11 · **작성**: opus 종합 (5개 sonnet 조사 리포트 취합)
- **원본 조사**: `reference-encar.md` · `reference-kcar.md` · `reference-kbchachacha.md` · `reference-hyundai-cpo.md` · `reference-kia-cpo.md`
- **목적**: 이번 증분의 4개 고도화 항목(이미지 추가 · UI 개편 · 반응형 · 랜딩)에 **바로 쓸 결정거리**만 교차 정리. 앱별 반복은 원본 참조.
- **필터링 원칙**: 우리는 **개인 직거래 데모**(엔카·헤이딜러류) + **AI 자연어 검색 이미 보유** + **Next.js 단일 반응형** + **기존 매물 100건(사진 없음)**. 이 4개 전제에 안 맞는 패턴(금융계산기·360VR·타임딜·워터마크·개인화추천엔진)은 **A2 단순함 원칙상 범위 밖**으로 명시 제외.

## 1. 이미지 (이번 증분의 핵심) — 가장 직결

**우리와 가장 강하게 연결되는 영역.** nullable 이미지 마이그레이션 설계에 그대로 반영.

- **① 비율 통일 (엔카의 명시적 교훈).** 엔카는 2025-06 개편에서 리스트/상세/마이페이지 전 화면 이미지를 **5:3으로 통일**해 "디바이스별 시각 차이"를 없앴다. → 우리도 이미지 계약에 **고정 비율 하나**(후보: 5:3 또는 4:3)를 정하면 카드·상세·썸네일 어디서든 동일 크롭 로직 재사용 → 이미지 확장 작업이 단순해진다. 업로드 UI에도 같은 비율 크롭 가이드.
- **② 복수 사진 전제 (KB차차차·기아).** KB차차차는 **리스트 카드 단계부터** 여러 장(데스크톱 4장 나열 / 모바일 스와이프 "1/4"), 기아는 카드에 **사진 개수 숫자 배지**("57"). → 이미지 계약을 대표사진 1장이 아니라 **배열(대표 + 추가 N장)**로 설계하고, 카드에 "N장" 배지로 사진 충실도를 목록에서 바로 구분. (성능: lazy-load·썸네일 리사이즈 동반)
- **③ 사진 없을 때 fallback (현대·KB) — 우리 필수 엣지케이스.** 현대는 사진 준비 안 된 매물을 **"판매예정(Coming Soon)" 상태로 분리 + 모델 대표컷 placeholder + "실제와 다를 수 있음" 고지**, KB는 `noimage.jpg`. → **우리는 기존 매물 100건이 전부 사진 없음** → nullable 컬럼 도입 시 "사진 없는 매물"의 placeholder UI("사진 준비중" 아이콘+문구)를 **반드시** 정의해야 함(빈 박스로 두면 신뢰 하락). 이게 이번 마이그레이션의 1순위 엣지케이스.
- **④ 사진 위 오버레이 = 신뢰 정보만, 액션 버튼은 밖.** KB차차차는 "무사고·실차주·진단보증"을 **사진 위 캡션 오버레이**로 노출. 반대로 엔카는 2025 개편에서 **찜/비교 버튼을 사진 "밖"으로** 뺐다(사유: "차량을 가리지 않기 위함"). → 규칙: **사진 오버레이엔 신뢰 라벨(무사고 등)만, 찜/비교 액션 아이콘은 카드 정보 영역으로.**
- **범위 밖**: 360도 VR·엔진음 클립·시트 6배줌(현대 "오감만족")·워터마크 합성 → 데모 과설계.

## 2. 매물 카드 정보 위계 (UI 개편)

- **표준 순서 = 차량명 → 연식 → 주행거리 → 연료 → 지역 → 가격.** 엔카가 PC·모바일 통일한 순서이며 5개 공통. **가격은 압도적으로 크게(대형 볼드)** — KB·현대·기아 모두 취소선 원가+실판매가 형태로 가격 타이포를 극단적으로 강조(우리는 할인 개념 없으니 실가격만 크게).
- **스펙은 한 줄 압축.** KB차차차는 "21/04식 35,126km 가솔린 경기"처럼 연식·주행·연료·지역을 **구분자 없이 한 줄**로. 스캔 용이.
- **뱃지는 구체적 문구로.** 기아는 "무사고" 대신 **"보험이력없음"**(근거 특정). 우리는 데모라 실제 조회 연동은 없지만 문구 톤 참고. **중요 차이**: 현대/기아 CPO는 브랜드 전체가 인증이라 카드에 뱃지를 안 붙이지만, **우리는 개인 직거래라 개별 매물 뱃지(무사고/판매자)가 오히려 맞다.**
- **특징 태그 2~4개.** K Car P2P 카드: `1인소유` `비흡연` `빠른거래가능` `가격제안가능`. → 우리 seed의 `accident_free`·`options`(후방카메라 등)를 **카드 태그로 노출**하면 저비용으로 정보 밀도↑. (PRD에서 후보로만 — 요청 없는 확장 지양)

## 3. 랜딩/홈 (랜딩 항목)

- **참조 최적 = K Car "안심직거래"(market.kcar.com).** 5개 중 유일한 **개인 P2P 마켓**이라 우리와 사업모델 최유사. 특징: **서비스 소개 없이 매물 그리드로 바로 진입** — "지금 뜨고 있는 차량"(인기) / "최근 등록된 차량"(최신) **2단 그리드**. → 우리 랜딩의 뼈대로 그대로 채택 가능(단순).
- **검색 위젯을 히어로 바로 아래(현대).** 우리는 **AI 자연어 검색을 이미 보유** → 랜딩 히어로에 "AI로 원하는 차 물어보기"를 주인공으로, 보조로 카테고리/모델 그리드(기아 아이콘 그리드). K Car의 "탐색 의도 3분기(모델/예산/차종)"는 AI 검색창이 상당부분 흡수 가능.
- **개인화 위젯(엔카: 최근 본/찜/인기 TOP)** = 재방문 UX엔 좋지만 **데모 규모엔 과설계** → "인기/최신 2단"까지만.
- **범위 밖**: 프로모션 배너 캐러셀·정시 신규오픈(현대 "매일 5시")·타임딜·금융 연계 위젯.

## 4. 반응형 (반응형 항목)

- **아키텍처 주의**: 엔카·K Car·KB차차차는 **m-dot 별도 도메인**(모바일 전용 사이트) 구조라 **우리(Next.js 단일 반응형)와 다름 → 레이아웃 직접 재사용 불가.** 반면 **현대·기아 CPO는 단일 도메인 반응형**(우리와 동일 아키텍처) → **이쪽이 참조점.**
- **인터랙션 전환 아이디어(재사용 가능)**:
  - 카드 사진: **데스크톱 그리드 나열 → 모바일 스와이프 캐러셀**(KB차차차).
  - 필터: **데스크톱 사이드바 → 모바일 바텀시트/드릴다운(전체화면+뒤로가기)**(K Car).
  - 내비: 데스크톱 상단 메뉴 → 모바일 햄버거+아이콘 그리드.
- **원칙(기아)**: 모바일은 "확인 → 선택 → 행동"이 **한 화면 흐름에서 안 끊기게**. 상세페이지는 스크롤 하나로 사진→핵심정보→CTA(문의/찜) 연결.

## 5. 필터·검색 + 매물 상세 (부수 참고)

- **필터 UI**: 사이드바 + **색상 스와치(SVG)** + **가격 프리셋 칩("2천만원 이하")+직접입력** 병행(현대·K Car). **반면교사(KB차차차)**: 가격 구간이 1500만 이후 500만 단위로만 나뉘어 실사용자 불만 → **우리 가격 필터는 촘촘히/자유입력.** 엔카 교훈: 필터 첫 화면 순서("차종 우선" vs "가격/주행 우선")는 논쟁적 → 신중히.
- **AI 검색 = 우리 차별점.** 5개 중 자연어 AI 검색을 제대로 하는 곳은 엔카(2025, **인증 매물 한정**)뿐. → 우리 AI 자연어 검색이 오히려 앞선 차별점. **엔카 교훈 채택**: AI를 전체 매물에 무리하게 넓히기보다 **신뢰 데이터 하위셋부터** 단계 적용.
- **매물 상세**: 신뢰 요소(진단/무사고)를 **가격 바로 아래 상단**에(엔카·K Car), 판매자 연락처는 하단. 많은 필드는 **긴 표 대신 섹션 카드로 분해**(기아). "최근 거래가/유사매물 평균가"(기아)는 **우리 AI 검색과 결합 시 차별화 포인트**(가격 불투명성 보완) — 여력 시 후보.

## 6. PRD로 넘길 결정거리 (요약)

1. **이미지 스키마** = `listings`에 **nullable 이미지 배열**(URL 여러 장) + 고정 비율 1개 + 대표사진 지정. 기존 100건 안 깨지게 non-null 금지. (계약: `docs/conventions.md §4 ListingCard` + `project-context 규칙 5`부터 확장)
2. **사진 없음 fallback** = 기존 100건용 placeholder UI 정의(이번 마이그레이션 1순위 엣지케이스).
3. **카드 리디자인** = 대표사진(고정비율) + "N장" 배지 + 표준 정보위계 + `accident_free`/`options` 뱃지·태그화. 찜/비교는 사진 밖.
4. **랜딩** = 매물 그리드형(K Car P2P 참조) + AI검색 히어로 + 인기/최신 2단.
5. **반응형** = 단일 Next 반응형 유지, 카드사진 그리드→캐러셀·필터 사이드바→바텀시트·내비 햄버거.
6. **AI 검색** = 랜딩 전면 배치(차별점), 신뢰 하위셋 우선 원칙.
7. **명시적 범위 밖(A2)**: 360VR·엔진음·워터마크·금융계산기·타임딜·정시오픈·개인화추천엔진·m-dot 분리.
