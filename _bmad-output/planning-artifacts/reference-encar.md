# UI/UX 레퍼런스 조사 — 엔카 (Encar)

- **대상**: 엔카(Encar, encar.com) — 국내 1위 중고차 직거래·딜러매물 플랫폼 (K Car와 함께 국내 양대 플랫폼)
- **조사일**: 2026-07-11
- **조사 방법**: agent-reach 스킬 (exa_search 웹검색 + Jina Reader 페이지 본문 추출 + YouTube 자막)
- **주의**: 스크린샷 캡처 불가 환경이라 전부 텍스트 기술. 확인 안 된 항목은 "확인 불가"로 명시. 추측 없이 출처 있는 사실만 기재.

## ① 웹(PC, encar.com / car.encar.com)

### 1. 매물 카드
- 2025-06-26 PC 전면 UI/UX 개편(엔카닷컴 공식 발표) 이후 기준:
  - **찜/비교 버튼을 사진 "안"에서 "밖"으로 이동** — 공식 사유: "차량을 가리지 않아 더 나은 사용자 경험을 제공"(엔카 공지사항 원문). 즉 사진 위에 아이콘을 얹지 않고 카드 하단/옆 별도 영역에 배치하는 방식으로 바뀜.
  - 정보 위계(모바일과 통일): **차량명 → 연식 → 주행거리 → 연료 → 지역 → 가격** 순. PC는 화면이 넓어 성능기록부 공개 여부도 추가로 노출.
  - 화물차/특장차는 적재규격·지역 정보를 별도 표시.
  - 카드에 표시되는 **서비스 라벨**: `집앞배송`, `환불가능`, `엔카종합보증`(엔카믿고 매물 한정). **차량 특성 아이콘**: `1인소유`, `비흡연`, `준신차` 등.
  - 진단 등급 배지(사진 좌상단 오버레이 텍스트 형태로 확인됨): `엔카진단`, `엔카진단+`, `엔카진단++` — 등급이 높을수록 검수 범위가 넓음(아래 4번 참조).
  - 실제 리스트 크롤 결과(car.encar.com/list/car) 예: "완무 레드시트 관리상태양호 비흡연 무도색,무판금 **BMW 1시리즈 (F40) 120i M 스포츠** 21/07식 · 32,482km · 가솔린 · 서울 · 2,550만원 · 주황색 계열" 형태로, 셀러가 입력한 한줄 카피(차량 상태 요약) + 스펙 라인 + 색상까지 카드에 노출됨.
  - 출처: encar.com 공지사항(`cs_helpdesk.do?boardId=014&regid=280858`), 매일경제(mk.co.kr/news/business/11353095), 이데일리(edaily.co.kr), car.encar.com/list/car 실제 크롤.

### 2. 이미지 처리
- **대표사진 비율 = 5:3으로 전 플랫폼 통일**(2025-06 개편). 이전에는 PC 차량목록만 4:3, 상세/모바일은 5:3으로 플랫폼별 상이했음 — 이를 5:3으로 일괄 통일해 "디바이스에 따른 시각 차이 최소화".
  - 셀프등록/직거래 사진 업로드 시에도 5:3 편집(자르기) 비율을 제공하도록 등록 UI에 추가.
  - 변경 전/후 표(공지 원문): PC 차량목록 4:3→5:3, PC 차량상세 5:3(변경없음), PC 시세·마이페이지 4:3→5:3, 모바일 차량목록/상세 5:3(변경없음), 관심차량·마이페이지(내차구매/판매관리/결제관리) 4:3→5:3.
  - 상세페이지에서는 리스트 썸네일과 별도로 대표 이미지 2장(외관 앞/뒤 등)을 우선 노출하고, 이후 진단 항목별 사진(하부 진단 사진, 셀링포인트 사진 등)이 순차 배치됨(fem.encar.com 상세페이지 실제 마크다운 추출 확인: 셀링포인트 이미지 1장 + 하부진단 UNDER_BODY 사진 1장).
- **워터마크**: 이미지 URL에 `wtmk=` 파라미터로 엔카 워터마크가 항상 합성됨(불법 도용/재사용 방지).
- **360도/실사 여부**: 확인 불가(공개 자료에서 360도 뷰 기능 언급을 찾지 못함).
- **사진 없을 때 플레이스홀더**: 리스트 크롤에서 일부 슬롯에 `trans.gif`(투명 1px)가 로딩 placeholder로 쓰이는 정황 확인 — 지연로딩(lazy loading)용으로 추정되며 "사진 없음" 전용 디자인은 확인 불가.
- 출처: encar.com 공지사항, fem.encar.com/cars/detail/41253089 (Jina 추출).

### 3. 목록/필터·검색
- 필터 카테고리(공식 검색 UI 메뉴 구조로 확인): **차종 대분류**(국산/수입/전기·친환경/화물·특장·버스) → **세부 검색**(제조사/모델/세부모델), **엔카진단 차량**, **헛걸음보상 차량**, **리스·렌트 차량** 등 서비스형 필터가 대분류 내비게이션 자체에 포함.
- 차량 목록 페이지 필터 UI(공식 리스트 페이지 텍스트 추출): "성능/보험공개", "사진우대", "진단, 보증차량" 등 체크성 필터가 존재.
- 유튜브 중고차 구매 팁 영상(자동생성 자막, 아래 출처)에서 실사용자가 실제로 거치는 필터 순서: **차종(모델) 선택 → 성능·보험이력 공개 여부 체크 → 일반/리스승계/렌트 구분 → 지역 제외 옵션 → 연식(범위) → 주행거리(범위) → 검색** 순으로 필터를 쌓는 방식. 정렬은 "가격 낮은 순"이 가장 흔히 쓰인다고 언급(정렬 옵션에 가격 오름차순 존재 확인).
- **AI/자연어 검색**: 2025년 8월 정식 론칭한 "엔카믿고 AI 챗봇" — 구글 Gemini 기반, RAG 대신 **"동적 프롬프트 생성"** 방식으로 매물 4만여 대의 실시간 데이터를 분석해 매물별 1:1 맞춤 상담 제공. 예시: "이 차 타이어 상태 어때요?" → "앞바퀴 트레드 잔량이 10mm로 양호한 상태입니다" 식으로 실시간 데이터 기반 응답. 2025-10 기준 AI 챗봇 도입 3개월 만에 일평균 이용자 8배 증가, 챗봇 경유 엔카믿고 신청 비중 약 15%. **AI 큐레이션 기능도 추가** — "AI 챗봇과 자연어로 대화하며 원하는 조건에 맞는 매물을 탐색"할 수 있다고 명시. 단, 현재는 **엔카믿고(엔카 진단완료) 매물에 한정**된 서비스로, 전체 매물 대상 자연어 검색은 아닌 것으로 보임.
- **필터 UI 개편에 대한 실사용자 불만**(구글 플레이 리뷰, mwm.ai 취합): 최근 업데이트로 "주행거리·거리·가격·차종"이 먼저 나오도록 순서가 바뀌었는데, 오히려 브랜드 로고/브랜드명으로 먼저 찾는 사용자층엔 불편하다는 피드백 다수. AMG처럼 브랜드 계열 필터 세분화 요청도 존재. → **필터 첫 화면에 뭘 먼저 보여줄지는 논쟁적인 지점**이라는 시사점.
- 출처: car.encar.com/list/car, YouTube 자동자막(`hD_4UwBrBqg`), fem.encar.com/company/encar-news/630, joongangenews.com, heraldcorp.com, mwm.ai.

### 4. 매물 상세
- 실제 상세페이지(fem.encar.com/cars/detail) 텍스트 추출로 확인된 **정보 구성 순서**:
  1. 헤더: 모델명 + 트림명, `연식 / 주행거리 / 연료 / 차량번호`
  2. 진단 배지(`엔카진단++` 등) + "엔카가 아래 사항을 모두 진단 하였습니다" 배너
  3. **옵션 정보** — 아이콘형 리스트(선루프/헤드램프LED/주차감지센서/후방카메라/자동에어컨/스마트키/내비게이션/열선시트/통풍시트/가죽시트 등 있음/없음 표기)
  4. **"이 차를 추천하는 이유"** — AI 또는 딜러가 작성한 것으로 보이는 한 줄 추천 카피 + 셀링포인트 사진 1장
  5. **차량 상태(진단 상세)** — 프레임 진단(정상), 외부패널 진단(정상), 차량키 개수, 틴팅 여부, 타이어트레드 잔량(앞/뒤 mm 단위)
  6. **진단 등급 설명** ("엔카진단 우수 등급을 확인했어요" 등 등급별 설명 문구) + 하부 사진
  7. **성능점검 상세** — 제시번호, 69개 검수항목, 교환/판금/부식 여부, 내차피해/타차가해 이력
  8. **보증 현황** — 제조사보증, 엔카보증(성능보험 30일/2,000km, 엔카성능보증 90일/5,000km)
  9. **엔카금융** — 대출 금리 조회 유도 카드(최저 3.9%~)
  10. **판매자 정보** — 판매중/판매완료 상태, 주소(구 단위), 종사원증번호
  11. **구매 가이드**(중고차 사기 주의사항 안내 텍스트 블록 — 신뢰 구축용 콘텐츠)
  12. **알림등록** — "비슷한 조건의 동일한 모델이 신규매물로 등록되면 알려드릴까요?" (유사매물 알림 구독)
- **엔카진단 3단계 등급 체계**(공식): `엔카진단`(무사고+외부패널 확인) → `엔카진단+`(+차량관리상태·타이어·옵션 작동 리포트) → `엔카진단++`(+내외부/하부 사진 제공, 엔카믿고 구매시 보증기간 연장). AI로 옵션/등급을 자동 분석하는 "AI 진단"도 진단+/++ 등급에 한해 적용.
- **CTA 버튼류**(문의/전화/찜/비교)는 마크다운 텍스트 추출에서는 아이콘 버튼이라 텍스트로 잡히지 않아 정확한 배치는 **확인 불가**. 다만 판매자 전화는 "050 안심번호"(통화 녹음)를 쓴다는 언급이 검색 결과에 있었음(가이드 문서, 출처: dc_carsearchlist 관련 안내 텍스트 — "판매자의 050 전화번호로 꼭 문의하세요! 통화 내용이 녹음되어 안전하게 거래").
- 출처: fem.encar.com/cars/detail/41253089 (Jina 전문 추출), car.encar.com/diagnosis/intro.

### 5. 랜딩/홈
- PC 홈(encar.com) 최상단 GNB(글로벌 내비게이션) 구조: **내차사기**(국산/수입/전기·친환경/화물·특장·버스/엔카믿고 Beta) — **내차팔기**(비교견적/직거래) — **이차어때**(자체 콘텐츠) — **시세** — **미디어** — **이벤트**. 우측: 딜러 셀프등록 / 로그인 / 채팅 / 전체메뉴.
- 홈 페이지 본문(히어로/추천 섹션 상세)은 자바스크립트 렌더링이라 텍스트 추출로는 확인 불가. 다만 **모바일 앱 홈 화면 개편(2024, "엔카믿고" 리브랜딩과 동시 진행) 공식 발표문**에서 홈 구성 요소가 명시됨:
  - "고객 검색 및 조회 차량 기반 **AI 추천 차량**"
  - "나와 관심 차량이 비슷한 고객의 선호 차량" (협업 필터링형 추천)
  - **"가장 많이 본 차량"** — 지역·연령·성별 기반 베스트 모델 분석
  - **"인기 상승 TOP 모델"** — 트렌드 노출 섹션
  - → 즉 모바일 홈은 "탐색 진입점" 성격의 그리드/필터가 아니라 **개인화 추천 위젯 중심**으로 구성됨.
- **"이차어때"**(mocha.do) — 별도 콘텐츠 서브 서비스로, 차량별 "장점/단점/평점"을 짧은 카피로 정리한 리뷰 카드 그리드. 예: "현대 더 뉴 캐스퍼 — 장점: 앙증맞은 막둥이 SUV / 단점: 능글맞은 할아버지 미션 / 평점 3.5". 필터(차종/정렬순서: 종합평가·디자인·성능승차감·공간·연비·안전·결함·최신순)와 무한스크롤형 "더보기"가 있음. 톤이 유머러스·솔직한 편.
- **최근 본 차량 / 찜한 차량 위젯화**: PC 개편에서 우측 사이드바에 있던 두 기능을 "위젯" 형태로 개편, 위젯 클릭 시 리스트 간 "비교하기" 기능 진입 가능.

### 6. 반응형
- **PC와 모바일 간 "정보 구조·시각 일관성" 확보가 2025-06 개편의 명시적 목표**(공식 보도자료 다수 재확인). PC는 모바일과 동일한 카드 정보 위계를 쓰되, 넓은 화면을 활용해 리스트 카드 크기를 키우고 한 화면에 더 많은 정보를 노출.
- PC 전용 추가 정보: 성능기록부 공개 여부 뱃지. 모바일 전용 차이점은 명시적으로 찾지 못함(대체로 모바일 기준으로 PC를 맞추는 방향).
- 이미지 비율은 5:3으로 디바이스 무관 통일(위 2번 참고) — **반응형 그리드는 이미지 비율 고정 + 카드 폭만 가변**하는 방식으로 추정됨(직접 그리드 열 수 확인은 불가, "화면 영역 확대"라는 표현만 확인).
- 출처: 매일경제, 이데일리, gpkorea.com, newstap.co.kr (동일 보도자료 기반 복수 매체).

## 우리 프로젝트 적용 시사점

1. **찜/비교 버튼은 사진 밖으로 빼라** — 엔카가 2025년에 직접 검증한 UX 개선 이유가 "차량을 가리지 않기 위함"이다. 우리 매물 카드도 하트(찜) 아이콘을 사진 위 오버레이로 얹기보다, 카드 하단 정보 영역이나 사진 옆 별도 영역에 배치하는 걸 검토할 것.

2. **대표 이미지 비율을 5:3 하나로 통일** — 엔카는 리스트/상세/마이페이지 등 전 화면에서 이미지 비율을 5:3으로 못박아 "디바이스별 시각 차이"를 없앴다. 우리도 매물 이미지 계약(스키마)에 고정 비율을 하나 정해두면(예: 4:3 또는 3:2), 카드·상세·썸네일 어디서든 크롭 로직을 재사용할 수 있어 이미지 확장 작업이 단순해진다. 업로드 UI에도 동일 비율 크롭 가이드를 넣는 게 좋다.

3. **카드 정보 위계 = 차량명 → 연식 → 주행거리 → 연료 → 지역 → 가격**이 업계 표준. 지금 매물 카드에 이 6개 항목이 이 순서로 다 있는지, 뱃지(무사고/인증 등)는 사진을 가리지 않는 위치에 있는지 점검할 근거가 됐다.

4. **AI 검색은 "전체 매물"이 아니라 "특정 조건 매물군"에 한정해도 된다** — 엔카조차 AI 챗봇/자연어 큐레이션을 신뢰 인증된 "엔카믿고" 매물에만 우선 적용하고 있다. 우리 프로젝트의 AI 검색 기능 범위를 전체 매물로 무리하게 넓히기보다, 신뢰할 수 있는 데이터가 확실한 하위셋부터 단계적으로 적용하는 접근이 검증된 선례로 뒷받침된다.

5. **필터 첫 화면 구성은 신중하게** — 엔카가 최근 "차종 우선"에서 "주행거리/가격/거리 우선"으로 필터 순서를 바꾸자 실사용자 반발이 상당했다(리뷰에 다수 언급). 필터 UI를 개편할 때는 "초보자에게 쉬운 순서"와 "숙련 사용자(브랜드/모델부터 찾는 사람)가 원하는 순서"가 충돌할 수 있다는 점을 미리 감안하고, 가능하면 둘 다 빠르게 접근 가능한 구조(칩 형태의 브랜드 바로가기 + 조건 필터 병행)를 고려할 것.

6. **모바일 홈 = 그리드 나열이 아니라 "개인화 추천 위젯" 중심**이 최신 트렌드다. 엔카는 "최근 본 차량/찜한 차량"을 위젯화하고, "가장 많이 본 차량", "인기 상승 TOP 모델" 같은 트렌드성 섹션을 홈에 배치한다. 우리 랜딩(홈) 개편 시, 단순 "전체 매물 그리드"보다 "최근 본 매물" 위젯이나 "인기 매물" 섹션을 작게라도 넣으면 재방문 사용자 경험이 훨씬 좋아질 근거가 된다(단, 데모 규모를 고려해 과설계는 피할 것 — CLAUDE.md A2 단순함 우선 원칙 참고).

7. (참고, 반영은 선택) **매물 상세 정보 순서**는 "헤더 스펙 → 진단/신뢰 배지 → 옵션 → 한줄 추천 카피 → 상태 상세 → 성능점검 → 보증 → 금융 → 판매자정보 → 안전가이드"로, 신뢰 요소를 상단에 가깝게 배치하고 금융/영업성 CTA는 중간에, 판매자 연락처는 하단에 두는 구조다. 우리 상세페이지도 "신뢰 뱃지(무사고 등)"를 스펙 바로 다음에 배치하는 게 업계 관행에 부합한다.
