# UI/UX 레퍼런스 조사 — KB차차차 (kbchachacha.com)

- **대상**: KB차차차 (KB캐피탈 운영, 국내 중고차 매물 대수 1위 표방 플랫폼). 금융·시세 정보 연계가 특징.
- **조사일**: 2026-07-11
- **조사 방법**: agent-reach 스킬 사용 — exa_search(웹검색), web(Jina Reader로 실제 페이지 텍스트 추출), video(yt-dlp로 유튜브 자막). 스크린샷 캡처는 불가하여 전부 텍스트로 기술.
- **한계**: 매물 목록/상세 페이지는 실제 URL을 Jina Reader로 열어 텍스트를 그대로 추출했으나(실측), 필터 패널·찜/비교 버튼 등 일부 UI는 JavaScript로 렌더링되어 텍스트 추출에 잡히지 않았다. 모바일 앱(안드로이드/iOS) 자체는 설치·조작이 불가능해 **앱스토어 리뷰·리뷰어의 화면 설명·광고 평론 기사**를 통한 간접 확인이다. 이런 항목은 각각 "확인 불가" 또는 "간접 확인(출처 명시)"로 표기했다.

## ① 매물 카드 (Listing Card)

**웹(데스크톱, www.kbchachacha.com/public/search/main.kbc — 실측)**
- 카드 한 개당 대표사진이 아니라 **사진 4장이 가로로 나란히 동시 노출**되는 "포토 스트립" 형태. 각 썸네일이 개별적으로 상세페이지 링크를 가짐(호버 시 각 사진마다 클릭 가능).
- 사진 위/옆에 "KB스타픽" 라벨 배지, 동영상이 있는 매물은 노란색 **PLAY 배지** 오버레이, "진단"·"홈배송" 텍스트 배지가 함께 붙음.
- 카드 하단 정보 위계(위→아래): ① 홍보 카피 한 줄(예: "★1인신조,300마력UP,B6휠★") → ② **차명(볼드, 모델명 풀네임)** → ③ 스펙 한 줄을 구분자 없이 붙여쓰기: "21/04식(21년형)35,126km 가솔린 경기" (연식·주행거리·연료·지역이 한 줄에 압축) → ④ 시트색상 → ⑤ **가격(가장 큰 볼드체, 숫자만 강조 "4,350"+작은 글씨 "만원")**.
- 무사고/비흡연 등은 별도 아이콘 뱃지가 아니라 **홍보 카피 텍스트에 섞여서** 노출(예: "무사고 · 비흡연 · 24상담", "인증 홈배송"은 이미지 위 오버레이 라벨).
- 찜·비교 버튼: 텍스트 추출로 명시적 확인 **불가**(하트 아이콘 등은 JS 렌더링). 상세페이지에서는 조회수로 추정되는 숫자("56,892 62 공유")가 있어 찜 카운트가 어딘가에 존재할 가능성은 있으나 라벨은 미확인.

**모바일 웹(m.kbchachacha.com/public/web/search/list.kbc — 실측)**
- 데스크톱과 달리 사진 4장을 동시 나열하지 않고 **스와이프 캐러셀 + "1/4" 카운터**로 표시(모바일 화면 폭에 맞춰 사진 하나씩 넘겨봄).
- 정보 위계는 데스크톱과 동일(홍보카피 → 차명 → 스펙 한줄 → 시트색상 → 가격 볼드).

**모바일 앱** — 앱스토어 리뷰 화면 설명(간접 확인, 출처: MWM.ai 앱 리뷰 요약)에 따르면 "KB스타픽" 영역은 **"딜러가 엄선한 스타급 중고차"를 한 줄 스펙 + 무사고 여부 시각 표시**로 보여준다는 설명이 있어 웹과 유사한 카드 패턴으로 추정됨. 실제 앱 카드 레이아웃 상세는 확인 불가.

## ② 이미지 처리

- **대표사진 다중 노출**: 리스트 카드 단계에서부터 이미 여러 장(데스크톱 4장 동시/모바일 캐러셀)을 보여줘 클릭 전에도 신뢰도를 준다.
- **상세페이지 이미지 구조**(carSeq=27923307 실측): 상단 대형 히어로 이미지 1장 + 그 아래 **가로 스크롤 썸네일 스트립**(해당 매물은 20장). 다음/이전(Prev/Next) 내비게이션 존재.
- **동영상 클립 포함**: 사진 갤러리 중간에 "영상으로 생생하게 PLAY! 엔진룸 소리까지 확인해보세요!" 항목이 삽입되어, 사진 사이에 **짧은 영상(엔진음 포함)**을 끼워 넣는 패턴(노란 PLAY 로고 배지로 식별).
- **신뢰 배지 오버레이**: 사진 몇 장 위에 "KB차차차 진단보증 차량입니다", "실차주가 직접 판매하는 차량입니다", "헛걸음보상서비스 대상 차량입니다" 같은 **캡션형 신뢰 문구가 사진 자체에 오버레이**되어 순서대로 노출됨(사진을 신뢰 근거 전달 수단으로도 활용).
- **사진 없을 때 플레이스홀더**: `https://www.kbchachacha.com/images/common/noimage.jpg` 확인됨(최근 본 차량 등에서 사용되는 회색 no-image 아이콘).
- **360도 뷰**: 이번 조사 범위에서 언급/확인되지 않음 → **확인 불가**.

## ③ 목록/필터·검색

- **정렬 옵션**(실측, 드롭다운으로 추정): 최신 등록순 / 낮은 가격순 / 높은 가격순 / 최근 연식순 / 오래된 연식순 / 짧은 주행거리순 — 총 6종.
- **검색바**: 상단에 "차량검색 **169,878 대**의 차량이 있습니다" 형태로 **현재 전체 매물 수를 실시간 카운트로 노출**하며 검색 인풋이 함께 배치됨. 홈 화면에는 별도로 "모델 검색" 섹션(전체등록 매물 수 재노출)도 존재.
- **AI/자연어 검색은 없음**. "AI 시세"라는 기능이 있지만 이는 자연어 질의가 아니라 **차량번호를 입력하면 3초 만에 시세를 산출**해주는 시세조회 기능(우리 프로젝트의 자연어 AI 검색과는 성격이 다름).
- **필터 패널(브랜드/차종/가격대 등) 구체적 UI**: JS 렌더링이라 텍스트 추출로 직접 확인 **불가**. 다만 간접 자료로: (1) 유튜브 앱 워크스루 자막에서 "주행거리, 연식, 색상 등 조건을 설정해서 검색"한다는 언급, (2) 앱스토어 리뷰(원문 인용) — *"연식 검색 항목에 월수를 입력할 수 없다", "가격 검색 범위가 너무 넓다. 1,500만 원 이후 구간부터는 500만 원 단위로만 설정 가능해 중간 금액대(예: 1,800만 원)를 찾을 때 불필요한 정보까지 함께 검색된다"* — 즉 **가격·연식 필터의 세분화 부족이 실사용자 불만**으로 반복 지적됨. (3) 3자 비교 아티클(hello109.com)도 "검색 필터 세분화 부족"을 KB차차차의 약점으로 명시.
- **금융 연계형 필터**: 홈 화면에 "예산으로 차량 찾기" 위젯 — 선납금·할부개월·월 납입액을 입력하면 그 예산에 맞는 차량으로 진입(단순 가격 필터가 아니라 **할부 시뮬레이션 기반 검색 진입점**)이라는 점이 독특함.

## ④ 매물 상세

**데스크톱(carSeq=27923307, 실측) — 화면 위→아래 순서**
1. 이미지 히어로(대) + 썸네일 스트립(20장, 신뢰배지 오버레이 포함) + Prev/Next
2. 차명 + **가격(대형 볼드) + "중고차시세" 링크**
3. 핵심 스펙 한 줄(연식/주행거리/연료/지역)
4. 조회수·찜수 추정 카운터 + 공유 버튼
5. 빠른 링크 4개 칩: "성능점검 상세보기" / "보험이력 0건" / "주행거리 매우짧음" / "기본정보 상세보기"

11. **차량정보 테이블**: 차량정보/연식/주행거리/연료/변속기/연비/차종/배기량/색상/시트색상/압류/저당/세금미납/제시번호

- **모바일 상세 페이지**: 이번 조사에서 데스크톱과 동일한 carSeq로 `m.kbchachacha.com/public/car/detail.kbc`를 열었으나 **404(경로 불일치)** — 홈 화면 링크를 보면 모바일 상세 URL은 `m.kbchachacha.com/public/web/car/detail.kbc?carSeq=...`로 `/web/` 세그먼트가 추가되는 구조. 시간 제약상 재시도하지 않고 **모바일 상세 화면 구성은 확인 불가**로 남김(데스크톱과 정보 항목 자체는 동일할 가능성이 높으나 배치 검증은 못 함).

- 그 아래: 같은 7개 서비스를 **텍스트 탭 메뉴**로 재노출(스타픽/홈배송/진단중고차/인증중고차/내차고/내차팔기/AI 시세).

## ⑥ 반응형

- **데스크톱↔모바일이 반응형(같은 코드/브레이크포인트)이 아니라 별도 도메인(m-dot) 구조**: `www.kbchachacha.com`(PC) vs `m.kbchachacha.com`(모바일 웹)로 완전히 분리된 사이트. 우리 프로젝트(Next.js 단일 코드베이스 반응형)와는 아키텍처가 다르다는 점을 유의.
- 레이아웃 전환 방식(레이아웃 자체 비교):
  - **내비게이션**: 데스크톱은 상단 풀 메뉴바(검색/내차사기/내차팔기/혜택/안내 5개 대메뉴 + 드롭다운 서브메뉴) → 모바일은 검색바 + "전체메뉴" 버튼(햄버거)으로 축소, 대메뉴는 아이콘 그리드로 재배치.
  - **매물 카드 이미지**: 데스크톱은 사진 여러 장을 **동시 나열**(정적 그리드형 프리뷰) → 모바일은 **스와이프 캐러셀**(1/4 카운터)로 전환. 같은 "여러 장 프리뷰"라는 목적을 화면 폭에 맞게 다른 인터랙션으로 구현한 점이 특징적.
  - **그리드 열 수**: 데스크톱 매물 리스트가 정확히 몇 열 그리드인지는 텍스트 추출로 판별 **불가**(이미지 나열 순서만 확인됨, 실제 CSS grid-column 값은 미확인).
- **정렬/필터 문구는 두 화면에서 동일**(최신 등록순 등 6종 정렬 옵션이 데스크톱·모바일 웹 모두에서 동일 텍스트로 확인됨) — 즉 기능은 동일하게 유지하되 배치/인터랙션만 화면 크기에 맞춰 바꾸는 전략.

## 우리 프로젝트 적용 시사점

1. **매물 카드 이미지 계약을 "복수 장" 전제로 확장**: KB차차차는 리스트 카드 단계에서부터 대표사진 1장이 아니라 3~4장을 프리뷰(데스크톱은 동시 나열, 모바일은 스와이프)로 보여준다. 우리 매물 카드도 대표사진 1장 고정이 아니라 최소 3장 이상을 프리뷰할 수 있는 이미지 계약(배열)으로 설계하면 신뢰도 체감이 올라간다. 단, 카드가 많아지므로 성능(lazy-load, 썸네일 리사이즈) 고려 필요.
2. **신뢰 배지를 "사진 위 오버레이"로 노출**: 무사고/실차주/진단 등 신뢰 정보를 순수 텍스트 뱃지가 아니라 사진 자체에 캡션으로 얹는 패턴이 효과적으로 보인다. 우리 데모에도 매물 사진 위에 "무사고" 등 짧은 신뢰 라벨을 오버레이하는 옵션을 고려할 만하다(과설계는 지양, 있으면 좋은 정도).
3. **사진 없을 때의 명시적 fallback UI 필요**: KB차차차는 `noimage.jpg` 같은 플레이스홀더를 명확히 두고 있다. 우리 이미지 계약에도 "사진 없음" 상태의 플레이스홀더(아이콘+문구)를 정의해야 한다 — 이는 이번 브라운필드 증분(이미지 추가)에서 반드시 짚어야 할 엣지케이스.
4. **정보 위계: 가격을 압도적으로 크게, 스펙은 구분자 없이 한 줄로 압축**: "가격(대형 볼드) → 차명 → 연식/주행거리/연료/지역을 붙여쓴 한 줄" 순서가 일관되게 반복된다. 우리 카드/상세의 타이포그래피 대비(가격 vs 나머지 텍스트)를 더 극단적으로 줘도 무방하다는 참고가 된다.
5. **랜딩 구성 뼈대(축소 적용)**: "검색 진입 → 서비스 아이콘 메뉴 → 추천매물 캐러셀 → 카테고리 그리드 → 콘텐츠 → 푸터"라는 패턴은 참고할 만하나, 우리는 데모/단일 서비스이므로 금융·보험 연계 섹션 등은 제외하고 **"AI 자연어 검색 진입 + 추천 매물 캐러셀 + (있다면) 카테고리 그리드"** 정도로 단순화해서 가져가는 게 A2(단순함 우선) 원칙에 맞다.
6. **반응형은 직접 재사용 어려움, 인터랙션 전환 아이디어만 참고**: KB차차차는 m-dot(별도 모바일 도메인) 구조라 우리(Next.js 단일 반응형)와 아키텍처가 다르다. 다만 "데스크톱은 카드 내 여러 사진을 정적으로 나열, 모바일은 같은 정보를 스와이프 캐러셀로" 바꾸는 **화면폭별 인터랙션 전환 아이디어**는 그대로 참고 가능하다.
7. **필터 세분화는 KB차차차의 약점으로 반복 지적됨(가격 구간이 넓음)** — 우리가 가격 필터를 만들 때는 이 실패 사례를 반면교사 삼아 더 촘촘한 구간(또는 자유 입력)을 제공하면 차별점이 될 수 있다.
