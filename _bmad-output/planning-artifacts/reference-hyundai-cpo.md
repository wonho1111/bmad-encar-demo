# UI/UX 레퍼런스 조사 — 현대 인증중고차 (Hyundai Certified / 현대 CPO)

- **대상**: 현대/제네시스 인증중고차 (certified.hyundai.com) — 2023년 출범한 현대자동차 제조사 직영 인증중고차(Certified Pre-Owned) 플랫폼. 현대 브랜드관 + 제네시스 브랜드관을 하나의 웹사이트/앱에서 함께 운영.
- **조사일**: 2026-07-11
- **조사 방법**: agent-reach 스킬 (exa_search + Jina Reader 웹 본문 추출 + YouTube 자막). 사이트가 로그인 없이도 매물 검색/상세 리스트는 열람 가능해, 실제 라이브 페이지(메인·검색/필터·제네시스 전용관·오감만족 소개·App Store 설명·공식 보도자료)를 직접 스크래핑해 확인. 개별 매물 **상세 페이지**는 JS 라우팅(SPA)이라 Jina Reader로 직접 URL 접근이 안 돼(3회 시도 후 확인 불가로 표기, 네비게이션 껍데기만 반환됨) — 대신 앱스토어 설명·공식 보도자료·이용후기 블로그로 상세 페이지 구성요소를 교차 확인함.

## ① 웹(데스크톱)

### 1. 매물 카드 (목록 페이지, `/p/search/vehicle`)
- **대표사진**: 카드 상단에 가로형(landscape) 사진 1장, 실제 크기 `380x253`(약 3:2 비율), 원본은 `2304x1536`을 크롭. 즉 **실사진**(일러스트 아님) 위주.
- **정보 위계** (사진 아래로 순서대로):
  1. 연식+모델명+트림 (예: "2024 싼타페(MX5) 하이브리드 AWD 6인승 캘리그래피") — 가장 굵고 큰 텍스트
  2. 서브정보 한 줄: `등록연월 · 주행거리 · 차량번호 · 지역` (예: "23년 10월 14,385km 109부7518 양산")
  3. **가격**: 취소선 처리된 원가 + 강조된 실판매가(굵고 크게, "만원" 단위) + "○○○만원 할인" 배지
  4. 할부 정보: "할부 월 53만원" (보조 텍스트, 산정 기준 각주 포함 — 선수금30%/금리4.8%/60개월)
  5. 하단 배지: "추가할인", "가격 Slim", "AI PICK", "신차급" 등 키워드형 뱃지 (복수 부착 가능)
- **찜하기·비교하기 버튼**: 사진 위(오버레이) 좌상단 또는 카드 상단에 아이콘 버튼로 배치, 리스트 항목마다 "찜하기 / 비교하기" 두 개가 사진과 함께 묶여 나옴 — 카드 액션이 사진 영역에 오버레이되는 구조.
- **인증/무사고 뱃지**: 카드 단위에는 별도 "무사고" 뱃지가 노출되지 않음(대신 서비스 전체가 "인증중고차"라는 브랜드 신뢰 프레임을 씀). 대신 프로모션성 키워드 뱃지(가격할인/AI PICK 등)가 카드 하단에 붙음.

### 2. 목록/필터·검색 (`/p/search/vehicle`)
- **좌측 사이드바 필터**(데스크톱): 차종(승용/SUV/승합/EV), 브랜드/모델(현대 893대 · 제네시스 404대처럼 브랜드별 매물수 병기), 가격(슬라이더+프리셋 버튼 1000만~8000만원 이상), 주행거리(슬라이더), 연식(2017~2026 범위), 연료(가솔린/디젤/LPG/하이브리드/전기), **색상 계열**(스와치 12색), **옵션**(체크박스, "+옵션 더보기"로 확장), 인증중고차 센터(지역: 양산/용인/군산), **키워드**(가격할인/기간한정혜택/오토할부금리2%할인/무료배송/가격Slim/신차급/AI PICK 등 프로모션 태그 필터).
- 상단 검색: "직접 입력 / 모델로 찾기 / 예산으로 찾기" 3개 탭형 검색 진입점, 인기 검색어 Top10(날짜 기준 표기), 예산 프리셋 칩("2천만원 이하" 등).
- **AI 관련**: 자연어 검색은 없지만 "AI 큐레이터's PICK", "Hi-LAB이 오늘의 시세보다 저렴한 차량을 추천"처럼 **AI 시세 기반 추천/큐레이션**이 홈·목록 곳곳에 노출. 순수 자연어 질의형 AI 검색은 확인 안 됨.
- 정렬: "인기순" 확인(그 외 정렬 옵션은 드롭다운으로 추정되나 페이지 텍스트 추출로는 목록만 확인 안 됨 — 확인 불가로 남김).
- 총 매물 수를 상단에 카운터로 노출("총 1,297대", "판매차량1,297 / 판매예정차량369").
- "구매 가능한 차량만 보기" 토글 존재(즉 매매 진행 중/예약 상태 차량도 목록에 섞여 노출되고 필터로 감출 수 있음).
- "더보기" 버튼으로 페이지네이션 없이 리스트를 이어서 로드하는 방식.

### 3. 이미지 처리
- 목록 카드: 가로형 1장 (2304x1536 원본을 380x253으로 크롭/리사이즈, webp 변환 — CDN에서 `dims/crop/.../resize/.../optimize/format/webp` 체이닝 URL 사용).
- 제네시스 전용관 카드: **세로형(portrait) 크롭**(1778x2032 원본 → 280x320) — 럭셔리 브랜드 존은 카드 사진 비율 자체를 다르게 써서 "쇼룸 화보"에 가까운 느낌을 줌. 일반(현대 혼합) 리스트는 가로형, 제네시스관은 세로형이라는 **표면별 사진 비율 차등화**가 확인됨.
- **상세 페이지 이미지**(App Store 설명 + 공식 보도자료 기준, 직접 URL 접근은 SPA라 불가):
  - **360도 VR** 콘텐츠(차량 내외부)
  - 차량 하부 사진(누유/누수 점검용, 실사)
  - 타이어 마모도 진단 화면 + 제동 검사 보고서
  - 엔진음 청취(엔진점검 AI가 녹음한 실제 엔진 소리)
  - 실내 공기질 쾌적도 수치화 화면
  - **시트 6배줌 초고화질 이미지**
  - 모바일 앱에는 "가상전시장(Virtual Showroom)" 기능도 있음(오프라인 전시장 체험을 온라인으로 재현)
  - 이 모든 게 "오감만족(五感滿足)" 콘텐츠로 브랜딩되어 매물 상세의 핵심 신뢰 장치로 쓰임.
- **272개(현대) / 287개(제네시스) 정밀진단 항목**을 검사 후 "품질인증"을 부여하고, 점검리포트가 발행되어 앱/웹에서 열람 가능(앱스토어 설명·보도자료 공통 확인).
- 사진 없을 때 플레이스홀더: "판매예정차량(Coming Soon)" 섹션에 "대표 이미지로, 실제 모델/등급과 상이할 수 있습니다" 문구와 함께 모델 대표 이미지(실제 개체 사진이 아직 없는 매물용 placeholder 패턴)를 사용 — 즉 **실사진이 없을 때는 "판매예정" 상태로 분리하고 모델 대표컷을 임시로 채움**.

### 4. 매물 상세 (직접 URL 접근 불가 — App Store 설명/보도자료/이용후기 기반 재구성, "확인 불가"로 표기한 부분은 명시)
- 주요 CTA: 상세보기 진입 후 견적확인 → 계약 → 결제(카드결제 최초 도입 + 현금 + 할부금융/제휴카드) → 배송(집 앞 배송)까지 **원스톱 온라인 프로세스**. "나의 할부 조건 조회하기"로 개인화된 할부 견적 제공.
- 인증·보증 정보: "워런티 플러스"(연장보증), 5년/10만km 이내 무사고 인증 기준, 구매 후 일정 기간(리뷰 기준 통상 3일/72시간) 내 단순변심 반품 가능한 "묻지마 환불" 제도가 상세 페이지 신뢰 요소로 강조됨.
- 점검리포트(272/287항목) 열람 가능.
- 상세 URL 구조는 `goodsDeatil('상품코드')`라는 JS 함수로 라우팅되는 클라이언트사이드 SPA라 URL 직접 크롤링 불가 — **확인 불가(로그인/JS 실행 필요)**.

### 5. 랜딩/홈 (`/p/display/main`)
- **히어로**: 상단 대형 프로모션 배너 캐러셀(1180x360, 6장 슬라이드) — 신차 할인/카드 캐시백/리스 이벤트 등.
- 그 아래 **통합 검색 위젯**(직접입력/모델로찾기/예산으로찾기 탭)이 히어로 바로 밑에 배치되어 "검색이 첫 화면의 주인공" 구조.
- 예산 프리셋 칩(2천/3천/4천/5천만원 이하) — 초보자가 가격대로 바로 진입할 수 있게 유도.
- "신차급 중고차 / 트레이드인(내차팔기+신차구매 연계) / AI 큐레이터's PICK / Slim한 기획전" 등 **테마 큐레이션 섹션**이 카드 캐러셀로 여러 개 나열(가로 스크롤, "1/3", "1/4" 페이지 인디케이터 병기).
- "매일 오후 5시! 새로운 현대인증중고차를 만날 수 있는 시간" — **정시 신규 매물 오픈**을 랜딩에서 카운트다운/공지처럼 노출(재방문 유도 장치).
- 모델별(그랜저/G80/GV70 등) **Hi-LAB 시세 위젯**: "오늘의 그랜저(GN7) 국내 평균 시세는 2,682~4,721만원" + 산점도 차트(주행거리 x축, 가격 y축)로 인증중고차 매물이 시세 대비 어디 위치하는지 시각화 + "지금 준비된 그랜저는 총 163대" — **AI/데이터 신뢰 장치를 랜딩에 바로 노출**.
- "Coming Soon 판매예정차량" 섹션: 입고 전 차량도 미리 보여주고 "입고알림 신청" CTA.
- 하단: 이용후기(사진후기/베스트순/최신순), 브랜드 메시지("Made by us, Cared by us"), 금융 혜택 안내.
- **오늘 그만 보기** 닫기 옵션이 있는 상단 공지 배너(재노출 억제 패턴).

### 7. 전반 디자인 언어
- 텍스트만으로는 정확한 HEX값 확인 불가하나, 관련 디자인 스튜디오(Double D, 현대 UX Studio 리뉴얼 프로젝트) 자료 기준 **현대 모빌리티 브랜드 공통 톤**: 블랙·화이트 모노톤을 메인으로, 오렌지·블루를 포인트 컬러로 쓰는 방향성(단, 이는 별도 오프라인 UX Studio 브랜딩 사례이며 CPO 이커머스 사이트 자체의 정확한 배색과 100% 동일하다고 단정할 근거는 부족 — **참고용으로만 표기**).
- 사이트 자체는 절제된 텍스트/숫자 중심 레이아웃(가격·연식·주행거리를 숫자로 크게 강조), 뱃지형 키워드 태그를 다용, "AI PICK/Hi-LAB" 같은 데이터·AI 신뢰 요소를 시각적으로 반복 노출해 "제조사가 데이터로 보증한다"는 톤을 강조.
- 브랜드 슬로건: "Made by us, Cared by us."

## ② 모바일 웹

- 사이트가 `p/display/main`(PC 경로)과 `m/display/main`(모바일 경로)을 **별도 URL prefix**로 나눠 서빙 — 즉 단순 반응형이 아니라 **UA 기반 적응형(adaptive) + 별도 모바일 템플릿** 구조로 추정(URL 자체가 분리돼 있음). 단, Jina Reader로 추출한 텍스트 콘텐츠/내비게이션 구조는 PC·모바일 두 URL이 거의 동일했음 — 실제 시각적 레이아웃(그리드 열 수, 카드 크기 변화)은 CSS 렌더링을 봐야 하는데 Jina는 CSS를 걷어내므로 **레이아웃 그리드 차이는 확인 불가**로 남김.
- 모바일에서도 검색 진입점(직접입력/모델로찾기/예산으로찾기), 인기검색어, 예산 프리셋 칩 등 핵심 검색 UX는 동일하게 유지.
- 앱스토어 리뷰 중 "현대차·제네시스 통합 검색이 편리, UI/UX가 전보다 개선됨"이라는 언급으로 미루어 모바일(앱 포함) UX가 지속 개선 중임을 확인.

## ③ 모바일 앱

- **별도 앱이 존재함** ("현대/제네시스 인증중고차" — iOS App Store 확인, id1599229682, 무료, 44.2MB, 평점 4.1/5.0(124명), 개발사 HYUNDAI MOTOR COMPANY). ※ 조사 브리핑에 "제조사 인증중고차는 앱이 없을 수 있다"고 가정했으나 **이는 사실이 아니며, 실제로는 웹+앱 투트랙 운영**임.
- 앱 설명 핵심 기능:
  - **내차팔기**: AI 프라이싱 엔진(빅데이터 기반 옵션단위 시세), 방문평가사 현장실사, 차량번호 입력만으로 시세 조회(로그인/블루링크 연동 시), 시세조회→방문평가 신청→대금지급까지 앱 이탈 없이 논스톱.
  - **내차사기**: 사용자 분석·해시태그 기반 차량 큐레이션, 272/287개 항목 정밀진단, 오감만족 콘텐츠(타이어마모/실내공기질/엔진음), 다양한 결제방식(할부금융/제휴카드/현금).
  - **Hi-LAB**: 트렌드통계, 시세, 차량번호로 정상매물/정비이력/사고이력/성능점검이력 통합조회, 거래꿀팁 콘텐츠.

## 우리 프로젝트 적용 시사점

1. **카드 액션(찜/비교)을 사진 오버레이로 배치**: 현재 우리 매물 카드에 이미지가 추가되면, 찜하기 아이콘을 사진 좌상단이나 우상단에 오버레이로 얹는 패턴을 참고할 만함(정보 텍스트 영역을 침범하지 않고 사진 위에서 즉시 액션 가능).
2. **가격 표기 위계(취소선 원가 + 강조 실판매가 + 할인액 배지)**: 우리는 할인 개념이 없어도, "가격을 카드에서 가장 크고 굵게" 두고 그 아래 보조 정보(연식·주행거리·지역)를 한 줄로 압축하는 정보 위계는 그대로 채택 가치가 있음 — 초보 사용자가 스캔하기 쉬움.
3. **사진 없을 때 placeholder 전략**: "판매예정(Coming Soon)"처럼 사진이 준비 안 된 매물은 상태 자체를 분리하고 "대표 이미지로 실제와 다를 수 있음"을 명시하는 방식 — 우리도 이미지 계약 확장 시, 사진 없는 매물을 그냥 빈 박스로 두기보다 "사진 준비중"류 플레이스홀더 + 문구로 신뢰를 깎지 않는 처리를 고려.
4. **필터를 좌측 사이드바 + 색상 스와치 + 프리셋 칩 조합으로 구성**: 가격/주행거리는 슬라이더보다 "프리셋 버튼(2천만원 이하 등) + 직접입력" 두 방식을 같이 주는 게 초보자 친화적 — 우리 매물탐색 필터 UI 개편 시 참고 가능.
5. **랜딩에 "검색 위젯을 히어로 바로 아래" 배치 + 예산 프리셋 칩**: 첫 화면에서 바로 조건 검색으로 유도하는 구조는 우리 랜딩 개편(브라운필드 증분 중 "랜딩" 항목)에 그대로 적용할 수 있는 패턴. 다만 이벤트 배너 캐러셀처럼 화려한 프로모션은 우리 데모 규모에는 과함 — 검색 위젯 우선 배치만 채택 권장.
6. **인증 뱃지는 카드 단위가 아니라 "서비스 전체 브랜드"로 승부**: 현대 CPO는 개별 카드에 "무사고" 뱃지를 달기보다 사이트 전체가 "인증중고차"라는 프레임이라 카드에는 프로모션 키워드 뱃지(가격할인/AI PICK)만 붙임. 우리 서비스는 개별 판매자 직거래라 신뢰 보증 주체가 다르므로, 오히려 우리는 개별 매물 카드에 "무사고/판매자인증" 같은 뱃지를 직접 붙이는 쪽이 우리 맥락에 더 맞음 — 이 차이를 UX 설계 시 인지하고 참고.

**표면별 확인 여부 요약**: 웹(데스크톱)·모바일웹·앱 3가지 모두 존재 확인. 웹/모바일웹은 라이브 페이지 직접 스크래핑으로 확인, 앱은 App Store 메타데이터+공식 보도자료로 기능 확인(실제 화면 스크린샷은 텍스트 조사 특성상 확인 불가). 매물 상세 페이지의 정확한 레이아웃 순서는 SPA 라우팅 제약으로 직접 확인하지 못해 2차 자료로 재구성했음을 명시.
