# UI/UX 레퍼런스 조사 — K Car(케이카, kcar.com)

- **대상**: K Car (케이카 — kcar.com / m.kcar.com / market.kcar.com, 모바일앱 "K Car - 케이카 직영중고차")
- **조사일**: 2026-07-11
- **조사 방법**: agent-reach 스킬 (exa_search + web/Jina Reader + youtube yt-dlp). 스크린샷 불가로 전부 텍스트 추출 기반. 카드 이미지의 실제 시각 디자인(색감·레이아웃 여백 등)은 자동 추출 텍스트만으로 확인 불가한 부분이 있어 해당 항목은 명시적으로 "확인 불가"로 표기.
- **참고**: K Car는 크게 두 서비스가 공존한다 — ① **직영 중고차**(kcar.com/bc/search, 회사가 직접 매입·진단해 파는 B2C) ② **직거래(안심직거래)**(market.kcar.com, 개인 간 P2P 마켓 — 우리 프로젝트와 사업모델이 가장 유사). 아래 소견에서 어느 쪽 화면인지 구분해 표기했다.

## 1. 매물 카드

### 직영 중고차 카드 (kcar.com/bc/search) — 확인 불가(부분)
검색 목록 페이지(`https://www.kcar.com/bc/search/`)는 클라이언트 사이드 렌더링이라 Jina Reader로는 필터 UI와 정렬 옵션만 확인되고 실제 카드 HTML(가격·뱃지 등)은 비어 있는 상태로만 확인됨(`검색된 차량이 없습니다` — JS 미실행 탓). 다만 목록 상단에 **"리스트형버튼"**(그리드/리스트 뷰 전환 토글로 추정)이 있고, 정렬 옵션은 `기본정렬 / 남은시간 적은순 / 남은시간 많은순 / 최근 연식순 / 오래된 연식순 / 적은 주행거리순 / 많은 주행거리순 / 낮은 가격순 / 높은 가격순` 9종으로 확인됨. 출처: https://www.kcar.com/bc/search/

### 직거래(P2P) 카드 (market.kcar.com/main) — 확인됨
서버사이드 렌더링이라 실제 카드 텍스트 구조를 그대로 확인. 카드 1건 텍스트 예시:
> "현대 그랜저 HG HG240 모던 13년 11월 (14년형) 15.1만km 가솔린 광주 북구 400만원 **가격제안가능** `2` 1인 소유 비흡연 차량 엔진오일 정기 교체"

정보 위계(카드 안에서 위→아래로 추정되는 순서):
1. 대표사진(썸네일, 1장)
2. 제조사+모델명+트림(가장 큼직한 텍스트로 추정)
3. 연식(등록년월 + 괄호 안 "OO년형" 출시연도 병기) + 주행거리 + 연료
4. 지역(시/구 단위, 정확한 주소 아님 — 프라이버시 고려로 추정)
5. **가격**(굵게) + "가격제안가능" 배지(네고 가능 표시)
6. 숫자 배지(찜/관심 카운트로 추정, 값 0~13 확인됨 — 정확한 라벨 텍스트는 미확인)
7. 특징 태그 2~4개 (예: `1인 소유` `비흡연 차량` `엔진오일 정기 교체` `외관 상태 우수` `신품급 타이어` `실내 주차` `6개월 이내 소모품 교체` `빠른 거래 가능` `시승 가능` `완전 무사고` `리콜 완료` `제조사 보증 남음`)

홈 화면에는 두 개의 카드 섹션이 그리드로 나열됨: **"지금 뜨고 있는 차량이에요"**(인기순 추정)와 **"최근 등록된 차량이에요"**(최신순). 각 섹션 20건 내외 카드가 그리드 배치.
출처: https://market.kcar.com/main

**무사고/인증 뱃지**: 명시적 "무사고" 배지는 `완전 무사고` 텍스트 태그로 확인(직거래 카드). 직영 중고차 쪽은 별도 "브랜드 인증관"이라는 인증 매물 전용 섹션이 존재(`https://www.kcar.com/br/brandCarMain`) — 카드 자체에 인증 배지가 표시되는지는 확인 불가.

**찜·비교 버튼**: 검색/목록 페이지 필터 옵션에 "찜한 차량" 메뉴가 별도로 존재(로그인 필요)하고, 상세 페이지 하단에는 **"비교하기" / "공유하기"** 버튼이 확인됨(아래 4번 참고). 목록 카드 자체에 찜 하트 아이콘이 오버레이되는지는 텍스트 추출로는 확인 불가.

## 2. 이미지 처리

- **대표사진**: 직거래 카드는 1장 썸네일만 목록에 노출(JPG, `img.kcar.com/kcarmall/market/CC_CAR_IMG/...` 경로).
- **갤러리/스와이프, 사진 개수**: 상세 페이지는 JS 렌더링이라 실제 갤러리 이미지 개수·스와이프 동작은 텍스트 추출로 확인 불가. 다만 뉴스 기사(카매거진, 2022-07-28)에 따르면 **2022년 개편 이후 상세 페이지에 "K Car 진단 결과를 보여주는 그래픽"이 추가**됐고(외관 상태를 도식화), **2022년 초부터 "3D 라이브 뷰"** 기능을 도입해 차량 내∙외부를 360도 회전하며 볼 수 있다고 명시됨. 인용: "단순히 사진만 나열하던 과거 방식에서 진화해 구매희망자가 차량 내∙외부를 360도 회전해가며 차량 상태를 구석구석 살펴볼 수 있도록 하고 있다." 출처: https://www.carmgz.kr/news/articleView.html?idxno=5430
- 앱스토어 유저 후기(2021)에서도 상세 페이지 사진 구성을 언급: "마음에 드는 차량을 선택하면 차량 외부부터 시작해 내부까지 디테일한 부분들을 사진으로 확인할 수 있습니다. 꼭 챙겨 봐야하는 부분들을 디테일하게 사진으로 구성해두고 있어..." 출처: https://ittong.kr/4525
- **사진 없을 때 플레이스홀더**: 확인 불가(직거래 목록에서 확인한 카드는 전부 사진 보유 상태였음).
- **360도**: 직영 중고차만 해당하는 기능으로 보임("3D 라이브 뷰"). 직거래(개인 P2P) 매물에도 적용되는지는 확인 불가.

## 3. 목록/필터·검색

### 필터 (직영 중고차, 데스크톱 `www.kcar.com/bc/search/`)
왼쪽/상단 사이드 패널 형태로 다음 필터가 확인됨(순서):
- 차종(중복선택) — 승용 세그먼트(경차~대형·스포츠카)뿐 아니라 SUV/RV/승합/화물/버스까지 포함
- 국산·수입 제조사/모델(중복선택, 트리형)
- 연식(범위 슬라이더 추정, `~`)
- 주행거리(범위)
- 가격(범위)
- 차량색상(중복선택, 색상 칩 — 흰색/회색/검정/갈색/빨강/녹색/주황/파랑/노랑/보라/기타 11종, **색상 스와치 SVG 아이콘** 사용)
- 시트색상(동일 패턴, 투톤 믹스 포함 12종)
- 연료(중복, 전기·가솔린+전기·가솔린·디젤·LPG 등 12종 세분화 — 하이브리드/PHEV까지 조합형으로 표기)
- 변속기(오토/수동/세미오토/CVT/기타)
- 인승(4~15인승)
- 구동방식(전륜/후륜/4륜)
- "주요 키워드" 칩: `무료배송` `특가` `렌트` `옵션` `지역/직영점` `판매구분`
- 하단에 `초기화` + `N개 직영중고차 보기` CTA 버튼(적용 전 카운트를 실시간 미리보여주는 방식으로 추정)

### 모바일 웹 필터 (`m.kcar.com/bc/search`)
데스크톱과 필터 항목은 동일하지만 **전체화면 바텀시트/드릴다운 방식**으로 보임(뒤로가기 버튼이 각 필터 진입마다 노출). 색상·시트색상은 체크박스+SVG 스와치가 그대로 유지. 하단 고정 CTA는 `새로고침 / 적용` 버튼 페어, 최종적으로 `새로고침 차량 보기(10,085대)`처럼 **적용 전 매칭 건수를 버튼에 실시간 표기**. 출처: https://m.kcar.com/bc/search

### 검색바 / AI·자연어 검색
- 홈 화면에 큼직한 "어떤 차를 찾으세요?" 검색 인풋 + 인기 검색어 칩(`#카니발 (4세대)` `#G80 (RG3)` `#GV80` `#GV70` `#그랜저 IG` 등 모델명 해시태그)이 배치됨.
- 검색 인풋 아래 3가지 탐색 진입점이 병렬 제시: `원하는 차가 있어요` / `예산이 정해져 있어요` / `차종별로 찾고 있어요` — **의도(모델명 vs 예산 vs 바디타입) 기준으로 검색 방법을 분기**하는 UX 패턴.
- 뉴스 기사(2022-07-28)에 따르면 2022년 개편으로 **의미 기반 자연어 처리(NLP) 검색**, 연관 검색어 제안, 검색조건 저장, 인기 검색어, **모바일 앱 음성검색**을 도입했다고 명시. 인용: "기존 키워드를 직접 입력해 모델명으로만 가능했던 검색 방법에서 벗어나 의미기반 검색을 가능하게 하는 자연어 처리 검색과 연관 검색어 제안 기능, 검색조건 저장, 인기 검색어 등을 도입했다. 모바일 앱에서는 음성검색도 지원한다." 출처: https://www.carmgz.kr/news/articleView.html?idxno=5430 — 단, 자연어 검색이 현재도 살아있는 기능인지, UX가 정확히 어떻게 동작하는지는 실사용 확인 불가(2022년 시점 보도).
- Google Play 앱 설명에도 "음성인식 검색을 위한 마이크 권한"이 명시되어 앱에 음성검색이 실재함을 뒷받침. 출처: https://play.google.com/store/apps/details?hl=ko&id=com.encardirect.app

## 4. 매물 상세

상세 페이지(`www.kcar.com/bc/detail/carInfoDtl?i_sCarCd=...`, 직영 중고차)의 **섹션 순서**(정적 마크업 기준, 실제 데이터값은 JS 렌더링이라 0으로 표시되지만 구조는 확인됨):

1. 가격(대표, 큰 글자) + "할부 월 OO만원" 병기
2. 연식(년/월식) + 주행거리
3. **"K Car 차량평가사가 직접 진단한 차량이에요"** — 신뢰 배지형 카피(직영 진단 강조)
4. "편의와 안전을 위한 차량옵션" → 주요 옵션 리스트
5. "K Car가 직접 꼼꼼히 확인한 진단 결과" → 외부 패널 및 프레임 진단(그래픽 다이어그램으로 추정) → 주요 진단결과: `사고진단(진단통과)` + 카테고리별 이력 건수(`실내 및 외관` `타이어 및 휠` `소모품` `하체` — 각 "-건" 형태로 정상/이상 건수 표기) + "알림사항" 섹션(경고성 항목 별도 그룹)
6. "K Car가 찾은 차량 과거이력" → `내차 피해 없음` / `소유주 변경 총 O회` / `주의이력`
7. "K Car 차량평가사의 이유 있는 추천" → 담당 평가사 소속·사원증(사람 얼굴을 건 신뢰 장치)
8. "더 쉽고 간편하게 내차 사기" 배너 → 내차사기 홈서비스(온라인 주문/결제 → 배송 → 3일 시승 후 최종결정, 3일 내 환불 시 위약금 없음)
9. "직접 보고 구매하고 싶다면, 예약 후 직영점 방문" → 매장 주소·연락처·영업시간·점심시간, 예약 안내(토요일 전화예약만 등)
10. 제휴 카드사 할부 안내(삼성카드 등)
11. "솔직한 구매 후기" 섹션
12. 법적 고지(과거이력 데이터 출처 한계, 옵션가 기준 등 6개 항목)
13. **총 예상 구매비용** 표: 차량가 + 이전 등록 관련 비용 + 기타 비용 + 배송비(지역별 상이) = 합계
14. 하단 CTA: **`비교하기` / `공유하기`**

**정보 구성 순서 특징**: 가격/진단신뢰 → 옵션/진단 상세 → 이력 → 담당자(사람) → 구매 전환 CTA(온라인/오프라인 두 갈래) → 비용 총액 투명화 → 비교/공유. **"평가사가 직접 진단"이라는 신뢰 메시지가 가격 바로 아래(2번째 블록)에 배치**되는 점이 눈에 띔 — 직영 모델 특유의 신뢰 강조 패턴.
출처: https://www.kcar.com/bc/detail/carInfoDtl?i_sCarCd=EC61275517

모바일 웹 상세 페이지(`m.kcar.com/bc/detail/...`)는 광고 리다이렉트(트래커 픽셀)로 2회 연속 텍스트 추출 실패 → **확인 불가**로 처리.

## 5. 랜딩/홈

홈(`www.kcar.com/`) 구성 순서:
1. 상단 프로모션 인터스티셜/배너 팝업 다수(오늘 하루 보지 않음 옵션 포함) — 안심직거래, 침수차 프로모션, 무료보증 이벤트 등
2. 퀵링크 바: `무료배송` `위클리 특가` `전기차·하이브리드 기획전` `렌트 특가` `테마기획전`
3. **히어로 섹션**: "어떤 차를 찾으세요?" 큰 검색창 + 인기 모델 해시태그 칩 + 3분기 탐색 진입점(모델명/예산/차종)

## 6. 반응형

- 데스크톱(`www.kcar.com`)과 모바일 웹(`m.kcar.com`)이 **별도 서브도메인**으로 분리되어 있음(반응형 그리드가 아니라 디바이스별 별도 사이트 — UA 감지로 자동 리다이렉트하는 구조로 추정). 즉 K Car는 "같은 페이지가 뷰포트에 따라 레이아웃만 바뀌는" 순수 반응형이 아니라 **m.도메인 분리형**.
- 모바일 웹 필터 UI는 데스크톱의 "패널 나열형"과 달리 **드릴다운/바텀시트형**(각 필터 항목을 탭하면 전체화면으로 진입, 뒤로가기로 복귀)으로 전환됨 — 필터 항목 수가 많을 때(11개 카테고리) 모바일에서 화면 전환식으로 바꾸는 패턴.
- 모바일 웹은 상단에 햄버거형 "메뉴열기/닫기"와 "전체메뉴" 오버레이(주요서비스 아이콘 그리드: 내차팔기/차량검색/렌트/안심직거래/할부리스/마이카/전국직영점/보증서비스/브랜드인증관)를 제공 — 데스크톱의 상단 드롭다운 내비게이션이 모바일에서는 **아이콘 그리드형 전체메뉴**로 전환.
- 그리드 열 수(카드 목록의 컬럼 수 변화)는 텍스트 추출만으로는 확인 불가.
- 네이티브 앱은 웹과 별개 코드베이스(iOS/Android 개별 앱, 패키지 `com.encardirect.app`)로 존재 — 웹뷰 하이브리드가 아닌 것으로 보이나 확정은 불가.

## 우리 프로젝트 적용 시사점

1. **카드에 "가격제안가능" 같은 협상 배지 + 특징 태그 2~4개(1인 소유/비흡연/무사고/빠른거래가능 등) 노출**: 우리 매물 카드도 사진·가격 외에 "정보 위계"를 세분화할 여지가 있다. 무사고 뱃지는 이미 계획돼 있다면, 추가로 "가격제안가능"류의 협상 가능 여부, "빠른거래가능" 같은 판매자 태도 배지를 저비용으로 추가하면 정보 밀도를 높일 수 있다(단, A2 원칙상 요청 없는 기능 확장은 지양 — PRD 논의 시 후보로만 제시).
2. **직거래(P2P) 홈은 "서비스 소개 없이 매물 그리드로 바로 진입"하는 마켓플레이스형**이 우리 서비스 성격(개인 직거래)에 더 가까운 참조점이다. 랜딩을 기획할 때 "지금 뜨고 있는 차량" / "최근 등록된 차량" 같은 **2단 그리드 섹션 분리(인기 vs 최신)**는 이미지+카드 UI 개편 시 바로 쓸 수 있는 단순 패턴이다.
3. **모바일에서 필터를 "드릴다운(전체화면 진입 + 뒤로가기)" 방식으로 전환**하는 것은 필터 항목이 많을 때 반응형 대응의 검증된 패턴 — 우리도 필터 개수가 늘어나면 사이드 패널(데스크톱) → 바텀시트/풀스크린(모바일) 전환을 고려할 수 있다.
4. **상세 페이지에서 "신뢰 메시지(진단/평가사 배지)"가 가격 바로 아래 최상단에 배치**되는 점은, 이미지 카드 계약을 확장할 때 대표사진 갤러리 다음으로 "무사고/판매자 신원확인" 같은 신뢰 요소를 상단 배치하는 순서 참고가 될 수 있다.
5. **디자인 언어는 "레드&화이트 대비 + 정보량이 많은 도메인이라 오히려 큼직하고 간결하게"** — 중고차처럼 정보 밀도가 높은 도메인에서 "다 보여주기"보다 "위계를 명확히 좁혀서 보여주기"가 검증된 방향이라는 점은, 우리 UI 개편 시 텍스트/필터 항목을 욱여넣기보다 단계적 노출(요약→상세 펼치기)을 우선 고려할 근거가 된다.
6. **랜딩의 "탐색 의도 분기"(모델명으로 찾기 / 예산으로 찾기 / 차종으로 찾기)**는 검색바 하나만 덩그러니 놓는 것보다 사용자가 진입 시점에 뭘 기준으로 찾을지 모르는 상황을 셋으로 쪼개 보여주는 패턴 — 랜딩 히어로 섹션 설계 시 참고할 만하다(단, 우리는 AI 자연어 검색이 이미 있으므로 검색창 자체가 이 3분기 역할을 흡수할 수도 있어, 별도 3분기 UI가 꼭 필요한지는 PRD에서 판단 필요).
