# 리서치: 중고차 "신뢰속성"(무사고·1인소유·비흡연 등) 데이터 출처·검증·표시 관행

- 조사일: 2026-07-11
- 조사 방법: agent-reach (exa_search + Jina Reader `r.jina.ai`)로 엔카(encar.com), KB차차차(kbchachacha.com), 카히스토리(carhistory.or.kr, 보험개발원 공식 서비스), 기아 인증중고차(cpo.kia.com), 케이카(K Car), 법령 원문(law.go.kr) 및 관련 블로그·기사를 조사
- 목적: 우리 프로젝트(브라운필드 증분)가 매물 카드에서 신뢰속성을 옵션(장비)과 분리 표시하려는 결정의 근거 마련. 우리는 데모라 실제 보험이력·성능기록부 API 연동이 없음(seed 데이터) — "객관 검증이 불가능한 환경에서 정직하게 저장·표시하는 법"이 핵심 질문

---

## 1. 무사고 판정 근거

국내 플랫폼(엔카/SK엔카, K Car, KB차차차)의 "무사고" 기준은 **자동차관리법 시행규칙 별지 제82호 서식 "중고자동차성능·상태점검기록부"**에 근거한다. 핵심 원칙:

- 사고 여부는 **"주요 골격(프레임) 부위의 판금·용접·수리·교환" 유무로만** 판단한다. 주요골격 = 프론트패널, 크로스멤버, 사이드멤버, 휠하우스, 필러패널(A/B/C), 대쉬패널, 플로어패널, 트렁크플로어, 리어패널 등.
- **범퍼, 도어, 후드(보닛), 펜더, 트렁크리드**는 볼트로 탈부착되는 "외판" 부품이라 **교환·판금·도색이 있어도 "무사고"로 표기**된다(자동차관리법상 "단순수리"로 사고에서 제외).
- 이 점검은 **"성능·상태점검자"(자격 보유 진단업체·기사)가 직접 육안·리프트로 차량을 점검해 자기 이름으로 서명·고지**하는 문서다. 법적 책임이 따른다 — law.go.kr 원문: *"중고자동차의 구조·장치 등의 성능·상태를 고지하지 아니한 자, 거짓으로 점검하거나 거짓 고지한 자는 [처벌]"*.
- **보험이력(카히스토리)과 성능기록부의 사고 판정은 서로 별개이며 반드시 일치하지 않는다.** 엔카 공식 설명: *"보험이력정보 유무와 사고진단결과가 반드시 일치할 수 없으며, 보험이력금액의 과소 역시 사고의 경중과 비례하지 않습니다."* (외판 도색·판금·교환은 보험처리해도 사고 판정에 영향 없음)
- 엔카는 자체적으로 더 세분화: **무사고 / 외부패널교환 / 사고(접촉) / 사고** 4단계로 구분해 법정 최소기준보다 세밀하게 표시.
- 엔카·케이카·KB차차차 등의 자체 "진단(엔카진단/케이카진단/KB진단)" 상품은 이 성능기록부 위에 **플랫폼이 직접 재점검하고 오진단 시 금전 보상(예: 최대 100만원)까지 거는 "보증형 무사고"** 상품이다. 다만 언론(시사위크 2021)은 "진단 통과 = 완전 무사고"로 오인되는 문제, 즉 진단이 있어도 보험처리 이력이 있는 차를 "무사고"로 광고하는 사례를 지적한다.

## 2. 1인소유 판정

- **판정 근거는 카히스토리(보험개발원)의 "소유자 변경이력" 항목**이다. 이 항목은 **소유자 변경 "횟수"**만 제공하며, 개인 간 명의변경뿐 아니라 **매매상사(딜러) 간 유통(상품용) 변경까지 전부 포함한 숫자**다(카히스토리 공식 안내: *"소유자 변경이력 정보는 개인 간의 소유 변경... 이외에도 매매상사 간 변경(상품용)까지 모두 포함된 횟수로 제공됩니다"*). 즉 **"소유자 변경 0회" = 신차출고 후 한 명의 등록명의자만 존재**했다는 뜻이며, 이는 **객관적으로 검증 가능**하다.
- 그러나 엔카 등 매물 목록 화면의 **"1인소유" 아이콘 자체는 카히스토리 API 실시간 연동이 아니라 판매자(딜러)가 매물 등록 시 선택하는 "핫마크" 뱃지**다. 엔카 공식 아이콘 안내 페이지(dc_carsearch_v13_pp0.htm)는 이 아이콘들 전체에 대해 다음 면책 문구를 명시한다: **"실제차량이 핫마크의 뜻과는 다를수 있으니 사실여부를 직접 확인해 보시기 바랍니다."**
- 즉 실무 관행은 **"아이콘/뱃지 = 판매자 자기신고" + "구매자가 카히스토리로 교차검증"** 구조다. 아이콘 자체가 검증된 데이터는 아니지만, 검증 가능한 원천 데이터(소유자 변경 횟수)가 별도로 존재한다는 점에서 비흡연과는 성격이 다르다.

## 3. 비흡연

- **비흡연은 객관적 검증 수단이 없다.** 보험이력·성능기록부 어디에도 "흡연 여부" 데이터 항목은 존재하지 않는다(카히스토리 항목: 일반사양/용도이력/특수보험사고/소유자변경/차량번호변경/보험사고이력 — 흡연 관련 항목 없음).
- 완전히 **판매자(또는 딜러)의 자기신고성 마케팅 뱃지**다. KB차차차는 "핫마크" 체계에서 비흡연을 **1인소유·여성소유·장애인·특옵션·급매 등과 같은 카테고리("상태 뽐내기" 마크)**로 취급 — 즉 플랫폼 스스로도 이를 검증된 스펙이 아니라 **판매 촉진용 광고 마크**로 분류한다.
- 엔카도 동일 아이콘 체계이며, 위 2번의 면책 문구가 비흡연에도 그대로 적용된다. **플랫폼들은 "검증됨"이라고 절대 주장하지 않고, 오히려 명시적으로 "사실과 다를 수 있다"고 고지해 면책한다.**

## 4. 데이터 표시 관행 (카드/상세에서 옵션과의 구분)

- **옵션(장비)**: 성능기록부/제조사 사양 기반의 체크리스트 형태(예: ABS, 열선시트, 선루프 등 boolean 목록). "차량옵션 및 상세설명" 섹션으로 명확히 분리되고, 세일카 등 일부 플랫폼은 옵션 옆에 *"아래의 옵션정보는 실제차량의 옵션과 다를 수 있습니다. 담당자와 꼭 확인하세요"*라는 면책도 붙인다.
- **신뢰/특징 속성(1인소유·비흡연·여성소유·무사고·급매·특옵션 등)**: 옵션 체크리스트와 별도로 **작은 아이콘/뱃지("핫마크") 형태**로 매물 카드 상단 또는 썸네일 옆에 노출된다. 엔카는 이를 개인/여성/비흡연/1인소유/장애인용/LPG/제조사AS/준신차/임시번호/짧은km/양호/정비/오디오/튜닝/특옵션/4륜구동/카드/할부/리스승계/세금계산서/최저가/급매/가격절충/핫마크/쿨마크/정식수입/24시간상담 등 **27종의 아이콘 팔레트**로 표준화해 운영한다.
- **근거 명시 문구 관행**: 기아 인증중고차(cpo.kia.com)는 매물 카드에 **"보험이력없음"**이라는 문구를 직접 노출한다 — 이는 "무사고"라는 모호한 주장 대신, **"카히스토리 조회 결과 보험처리 이력이 없다"는 구체적 데이터 출처를 밝히는 표현**이다. "무사고"(주관적·법정기준 해석 필요)보다 "보험이력없음"(원천 데이터 그대로 노출)이 더 정직하고 검증 가능한 근거 제시 방식으로 평가된다.
- **KB차차차 "실차주" 마크**: 매물이 딜러가 아닌 실제 소유자 본인이 직접 등록했음을 뜻하는 마크로, 이 또한 검증 주체(KB차차차 진단매니저)가 있는 것과 없는 것을 구분해 운영한다(예: "KB진단"은 매니저 실명·얼굴 공개 + 오진단 보상 100만원 vs 일반 매물은 보상 없음).

## 5. 검증 티어 — 실제 3단 구분이 존재하는가

**존재한다.** 실무에서 관찰되는 구조는 다음과 같이 3단으로 정리된다.

| 티어 | 근거 | 예시 |
|---|---|---|
| **① 객관 검증(제도적 근거)** | 법정 서식(성능·상태점검기록부, 자동차관리법 시행규칙 별지82호) + 보험개발원 공식 데이터(카히스토리) | 사고유무(주요골격), 주행거리, 소유자변경횟수, 침수/전손/도난, 용도이력(렌트/영업) |
| **② 플랫폼/딜러 진단(사업자 책임 근거)** | 플랫폼 자체 진단 인력이 재점검 + 오진단 시 금전보상 명시 | 엔카진단(90일/5000km 보상), K Car 진단, KB진단(100만원 보상), 기아 인증중고차 200개 항목 검수 |
| **③ 판매자 자기신고(면책)** | 판매자가 광고 목적으로 선택하는 뱃지, 플랫폼이 사실확인 책임을 지지 않음(명시적 면책 문구 부착) | 1인소유·비흡연·여성소유·장애인용·급매·특옵션 등 "핫마크" |

1인소유는 ①(카히스토리 소유자변경이력)로 검증 가능하지만 매물 카드의 아이콘 자체는 대개 ③(자기신고)로 운영된다는 점이 중요한 뉘앙스다. 비흡연은 원천적으로 ①·②가 존재하지 않고 항상 ③이다.

## 보험이력(카히스토리)·성능기록부 데이터 항목 표

### 카히스토리(보험개발원, carhistory.or.kr) 제공 항목

| 항목 | 설명 |
|---|---|
| 자동차 일반사양 | 제작사, 차명, 연식, 배기량, 사용연료, 차체형상, 최초 보험가입일자 |
| 자동차 용도이력 | 대여용(렌터카), 영업용(택시 등), 관용 등 사용 이력 — 과거 등록번호판 분류기호로 판별 |
| 특수보험 사고정보 | 전손 보험사고, 침수 보험사고(분손 포함), 도난 보험사고 |
| 소유자 변경이력 | 소유자 변경 **횟수**(개인 간 + 매매상사 간 유통 포함, 상세 사유는 비공개) |
| 차량번호 변경이력 | 차량번호 변경 횟수(최초 번호판부터 현재까지) |
| 보험사고 이력정보 | 내차 보험처리(자기차량손해) / 상대 보험처리(대물가해) / 타인 재물가해정보, 사고일자·수리(견적)비용(부품/공임/도장 분리), 일부 건은 수리부위 다이어그램·사진 제공 |
| 주행거리 이력 | 보험사·중고차쇼핑몰이 제출한 최근 주행거리(비검증 참고용, 정확성 보장 안 함) |
| 첨단안전장치 장착여부 | ABS, ESC, BSD, FCW 등 제조사 사양(○/△/－/☓ 4단계, "확인불가"는 실제와 다를 수 있음 명시) |
| 차량기준가액/시세 | 보험개발원 AI 산출 표준시세·개별시세(참고용, 실거래가 아님) |
| **한계** | 12개 손해보험사의 1996년 이후 보험수리 지급기록에 근거 — **자기부담 미가입 기간, 현금수리(보험미처리), 비가입 손보사 처리분 등은 조회 불가**. "미가입기간"으로 명시 표기됨 |

### 중고자동차성능·상태점검기록부(법정 서식, 자동차관리법 시행규칙 별지 82호) 제공 항목

| 항목 | 설명 |
|---|---|
| 사고유무 판정 | 주요골격(프론트/리어/사이드패널, 크로스멤버, 사이드멤버, 휠하우스, 필러A/B/C, 대쉬패널, 플로어패널 등) 판금·용접·교환 여부만으로 판정 |
| 외판부위 상태 | 범퍼·도어·후드·펜더·트렁크리드의 교환/판금/도색 — 사고 판정에서는 제외되나 상태정보로는 기록 |
| 원동기(엔진) 상태 | 오일누유, 냉각수 누수, 이음 등 |
| 변속기 상태 | 오일누유, 이음, 자동/수동 변속 상태 |
| 주요 안전장치 | 제동장치, 조향장치, 전기장치 이상 유무 |
| 사고이력 관련 부가정보 | 침수/색상변경/차대번호 변조 여부 등 |
| **책임 주체** | "성능·상태점검자"가 자격을 걸고 직접 서명 — 허위 고지 시 법적 처벌 규정(자동차관리법) |
| **보증** | 점검 후 일정 기간/거리 내 보증(법정 최소 기준 + 플랫폼별 자체 연장 보증) |

## 우리 프로젝트 적용 시사점

**전제**: 우리는 데모이며 보험개발원 카히스토리·성능점검기록부 API 연동이 없다(seed 데이터로 매물을 채움). 위 리서치를 근거로 아래와 같이 처리하는 것을 제안한다.

### (1) 신뢰속성 스키마 분리 방향
- 업계 관행 자체가 "옵션(장비 체크리스트)"과 "신뢰/특징 뱃지"를 **테이블/컬럼 레벨에서 이미 분리**해서 다룬다(성능기록부 vs 핫마크 아이콘). 우리도 `listing_options`(장비, boolean 다건)와 `listing_trust_badges` 또는 `listings` 테이블의 별도 컬럼 그룹(예: `is_single_owner`, `is_non_smoker`, `accident_status`)으로 **명확히 분리**하는 것이 실제 업계 데이터 모델과 일치한다.
- `accident_status`(무사고 여부)는 옵션이 아니라 **"차량 상태 판정" 필드**로, 이상적으로는 enum(`무사고`/`외판교환`/`사고`) 형태가 업계 표준(엔카 4단계)에 부합한다. 데모에서는 최소 `무사고`/`사고` 2단계로 단순화해도 무방(A2 단순함 원칙).

### (2) 검증 티어 표기 — 우리 데모에 그대로 적용 가능한 3단 구조
- **① 객관 검증**: 데모에서는 실제 카히스토리 연동이 없으므로 이 티어는 **"확보 불가"**임을 스스로 인지해야 함. 흉내 내려 하지 말고(가짜 API 연동처럼 보이는 UI를 만들지 말고), 이 티어가 필요한 속성(무사고·1인소유 등)은 ③으로 격하해 표시하는 것이 정직하다.
- **② 플랫폼 진단**: 데모에는 실제 진단 인력이 없으므로 이 티어도 존재하지 않음.
- **③ 판매자 자기신고**: 우리 서비스의 매물 등록은 실제로 일반 판매자(개인 직거래) 입력 기반이므로, **모든 신뢰속성(무사고·1인소유·비흡연)을 "③ 자기신고"로 통일 처리하는 것이 가장 정직한 모델**이다. 이는 실제로 엔카·KB차차차의 "핫마크" 운영 방식과 동일한 선례가 있어 억지스럽지 않다.

### (3) 데모에서의 정직한 처리 — 구체 제안
- UI 라벨에 **"판매자 제공 정보"** 또는 **"판매자 신고"**라는 문구를 신뢰속성 뱃지 옆(또는 상세 페이지 하단)에 명시해, 엔카의 *"실제차량이 핫마크의 뜻과는 다를수 있으니 사실여부를 직접 확인해 보시기 바랍니다"*와 같은 면책 관행을 재현한다. 이렇게 하면 "검증 없이 왜 표시하냐"는 질문에 업계 선례로 답할 수 있다.
- 옵션(장비) 체크리스트와 신뢰속성 뱃지는 **카드/상세 화면에서 시각적으로 분리된 섹션**(아이콘 줄 vs 체크리스트 표)으로 배치 — 이는 CLAUDE.md의 "요청 범위 내 최소 변경" 원칙과도 맞는, 이미 검증된 업계 UI 패턴이므로 새로 발명할 필요 없음.
- "무사고"라는 표현은 업계에서도 논란(외판 교환은 무사고로 표기되는데 소비자는 오인)이 있는 용어이므로, 데모에서 문구를 쓸 때 과장하지 않는 것이 좋다. 기아 CPO의 "보험이력없음"처럼 **출처를 명시하는 짧은 표현**(예: "판매자 무사고 신고")을 쓰면 A1(가정 명시) 원칙에도 부합한다.

### (4) 각 속성별 데이터 출처와 한계 요약
| 속성 | 실제 업계 데이터 출처 | 검증 가능성 | 우리 데모에서의 처리 |
|---|---|---|---|
| 무사고 | 성능·상태점검기록부(법정 서식, 자격자 서명) + 보험이력(참고용, 불일치 가능) | 원칙적으로 가능(자격 있는 검사자 필요) | 검증 인프라 없음 → "판매자 신고" 라벨 필수 |
| 1인소유 | 카히스토리 소유자변경이력(횟수, 객관적 원장 데이터) | 가능(카히스토리 조회로 교차검증) — 단 매물 카드의 아이콘 자체는 대개 자기신고 | 카히스토리 연동 없음 → "판매자 신고" 라벨 필수 |
| 비흡연 | 없음(원천적으로 검증 불가한 속성) | 불가능 — 업계도 순수 마케팅 뱃지로 취급 | 그대로 "판매자 신고"로 표시(업계와 동일 수준의 정직성 확보) |
