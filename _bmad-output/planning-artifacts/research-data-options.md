## 질문별 소견

### 1. 옵션 데이터 구조 — 분류 체계가 있는가?

**확인됨.** 엔카는 자유 텍스트가 아니라 **회사가 관리하는 통제된 옵션 사전(카테고리 + 표준 항목명)**을 쓴다.

엔카 공식 공지사항(2017-12-22, 2018-01-06 적용)에 따르면:
- 옵션은 **카테고리로 분류**되며, 카테고리 자체가 회사 정책에 따라 개편된다. 예: "파워 도어록"은 (안전 → 내장/외관)으로, "가죽시트"는 (내장 → 시트)로 카테고리가 변경됨.
- 옵션 **항목 자체도 추가/변경**된다.
- 옵션 **표기법(이름)도 표준화**된다. 예: "후방감지센서" → "주차감지센서(후방)"로 명칭 통일.
- 판매자는 등록된 매물의 옵션을 직접 수정해야 하는 마이그레이션 안내까지 있었음 — 즉 판매자가 **정해진 항목 중에서 체크**하는 방식이지, 자유 텍스트 입력이 아니다.

실제 매물 상세 페이지(m.encar.com)에서 확인되는 옵션 카테고리 탭: **외관/내장, 안전, 편의/멀티미디어, 시트, 기타옵션** (5개 그룹).

제네시스 차종에 한해서는 한 단계 더 나아가 **제조사 패키지 단위** 검색도 지원한다(2025-05, 카가이 보도): 스포츠 패키지(기본/스포츠), 외장 컬러(유광/무광), 휠 타입, 내장 디자인(스탠다드/시그니처 디자인 셀렉션/스포츠 디자인 셀렉션), 패키지(드라이빙 어시스턴스 패키지, 파퓰러 패키지 등), 선택 품목(프리뷰 전자제어 서스펜션, 파노라마 선루프 등). 이는 "제조사 기본옵션 vs 선택(추가)옵션" 구분에 대응하는 사례다. 다만 이 패키지 단위 구조는 **제네시스에 한정된 업계 유일 기능**이며, 일반 국산차 전체에 적용되진 않는다.

KB차차차·K Car는 검색 필터에 "옵션" 항목이 존재한다는 사실은 다건의 블로그/비교 글에서 확인되나, **옵션 카테고리 체계(안전/편의 등 세부 분류)의 공식 문서나 구체적 목록은 이번 조사에서 찾지 못함 — 확인 불가.**

### 2. 보편 vs 희소 구분 — 모든 옵션을 다 보여주나, 셀링포인트만 큐레이션하나?

**부분적으로 확인됨. 결론: 업계는 "전량 나열"이 기본이고, 우선순위 큐레이션은 드물다.**

- 엔카 매물 상세는 옵션을 **아이콘 목록으로 전량 노출**하며, 있음/없음을 토글로 표시한다(기존 reference-encar.md 기록과 일치). 즉 희소도 기준으로 걸러내지 않고 사전에 등록된 전체 항목을 다 보여주는 방식.
- 중고차 플랫폼 "첫차"의 UX 분석 글(weeklyuxuichallenge, 2023)에 따르면 첫차도 "주요 옵션"이라는 **별도 섹션**을 상세 페이지 상단부(가격·주행거리·수리이력·보험사고 다음, FAQ 앞)에 배치하지만, 아이콘 활성/비활성 방식이라 **어떤 기준으로 "주요"를 선정하는지는 불명확** — 이 UX 리뷰어도 "활성화된 아이콘이 하나도 없어서 옵션이 다 있는 줄 오해했다"는 설계 결함을 지적함. 즉 "주요 옵션"이라는 이름을 달았지만 실제로는 **표준 항목 세트를 전량 노출하는 것**이지, 희소성 기준 큐레이션은 아닌 것으로 보인다.
- 명확한 **희소도/가치 기반 우선순위 로직**을 공식적으로 카드·목록에 적용한 사례는 확인하지 못함 — "핵심 옵션 N개만 노출" 같은 대놓고 큐레이션하는 UI 패턴은 이번 조사 범위(엔카·K Car·KB차차차·첫차)에서 발견되지 않았다.
- 대신 **"가치·감가율" 기준의 우선순위 데이터**는 존재한다 (질문 3 참고) — 이게 희소·셀링포인트 옵션을 가려내는 데 쓸 수 있는 가장 근접한 실제 근거다.

### 3. 옵션의 신차 당시 금전 가치 환산 — 희소 옵션 강조 사례

**확인됨. 두 가지 뚜렷한 사례.**

**(A) 현대/제네시스 인증중고차 (certified.hyundai.com)** — 개별 매물 상세에 **"선택 옵션" 섹션을 원가와 함께 명시**한다. 실제 매물(2023 그랜저 IG) 예시:
> "4개의 선택사양(총 338만원 상당)이 적용되어 있어요!" — 플래티넘 플러스 146만원 / 현대 스마트 센스Ⅰ 99만원 / 프리미엄 초이스 70만원 / 18인치 알로이 휠&타이어 23만원

또한 "신차가격대비 X원 절약" 문구로, 신차 기본가 + 옵션가를 합산한 기준가 대비 현재 판매가의 절약분을 보여준다. 공식 설명: "신차가는 현대 인증중고차에서 산정한 신차 기본가격 + 옵션가 기준으로 산출". **이는 옵션 하나하나를 개별 항목·개별 금액으로 명시하는, 질문3에 정확히 부합하는 사례다.**

**(B) 엔카닷컴 — "옵션별 감가율 방어 효과" 데이터 (모빌리티포스트, 2025-04-29 보도)**
엔카닷컴은 **60개 이상의 범용 선택 기능**을 시세·감가율 분석에 반영하고 있으며, 빅데이터로 산출한 항목별 감가율 방어 효과는 다음과 같이 랭킹화되어 있다:
- 선루프: 차 가격의 **1.5~2%** 감가 방어 (가장 큰 효과)
- 내비게이션: **1%**
- HUD(헤드업 디스플레이): **1%**
- 차로이탈방지보조: **0.5~1%**
- 이 외 크루즈 컨트롤, 후측방 경고 시스템, 서라운드 뷰 모니터도 감가율 방어에 기여한다고 명시(구체 수치는 미공개).

이 데이터는 **"인기·수요 대비 옵션별 시세 반영률"** 을 정량화한 것으로, 우리 프로젝트가 찾는 "희소·가치 기준 우선순위"에 가장 가까운 실증 사례다. 다만 이 수치는 엔카 내부 빅데이터 기반이라 외부에서 원본 리스트(60개+ 전체 순위)를 확인할 수는 없었음 — 상위 7개 항목만 보도자료로 공개됨.

### 4. 카드·목록 표시 관행 — 태그 개수·순서·상세 아코디언

**부분 확인.**
- 상세 페이지: 엔카는 카테고리 탭(외관/내장, 안전, 편의/멀티미디어, 시트, 기타옵션)으로 나눠 펼치는 구조 — "아코디언"이라기보다 **탭/섹션 전환** 방식으로 확인됨(정확한 UI 컴포넌트명은 페이지 렌더링 확인이 필요해 "탭 유사 구조"로만 확언 가능).
- 목록(카드) 단계에서 옵션을 몇 개까지 노출하는지 — 즉 매물 검색 결과 리스트 카드에 옵션 태그가 몇 개 붙는지에 대한 **구체적 숫자나 정렬 규칙은 확인하지 못함**(엔카 카드 UI를 실제 렌더링해 세는 검증은 이번 조사 범위—텍스트 기반 exa_search/Jina Reader—로는 한계가 있었음). **확인 불가로 명시.**
- 첫차(chutcha)는 최근(2025년, 케이벤치 보도) 매물 정보 화면을 개편해 **AI가 이력을 요약해 "중요도 기준으로 선별한 핵심 정보를 최상단에 문장형으로 노출"**하는 방향으로 가고 있고, "차량 출고 당시의 유료 옵션 리스트와 실제 출고가"도 별도로 강조해서 보여준다 — 이는 옵션을 포함한 정보 전체에서 "양보다 우선순위·해석"을 지향하는 최신 업계 트렌드로 참고할 만하다.

### 5. 표준화 — 자유 텍스트 vs 표준 코드/사전

**확인됨.** 엔카는 위 1번에서 확인했듯 **표준 옵션 사전(카테고리+명칭 통제)**을 쓰며, 명칭 동의어 통합("후방감지센서"→"주차감지센서(후방)")도 회사가 직접 수행한다. 자유 텍스트가 아니다.

참고로 독일 완성차(BMW 등)는 **공장 출고 시점의 옵션 코드(3자리 영숫자, VIN 연동)** 를 표준으로 쓰지만, 이는 제조사 자체 체계이며 국내 중고차 플랫폼이 이 코드를 그대로 노출하는 사례는 확인하지 못했다(국내 플랫폼은 한글 표준명 사전을 자체 운영).

## 우리 프로젝트 적용 시사점

### 옵션 데이터 구조 방향
현재 우리 seed는 `options text[]`(자유 배열)로 저장 중인데, 엔카 사례가 보여주듯 업계는 **자유 텍스트가 아니라 통제된 옵션 사전**(표준 명칭 + 카테고리)을 쓴다. 데모 프로젝트 규모에선 굳이 문자열 정규화 테이블까지 만들 필요는 없지만(A2 단순함 우선 원칙), 최소한 애플리케이션 레이어에 **"표준 옵션명 목록" 상수 하나**를 두고 그 목록 밖의 값은 seed에 넣지 않는 규칙 정도는 지키는 게 좋다 — 이렇게 해야 "보편 vs 희소" 판정 로직(아래)이 일관되게 동작한다.

### 보편/희소 구분 구현법
업계에서 "옵션 자체를 안 보여주는 큐레이션"은 못 찾았지만, **가치·희소성 순위를 매기는 근거 데이터**(엔카의 감가율 방어 효과 랭킹)는 확인했다. 이를 응용해 두 갈래로 구현 가능:

1. **보편 옵션 하드코딩 목록** (숨김/저순위 대상): 스마트키, 블루투스, 후방카메라, 열선시트, 자동에어컨, 크루즈컨트롤 등 — 엔카가 표준 사전에 포함시킨 "흔한 항목"과 겹치는 것들. 이 목록은 코드에 상수 배열로 두고 프로젝트 옵션 데이터와 대조해 필터링.
2. **희소도 순위 부여**: 엔카의 감가율 방어 랭킹(선루프 > 내비 ≈ HUD > 차로이탈방지보조)을 참고해, 우리 seed 옵션들에 수동으로 우선순위 점수를 매긴 매핑 테이블(옵션명 → priority 정수)을 하나 만드는 게 실용적이다. 정교한 통계 대신 "엔카가 실증한 순위 + 우리 seed에 실제로 등장하는 희소 옵션(오토파일럿, 파노라마선루프, V2L 등)을 수동 추가"하는 식으로 단순하게 구성.

### 카드 표시 우선순위 규칙
1. 카드에는 옵션 전체가 아니라 **N개(예: 3~4개)만** 노출 — 순위 매핑 테이블에서 priority가 높은 순으로 정렬해 상위 N개 표시.
2. 위 1번의 "보편 옵션 하드코딩 목록"에 있는 항목은 priority 매핑에 없어도 항상 최하위로 취급해 카드에서 밀려나게 처리(즉 희소 옵션이 하나도 없는 매물만 어쩔 수 없이 보편 옵션을 채워 넣는 fallback).
3. 상세 페이지에서는 엔카식으로 **카테고리 탭/섹션(안전·편의·외관/내장·시트·멀티미디어)** 으로 전체 옵션을 나눠 보여주면 되고, 여기선 희소도 필터링을 적용하지 않고 전량 노출(업계 관행과 일치).
4. "옵션의 금전 가치"까지 보여주는 건(현대 인증중고차식) 데모 프로젝트 스코프상 과설계 소지가 있다 — seed 데이터에 옵션별 가격 정보가 없다면 이번 스코프에서는 제외하고, 우선순위 정렬(위 1~2번)까지만 구현하는 게 A2 원칙에 맞다.
